## 3. 방향별 상세 분석

### 3.1 방향 1 — Centered NCE 단독 (H(p̄) 제거)

**질문:** H(p̄) 없이 L_ent + L_centered_NCE만으로 collapse가 방지되는가?

| Run | Loss | acc | cat% | H_pbar_final |
|---|---|---|---|---|
| E1-a | L_ent | 0.1463 | 93.4% | 0.000083 |
| E1-b | L_ent + NCE(w=1,τ=1) | 0.1998 | 87.1% | 0.927 |
| E1-c | L_ent + NCE(w=2,τ=1) | 0.2718 | 77.5% | 1.067 |

**관찰:** NCE weight를 높일수록 cat% 감소(93%→77%)하지만 acc 복구에는 실패한다. E1-c는 최대 w=2에서도 acc=0.272에 머문다. H_pbar_final이 E1-b(0.93), E1-c(1.07)로 E1-a(0.00008)보다 크게 오르지만, cat collapse를 막기에는 부족하다.

**결론: 방향 1 기각.** H(p̄) 없이 NCE 단독으로 collapse를 억제하는 것은 불가능하다. NCE는 반드시 H(p̄)와 결합해야 한다.

---

### 3.2 방향 2 — Relational Anchor

**질문:** Text의 centered relational structure를 prototype에 KL match하면 centered NCE보다 강한가?

| Run | Loss | acc | cat% | H_pbar_final | mean_entropy |
|---|---|---|---|---|---|
| E2-a | L_ent - 2H(p̄) + L_rel(w=1) | 0.6746 | 13.1% | 2.290 | 0.477 |
| E2-b | L_ent + L_rel(w=1) | 0.1473 | 93.2% | 0.047 | 0.144 |

**관찰 (E2-a):** H(p̄)와 결합 시 acc=0.6746으로 CALM v1(+2.88pp), CALM v2.2(+0.51pp) 모두 초과. 예측 분포가 균등에 가깝게 형성된다 (airplane 9.9%, automobile 10.5%, bird 9.4%, **cat 13.1%**, deer 10.5%, dog 9.5%, frog 10.0%, horse 8.4%, ship 9.7%, truck 9.1%). L_rel이 prototype의 relational geometry를 text space에 anchor해서 prototype discriminability를 유지하는 것으로 해석된다.

**관찰 (E2-b):** H(p̄) 없으면 L_rel도 collapse를 막지 못한다(cat=93.2%). E2-a와 E2-b의 acc 격차(0.6746 vs 0.1473)는 H(p̄)의 역할이 결정적임을 다시 확인한다.

**한계:** E2-a는 tau_t=1.0, tau_nce=1.0의 기본 설정만 검증했다. 다른 τ 조합이나 w_rel 범위는 미탐색.

**결론: E2-a 유망, E2-b 기각.** `L_ent - 2H(p̄) + L_rel(w=1)` 조합이 현재 2위.

---

### 3.3 방향 3 — Output Correction (Frozen, adaptation-free)

**질문:** Parameter 업데이트 없이 frozen CLIP의 logit 후처리만으로 accuracy를 올릴 수 있는가?

| Run | 방법 | acc | cat% | H_pbar_final | 소요(초) |
|---|---|---|---|---|---|
| E3-a | zero-shot raw | 0.3796 | 53.0% | 2.092 | 15.2 |
| E3-b | zero-shot centered | 0.3712 | 48.3% | 1.791 | 15.7 |
| E3-c | LAME σ=0.1, iter=5 | 0.1000 | 100.0% | 2.066 | 13.7 |

**관찰 (E3-b vs E3-a):** Centering이 zero-shot 설정에서 acc를 오히려 -0.84pp 악화시킨다. cat%는 53%→48%로 소폭 감소하지만 acc는 하락한다. 이는 text embedding의 effective rank=2.03(Gate 1, reports/23)으로 인해 centered space의 discriminability가 더 낮기 때문으로 해석된다. 공통 방향 성분을 제거하면 이미 near-collinear한 text 임베딩이 더 구분하기 어려워진다.

**관찰 (E3-c LAME):** Gaussian noise 하에서 affinity matrix가 cat-biased 특성을 반영하여 label propagation이 cat 방향으로 증폭된다. 결과적으로 모든 샘플이 cat으로 분류(cat=100%, acc=0.10). CIFAR-10에서 uniform prior를 가정하면 최소 acc=0.10이므로 이는 사실상 최악의 결과다.

**결론: 방향 3 전면 기각.** Centering 단독 역효과, LAME은 cat sink를 오히려 증폭. Adaptation 없는 후처리 방향은 이 설정에서 작동하지 않는다.

### 3.4 방향 4 — Augmentation Consistency (Horizontal Flip)

**질문:** Flip consistency가 collapse 방지 또는 accuracy 개선에 기여하는가?

| Run | Loss | acc | cat% | H_pbar_final | mean_entropy |
|---|---|---|---|---|---|
| E4-a | L_ent + L_flip(w=1) | 0.1527 | 92.6% | 0.006 | 0.150 |
| E4-b | L_ent - 2H(p̄) + L_flip(w=1) | **0.6760** | 13.1% | 2.264 | 0.525 |

**관찰 (E4-b):** 이번 sweep 전체 최고 acc=0.6760. CALM v1 대비 +3.02pp, CALM v2.2 대비 +0.65pp. 예측 분포가 E2-a와 거의 동일한 패턴으로 균등하게 형성된다(cat 13.1%). mean_entropy=0.525로 E2-a(0.477)보다 약간 높다 — flip consistency가 모델이 더 불확실한(덜 날카로운) 예측을 유지하게 한다는 해석과 일치한다.

**관찰 (E4-a):** H(p̄) 없으면 즉시 collapse(cat=92.6%). flip consistency 단독은 anti-collapse 역할을 전혀 하지 못한다.

**비용:** 추가 forward 1회(flip 이미지). Gradient는 원본에서만 역전파.

**E4-b vs E2-a (0.6760 vs 0.6746):** 미묘한 차이(0.14pp)이며 단일 run으로는 통계적 유의성 불명확. Flip의 invariance 압박이 relational KL 매칭보다 gaussian_noise 환경에서 약간 더 효과적일 가능성이 있다. 단, gaussian noise가 flip-invariant 특성이 있어서 flip consistency가 유효한 inductive bias가 됨을 시사한다.

**결론: E4-b 유망(전체 최고), E4-a 기각.** 공식: `L_ent - 2H(p̄) + L_flip(w=1)`.

### 3.5 방향 5 — Adaptive Prior (Teacher EMA)

**질문:** Uniform 대신 frozen teacher EMA prior를 target으로 하면 skewed distribution에서도 작동하는가?

| Run | Loss | acc | cat% | H_pbar_final | mean_entropy |
|---|---|---|---|---|---|
| E5-a | L_ent + KL(p̄‖π̂)(λ=2) | 0.5614 | 30.2% | 2.083 | 0.433 |

**관찰:** acc=0.5614로 BATCLIP(-4.46pp), CALM v1(-8.44pp) 모두 하회. Cat%=30.2%로 완전 collapse는 아니지만 여전히 높다. BATCLIP 보다 낮은 acc는 teacher prior가 오히려 성능을 해치고 있음을 의미한다.

**해석:** Frozen teacher도 gaussian_noise에서 cat-biased 예측을 산출한다. EMA로 π̂를 누적하면 시간이 지날수록 cat 방향으로 prior가 편향된다. Student가 이 cat-biased π̂를 target으로 삼으면 오히려 cat쪽으로 유도되는 self-defeating cycle이 형성된다. EMA momentum=0.9이면 각 배치의 teacher 예측이 빠르게 π̂에 반영되어 cycle이 가속된다.

**한계:** EMA momentum(0.9) 및 λ(2.0)은 단일 설정만 검증. 낮은 momentum(예: 0.99)이나 첫 배치만 teacher로 static prior를 쓰는 방식은 미탐색. 그러나 근본 문제(teacher도 oisy image에서 cat-biased)는 momentum 변경으로 해결되지 않을 가능성이 크다.

**결론: 방향 5 실패.** 구조 개선 없이 재실험 불필요.

---

### 3.6 방향 6 — Active-set 진단

**질문:** Cat sink에서 top-1=cat인 샘플의 top-3, top-5에 true class가 얼마나 남아있는가?

| 측정 | R=2 | R=3 | R=5 |
|---|---|---|---|
| topR_raw_recall (전체) | 0.534 | **0.652** | 0.807 |
| topR_cen_recall (전체) | 0.516 | 0.641 | 0.805 |
| cat→topR_raw_recall | 0.343 | **0.494** | 0.708 |
| cat→topR_cen_recall | 0.317 | 0.472 | 0.695 |

참고: n_cat_raw=5,305 (전체 10,000 샘플 중 53.0%가 cat으로 예측됨).

**관찰:**
- 전체 샘플 기준: top-3에 true class가 65.2% 존재. 판단 기준(>60%) 달성.
- Cat으로 잘못 예측된 샘플 기준: top-3에 true class가 49.4%, top-5에 70.8% 존재.
- Centered logit의 topR recall이 raw logit보다 낮다(top-3: 64.1% vs 65.2%). 방향 3과 동일한 패턴 — centering이 zero-shot에서 discriminability를 개선하지 않는다.

**함의:** Cat으로 잘못 분류된 샘플의 절반 이상이 top-3 이내에 correct class를 보유한다. Re-ranking 기반 TTA(top-k에서 re-score)가 이론적으로 유효한 방향임을 시사한다. 단, re-ranking 자체가 supervision 없이 어떻게 correct class를 선택할지는 별도 설계 필요.

**결론: top-3 recall=65.2% > 기준 60%. 방향 6 기준 달성. Re-ranking 기반 TTA 추가 탐색 권장.**

### 3.7 방향 7 — Weight Anchor (Parameter Regularization)

**질문:** LayerNorm이 원래에서 멀어지지 않게 L2 제한하면 collapse가 줄어드는가?

| Run | Loss | acc | cat% | H_pbar_final |
|---|---|---|---|---|
| E7-a | L_ent + L2(w=0.1) | 0.1523 | 92.6% | 0.122 |
| E7-b | L_ent + L2(w=1.0) | 0.1840 | 88.0% | 0.406 |

**관찰:** E7-a(acc=0.1523)는 E1-a(L_ent only, acc=0.1463)와 사실상 동일. L2 anchor w=0.1로는 entropy collapse를 막지 못한다. E7-b(w=1.0)는 약간 개선(acc=0.184)되지만 여전히 collapse 상태(cat=88%). w가 클수록 adaptation 자체가 억제되지만 그렇다고 accuracy가 의미있게 오르지 않는다.

**결론: 방향 7 전면 기각.** L_ent collapse의 원동력을 L2 anchor로 상쇄하는 것은 불가능하다. 근본 해결책은 H(p̄)와 같은 marginal entropy 제어 항이다.

## 4. 핵심 발견

### 4.1 H(p̄)는 필수 구성요소

H(p̄) 없는 모든 run(E1-b/c, E2-b, E4-a, E7-a/b)이 예외 없이 collapse(cat% 77~100%). H(p̄)를 포함한 run(E2-a, E4-b)은 모두 CALM v2.2를 초과한다. 이 대비는 단순하고 명확하다.

| H(p̄) 포함 여부 | run 수 | acc 범위 | cat% 범위 |
|---|---|---|---|
| 포함 | 2 | 0.6746-0.6760 | 13.1% |
| 미포함 (adaptation) | 8 | 0.1463-0.5614 | 30.2-93.4% |
| frozen (무적응) | 3 | 0.1000-0.3796 | 48.3-100.0% |

### 4.2 방향별 최종 판정

| 방향 | 판정 | 이유 |
|---|---|---|
| 1. NCE 단독 | ❌ 기각 | NCE 단독 anti-collapse 불가 |
| 2. Relational Anchor | ✅ 유망 (E2-a) | H(p̄)+L_rel = 0.6746, 2위 |
| 3. Output Correction | 💀 전면 기각 | centering 역효과, LAME 최악 |
| 4. Aug Consistency | ✅ 최고 (E4-b) | H(p̄)+L_flip = 0.6760, 1위 |
| 5. Adaptive Prior | ❌ 실패 | self-defeating cycle |
| 6. Active-set | ✅ 조건부 | top-3=65.2% > 60% 기준 |
| 7. Weight Anchor | ❌ 전면 기각 | collapse 억제 불가 |

### 4.3 E4-b vs E2-a 미묘한 차이

E4-b(0.6760) vs E2-a(0.6746) 격차는 0.14pp. 단일 run이므로 통계적 유의성은 불명확하다. 두 방향의 예측 분포는 거의 동일하다(둘 다 cat 13.1%). 차이점은 mean_entropy — E4-b(0.525) > E2-a(0.477) — flip consistency가 과적응을 억제하면서 더 높은 불확실성(= 덜 날카로운 예측)을 유지함을 시사한다.

## 5. 제한사항

1. **단일 corruption 평가:** 전체 실험이 gaussian_noise sev=5에서만 수행됨. E4-b의 flip consistency 이득이 다른 corruption에서도 유지되는지 불명확. 특히 flip-sensitive한 corruption(예: pixelate, jpeg_compression)에서는 이득이 다를 수 있다.

2. **단일 run (seed=1):** 각 방향당 1-2 run. 결과의 분산을 추정할 수 없다. E4-b vs E2-a 격차(0.14pp)는 random seed에 따라 역전될 수 있다.

3. **하이퍼파라미터 단일 설정:** E4-b(w_flip=1.0, λ_MI=2.0), E2-a(w_rel=1.0, λ_MI=2.0)는 기본값만 검증. 최적 HP는 탐색하지 않았다.

4. **E5-a EMA momentum 단일 설정:** momentum=0.9만 검증. 낮은 momentum(0.99)이나 static prior(첫 배치만 teacher)는 미탐색.

5. **E6 re-ranking 방향:** top-3 recall 65.2%가 활용 가능한지는 re-ranking 알고리즘 설계에 달려있음. 현재는 진단만 수행.
