# Manual Script: Hypothesis Testing for MRA-TTA & BATCLIP Diagnosis

> **Purpose:** This document is a Claude Code task specification for running hypothesis tests to (i) determine whether MRA-TTA survives as a design, (ii) diagnose root causes in CLIP-based online TTA (BATCLIP and variants), and (iii) guide design modifications based on test outcomes.
>
> **Definitions:**
> - **BATCLIP**: uses `cos(μ̂_k, t_k)` where `μ̂_k = normalize(mean(x_i))` for I2T term
> - **MRA I2T**: uses `(mean(x_i)) · t_k = r̄_k · cos(μ_k, t_k)` (no normalization — raw dot product)
> - **MRA InterMean**: scales BATCLIP's inter-class repulsion term by `r̄_k · r̄_ℓ`
> - **r̄_k**: mean resultant length = `‖Σ x_i‖ / n_k` (x_i are L2-normalized image features)

---

## PART 1: The 3 Decision Tests for MRA's Fate

These three tests determine whether MRA's core assumptions hold. Run them in order. Each has a clear pass/fail criterion and a remediation path if failed.

### Decision Test #1 — Is r̄_k Actually a Reliability Proxy? (n_k Debiasing Test)

**Motivation:** Current H1/H4 results may be confounded by small n_k. On a hypersphere, a random unit vector set of size n gives `E[r̄] ≈ 1/√n`, so small classes appear artificially concentrated. This bias can flip the sign of the correlation with purity. MRA's core mechanism depends on r̄_k being a valid reliability gate — if it's not, MRA may *increase* the loss weight for unreliable classes and *decrease* it for reliable ones.

**Hypothesis:**
> After correcting for n_k bias, the debiased r̃_k is positively correlated with (a) pseudo-label purity, (b) text alignment, and (c) per-sample correctness.

**Implementation Steps:**

```python
# Step 1: Collect per-batch, per-class statistics
for each batch:
    for each class k in {0..C-1}:
        idx_k = [i for i if y_hat[i] == k]
        n_k   = len(idx_k)
        S_k   = sum of x_i for i in idx_k        # x_i: L2-normalized image features
        r_bar_k = norm(S_k) / n_k

        # Option A — rescale so random => ~1, concentrated => >1
        r_tilde_k_A = r_bar_k * sqrt(n_k)

        # Option B — 0-to-1 re-normalized (preferred for correlation analysis)
        floor = 1.0 / sqrt(n_k)
        r_tilde_k_B = clip((r_bar_k - floor) / (1.0 - floor), 0, 1)

        purity_k     = mean(y_gt[i] == k for i in idx_k)     # GT required for diagnosis
        alignment_k  = cos(mean(x_i for i in idx_k), t_k)    # text template embedding

# Step 2: Compute Spearman ρ across (batch, class) pairs
rho_purity    = spearmanr(r_tilde_k_B_list, purity_list)
rho_alignment = spearmanr(r_tilde_k_B_list, alignment_list)

# Step 3: Sample-level AUC
# score[i] = r_tilde_k for the class k = y_hat[i]
# label[i] = int(y_hat[i] == y_gt[i])
auc_correctness = roc_auc_score(labels, scores)
```

**Report per corruption type + severity:**
- Spearman ρ(r̃, purity) with p-value
- Spearman ρ(r̃, alignment) with p-value
- AUC(r̃ → correctness)
- Distribution plots: r̃ vs purity scatter, split by n_k buckets (n_k < 10, 10–30, 30+)

**Pass Criterion:**
- AUC ≥ 0.60 **and** ρ(r̃, purity) consistently positive across ≥ 3 of 5 noise corruptions → **MRA reliability gating is valid**

**Fail Criterion:**
- AUC ≈ 0.50 and/or ρ near 0 or negative → **r̄ is not a reliability signal in this setup**

**If FAILED — Design Modification:**
Replace r̄_k with a margin-based class weight:
```python
# For each class k, compute margin-based gate
margin_i  = logit_top1[i] - logit_top2[i]          # per-sample margin
q_k       = mean(sigmoid(margin_i / tau) for i in idx_k)

# Use q_k in place of r̄_k for all MRA weighting
# tau is a temperature; use tau=0.1 as default and sweep {0.05, 0.1, 0.2}
```
This aligns with **ICLR 2025 ZS-NTTA / Noisy TTA** approaches that emphasize confidence-based filtering over geometric concentration proxies.

### Decision Test #2 — Does MRA Improve Gradient Fidelity? (Oracle Gradient Alignment Test)

**Motivation:** MRA's stated justification is that removing `normalize()` from the mean reduces damage from noisy pseudo-labels. If true, the gradient produced under MRA should be closer to the oracle (GT-label) gradient than BATCLIP's gradient. This is the *mechanistic* test — if it fails, MRA's normalization argument is post-hoc rationalization.

**Hypothesis:**
> Under the same model weights θ, the cosine similarity between pseudo-label gradient and oracle gradient is higher for MRA than for BATCLIP, especially under additive noise corruptions.

**Implementation Steps:**

```python
# For each batch, at the SAME θ (no update applied):
for method in ['BATCLIP', 'MRA']:
    # Compute loss using pseudo-labels (y_hat)
    loss_pseudo = compute_loss(method, batch, labels=y_hat, theta=theta_current)
    g_pseudo    = autograd.grad(loss_pseudo, theta_current)

    # Compute loss using GT labels (y_gt) — for diagnosis only, never update with this
    loss_oracle = compute_loss(method, batch, labels=y_gt, theta=theta_current)
    g_oracle    = autograd.grad(loss_oracle, theta_current)

    # Flatten and compute cosine similarity
    cos_align = F.cosine_similarity(flatten(g_pseudo), flatten(g_oracle), dim=0)

# Record: cos_align per (method, corruption_type, severity)
# Aggregate: mean ± std across batches
```

**Report:**
- Table: `cos_align` mean ± std for [BATCLIP, MRA] × [corruption types] × [severity 1, 3, 5]
- Focus on additive noise types: gaussian_noise, shot_noise, impulse_noise
- Statistical test: paired t-test or Wilcoxon signed-rank per corruption type

**Pass Criterion:**
- MRA achieves meaningfully higher `cos_align` vs BATCLIP (Δ ≥ 0.05) for ≥ 3 noise corruptions at severity 5 → **MRA's normalization removal is mechanistically justified**

**Fail Criterion:**
- No improvement or degradation in `cos_align` → **The bottleneck is pseudo-label quality itself, not normalization**

**If FAILED — Design Modification:**
Focus on pseudo-label noise reduction *before* any geometric refinement:
1. **Confidence selection gate:** only use samples where `margin_i > threshold` for loss computation (threshold sweep: 0.0, 0.1, 0.2, 0.3)
2. **Temperature calibration first:** tune logit temperature τ on the current batch via entropy minimization before computing prototypes
3. Reference: **NeurIPS 2024 ZERO** — shows temperature + selection outperforms complex backprop-based TTA as a baseline

### Decision Test #3 — Does MRA's r̄·r̄ Scaling Kill Inter-Class Separation? (Var_inter Preservation Test)

**Motivation:** Your H7 result (Spearman ρ = 0.957, Var_inter ↔ accuracy) and NeurIPS 2025 Mint both confirm that inter-class variance is *the* dominant signal for accuracy. MRA scales the InterMean (inter-class repulsion) term by `r̄_k · r̄_ℓ`. Under hard corruptions where r̄ is low for all classes, this could suppress the exact repulsion that needs to be strongest.

**Hypothesis:**
> MRA's `r̄_k · r̄_ℓ` scaling does not significantly reduce the effective InterMean weight `w_inter` under corruptions where Var_inter is already low.

**Implementation Steps:**

```python
# Per batch, compute the effective inter-class weight
w_inter_list = []
for batch in dataloader:
    r_bar = {k: compute_r_bar(batch, k) for k in classes}

    # Average of r̄_k * r̄_ℓ for all k ≠ ℓ
    pairs = [(k, l) for k in classes for l in classes if k != l]
    w_inter = mean(r_bar[k] * r_bar[l] for (k, l) in pairs)
    w_inter_list.append((corruption_type, severity, w_inter, var_inter, accuracy))

# Analysis:
# 1. Plot w_inter vs severity for each corruption type
# 2. Compute: w_inter(severity=5) / w_inter(severity=1) — ratio (< 0.5 is alarming)
# 3. Correlation: ρ(w_inter, Var_inter) and ρ(w_inter, accuracy)
# 4. For additive noise corruptions: does w_inter drop precisely when Var_inter is lowest?
```

**Report:**
- Line plots: `w_inter` vs severity for each corruption type (overlay with Var_inter)
- Key diagnostic: ratio `w_inter(sev=5) / w_inter(sev=1)` per corruption
- Scatter: `w_inter` vs `Var_inter` across all (corruption, severity) pairs

**Pass Criterion:**
- `w_inter` does not drop by more than 2× from sev=1 to sev=5 for additive noise **OR** even when `w_inter` drops, Var_inter recovery is not harmed in the actual update

**Fail Criterion:**
- `w_inter` drops >2× in additive noise corruptions where Var_inter is already critically low → **MRA is suppressing the separation it needs to restore**

**If FAILED — Two Clean Design Options (no new hyperparameters):**

**Option A (recommended):** Apply r̄ gating to I2T only; keep InterMean always-on (like BATCLIP):
```
L_MRA = λ₁ · r̃_k · L_I2T(k) + λ₂ · L_InterMean(k,ℓ)   # no r̄ on InterMean
```
Rationale: "align carefully (with reliability), separate always."

**Option B:** Floor the r̄ weight at the random-baseline level:
```python
r_bar_floored_k = max(r_bar_k, 1.0 / sqrt(n_k))
# This ensures the weight never drops below what a random distribution would give
w_inter = mean(r_bar_floored_k * r_bar_floored_l for k ≠ l)
```
Both options are hyperparameter-free and preserve BATCLIP's core architecture.
