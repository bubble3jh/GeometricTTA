# Hypothesis Testing Plan: Diagnosing BATCLIP & Validating MRA

## 1. Objective and Rationale
Prior to executing the full MRA-TTA (Mean Resultant Alignment) implementation, we must diagnose exactly where BATCLIP breaks under severe noise and validate the core premise of MRA. MRA relies on the **mean resultant length ($\bar{r}$)** of the class prototype as a proxy for reliability/confidence. We need to empirically verify:
1. Is $\bar{r}$ a valid proxy for confidence?
2. Does it directly connect to BATCLIP's failure mechanism (especially in noise)?
3. Can it filter out consistent misclassifications (false confidence)?

This plan focuses on a "Logging & Offline Statistics" pipeline. **No optimizer steps are required** (except for a single dummy backward pass to measure gradient scale).

## 2. Experimental Setup & Data Logging Pipeline
* **Setting:** CIFAR-10-C, severity=5, N=1000/corruption, batch=200, `reset_each_shift`.
* **Action:** Do NOT update model parameters. Perform a forward pass and save the following variables for each batch across corruptions:

### 2.1 Base Variables to Log
* Image unit features: $x_i \in S^{d-1}$ (L2-normalized)
* Text prototypes: $t_k \in S^{d-1}$ (L2-normalized)
* Logits / Posteriors: $p_{ik} = \text{softmax}(\tau x_i^\top t_k)$
* Hard pseudo-labels: $\hat{y}_i = \arg\max_k p_{ik}$
* Ground truth labels: $y_i$

### 2.2 Derived Class-Level Vectors
* Hard class mean: $m_k = \frac{1}{n_k} \sum_{i: \hat{y}_i=k} x_i$
* Mean resultant length (Confidence): $\bar{r}_k = \|m_k\|$

### 2.3 Derived Sample-Level Metrics
* Margin: $\Delta_i = x_i^\top t_{\hat{y}_i} - \max_{j \neq \hat{y}_i} x_i^\top t_j$
* Entropy: $H(p_i)$
* Conformity (to visual modality): $c_i = x_i^\top \bar{x}$ (where $\bar{x}$ is the mean of all image features in the shift)

## 3. The 6 Priority Hypotheses to Test
Please implement offline testing for the following 6 hypotheses using the logged data.

### H1: $\bar{r}_k$ represents the true reliability of the class prototype.
* **Hypothesis:** Classes with a higher $\bar{r}_k$ have higher pseudo-label purity and better alignment with text anchors.
* **Metric:** Compute Pseudo-label Purity $\mathbb{P}(y_i=k \mid \hat{y}_i=k)$ and Alignment $a_k = \frac{m_k}{\|m_k\|}^\top t_k$.
* **Test:** Spearman correlation $\text{corr}(\bar{r}_k, \text{purity}_k)$ and $\text{corr}(\bar{r}_k, a_k)$ per corruption.
* **Expected:** High correlation, especially dropping in `noise` corruptions.

### H3: BATCLIP's `normalize(mean)` exaggerates update signals for low-confidence clusters.
* **Hypothesis:** Because BATCLIP normalizes the prototype, the gradient norm with respect to $m_k$ scales proportionally to $1 / \|m_k\|$ (i.e., $1 / \bar{r}_k$).
* **Metric:** Perform one dummy backward pass on the BATCLIP I2T loss. Measure the gradient norm $\|\nabla L\|$ (w.r.t LN/BN parameters or $m_k$).
* **Test:** Spearman correlation $\text{corr}(\|\nabla L\|, 1/\bar{r}_k)$.
* **Expected:** Strong positive correlation. This proves MRA mathematically targets the exact root cause of BATCLIP's failure.

### H6: Consistent misclassification outlier detection.
* **Hypothesis:** In specific noise corruptions, a class might be consistently pulled in the wrong direction, resulting in high $\bar{r}_k$ but low purity.
* **Test:** Generate a scatter plot (or outlier frequency count) of $\bar{r}_k$ vs. $\text{purity}_k$. 
* **Expected:** Identify if "High $\bar{r}$, Low Purity" clusters exist. If they are frequent, MRA will need a sample-level margin sanity check.

### H4: Low $\bar{r}_k$ correlates with augmentation instability.
* **Hypothesis:** Clusters with low $\bar{r}_k$ suffer from severe label flipping across augmented views.
* **Metric:** Generate $N$ augmented views for images. Measure label agreement across views.
* **Test:** Correlation between cluster/batch mean agreement and $\bar{r}_k$.

### H7: Inter-class variance collapse occurs and correlates with accuracy.
* **Hypothesis:** Embedding variance collapse (Mint, 2025) is reproducible, and inter-class variance is strongly tied to accuracy.
* **Metric:** $\text{Var}_{inter} = \frac{1}{K}\sum_k \|\mu_k - \bar{\mu}\|^2$ (using GT or predicted class means $\mu_k$).
* **Test:** Correlation between $\text{Var}_{inter}$ and accuracy across corruptions.

### H9: Conformity is a better confidence proxy than raw norm.
* **Hypothesis:** The conformity score $c_i = x_i^\top \bar{x}$ (cosine similarity to the visual modality mean) predicts correctness better than the raw feature norm.
* **Metric:** AUC for predicting correctness ($\mathbf{1}[\hat{y}_i = y_i]$) using $c_i$, raw norm, margin, and entropy.
* **Test:** Compare AUC scores, especially under additive noise.

---

## 4. Actionable Decision Rules (Post-Test)
The results of these tests will directly dictate our MRA-TTA design choices:
* **If H1 & H3 hold:** MRA's core logic (removing `normalize` and using $\bar{r}$) is fully justified.
* **If H6 holds (many outliers):** We must incorporate sample-level confidence (margin filtering) into MRA.
* **If H7 holds:** We must NOT weaken the InterMean loss too much, as preserving inter-class variance is critical. (May require setting a floor weight).
* **If H9 holds:** We should use conformity ($c_i$) as an alternative or supplementary signal to the raw norm.
