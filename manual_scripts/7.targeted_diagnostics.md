# Experimental Specification: Targeted Diagnostics & Bifurcation (G1, G2, G3)

## 0. Overview & Objectives
Previous diagnostics (D1-D3) established that restricting the feature space to the text-aligned subspace (`Component 1: Projection`) is theoretically correct, but applying it naively causes catastrophic failure (-6.8pp accuracy, increased sink class mass). The hypothesis is that shrinking the feature space concentrates the "poison" of overconfident-wrong (OC-wrong) samples, and re-normalizing these shrunk vectors causes gradient explosion. 

This specification outlines the next three critical diagnostic tests (G1, G2, G3) required to determine the path forward. Depending on the results, the agent will implement either **Route A (Projected Defender)** or **Route B (Gentle Navigator)**.

---

## 1. G1: The "Coupled Mechanism" Test (B vs. B+G vs. P+G)
**Goal:** Determine if the projection mechanism (Component 1) becomes effective only when strictly coupled with evidence gating (Component 2).

**Experimental Setup:** Run 3 parallel 10K / 50-step runs on `gaussian_noise` (severity 5).
1.  **[B] Baseline:** Standard BATCLIP (InterMeanLoss + I2T, no gating). => i think this setting already done experiments.
2.  **[B+G] Baseline + Gating:** Standard BATCLIP, but samples with $s_{max} < \tau$ are masked out from prototype updates and InterMeanLoss.
3.  **[P+G] Projected + Gating:** InterMeanLoss is calculated *only* in the text subspace ($P_Z$), AND samples with $s_{max} < \tau$ are masked out.

**Metrics to Log per Step:**
- Accuracy.
- Sink column mass (Class 3 / "cat" ratio in predictions).
- $\Delta d_{eff\_par}$ (Growth rate of text-aligned effective dimensionality from step 1 to 5).

**Pass/Fail Criteria & Interpretation:**
- **PASS:** If [P+G] > [B] in accuracy, sink mass decreases, and $d_{eff\_par}$ does not monotonically collapse (unlike D3).
  - *Meaning:* Projection is valid, but it requires a clean subspace. Proceed to **Route A**.
- **FAIL:** If [B+G] > [P+G], or if [P+G] still collapses.
  - *Meaning:* Hard projection in the loss landscape is structurally unstable regardless of noise. Proceed to **Route B**.

## 2. G2: The "Normalization Bottleneck" Diagnostic

**Goal:** Diagnose exactly *why* projection without gating (D3) failed. Did the orthogonal projection shrink the vectors so much that re-normalizing them back to length 1 caused the remaining noise to explode?

**Measurements & Logging:**
In the [P+G] and a temporary [P] (Projection only) setup, log the following at each step:
1.  Sample-wise projection norm: $r_i = \|P_Z v_i\|$ (for all samples in the batch).
2.  Prototype-wise projection norm: $r_k = \|P_Z \bar{v}_k\|$ (for the 10 class means).
3.  Analyze the lower 10% quantile of the $r_i$ distribution.
4.  Compute the correlation between $r_i$ and the probability of being predicted as the sink class ("cat").
5.  Compute the correlation between $r_k$ and the gradient norm of the projected InterMeanLoss for that class.

**Interpretation:**
- If samples with small $r_i$ are overwhelmingly classified as the sink class, and small $r_k$ leads to massive gradient norms ($\sim 1/r$ explosion):
  - *Action:* We must change the metric definition. Instead of re-normalizing vectors inside the subspace (`v_proj = v_proj / norm(v_proj)`), we should compute the loss using the dot product bounded by the original full-space norm, or apply a `stop-gradient` / clamp to the denominator to prevent gradient explosion.

## 3. G3: The "Initial Collapse Direction" Diagnostic

**Goal:** The sink class is not a text hub (proven by D2), but why does everything collapse into "cat"? We test if the natural geometry of the visual corruption inherently points towards the "cat" text embedding.

**Measurements & Logging:**
1.  At Step 0 (before any adaptation) and Step 1, compute the 1st Principal Component ($u_1$) of the corrupted image feature batch.
2.  Compute the cosine similarity between this dominant visual collapse direction and all 10 text embeddings: $\cos(u_1, z_c)$.
3.  Rank the classes based on this similarity. Track this across the first 5 steps.

**Interpretation:**
- If "cat" consistently ranks #1:
  - *Meaning:* The sink is a geometric prior caused by the specific corruption (e.g., Gaussian noise crushes features into a shape that happens to align with "cat").
  - *Action:* Justifies adding a simple "Logit Prior Correction" (balancing heuristic) because the error is structurally biased from step 0.
- If "cat" is NOT #1:
  - *Meaning:* The sink is an "attractor" formed dynamically during the moving-average updates. 
  - *Action:* Reaffirms that gating and early-stopping (preventing drift) are the only ways to solve it.

## 4. Decision Matrix: Route A vs. Route B (For the Agent)

Based on the results of G1, G2, and G3, implement one of the following candidate architectures:

### Route A: "The Projected Defender" (If G1 [P+G] PASSES)
- **Gating ($s_{max}$):** Filter out OC-wrong samples using a dynamic threshold (e.g., batch median of absolute cosine similarity).
- **Safe Projection:** Apply $P_Z$ to the features for InterMeanLoss. *Crucially, apply the fix found in G2:* Do not re-normalize $v_\parallel$ blindly. Use `(v @ P_Z @ v.T) / (norm_original)` or clamp the denominator to prevent $1/r$ explosion.
- **Early Stopping (Control):** Recirculate the 1K batch up to 50 steps, but monitor the EMA of $d_{eff\_par}$. Stop training when $\Delta d_{eff\_par}$ hits a plateau to prevent late-stage drift.

### Route B: "The Gentle Navigator" (If G1 [P+G] FAILS, but [B+G] PASSES)
- **Drop Hard Projection in Loss:** Do not modify the InterMeanLoss with $P_Z$. The loss landscape is too fragile.
- **Gating ($s_{max}$):** Strictly filter samples entering the standard InterMeanLoss and prototype updates.
- **Subspace as a Monitor Only:** Calculate $d_{eff\_par}$ purely as an observational metric (detached from the computation graph) to trigger the early stopping during 1K recirculation.
- **Prior Correction:** If G3 showed structural bias towards "cat", apply a class balancing penalty to the logits.
