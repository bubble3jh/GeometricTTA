## 1. 배경

### Exp 7B의 문제

Theorem 3 예측: λ에 따라 equilibrium p†_k = π_k^α / Σ π_j^α (α = λ/(λ-1))이 변하므로 H(p†)도 변해야 함.

실측: H(p̄) ≈ 2.280~2.290으로 flat (range = 0.010).

### 두 가지 원인

**원인 1 (π가 λ와 함께 변함):**
Theorem 3은 π를 상수로 놓고 분석. 실제로는 λ가 바뀌면 adaptation trajectory가 달라지고, evidence도 달라지고, π도 달라짐. λ가 작으면 collapse 방향이라 π가 skewed, λ가 크면 diverse해져서 π가 uniform에 가까움. 이 self-regulation이 p†의 변화를 상쇄.

**원인 2 (I_batch ≠ 0):**
Theorem 3의 가정 A1은 exact cone compression (I_batch = 0, 모든 sample의 prediction이 동일). 실제 I_batch = 0.56 (step 1). Imperfect compression이 이론적 equilibrium과의 괴리를 만듦.

### 이론적 예측 (π 고정 시)

```python
"""
π = PI_FIXED 고정일 때, Theorem 3의 equilibrium:
p†_k = π_k^α / Σ_j π_j^α,   α = λ/(λ-1)

각 λ에 대해 예측값을 미리 계산.
"""

import torch
import math

PI = torch.tensor([0.095, 0.098, 0.102, 0.115, 0.097,
                    0.105, 0.094, 0.100, 0.096, 0.098])

for lam in [1.2, 1.5, 2.0, 3.0]:
    alpha = lam / (lam - 1)
    p_dag = PI ** alpha
    p_dag = p_dag / p_dag.sum()
    H_dag = -(p_dag * p_dag.log()).sum().item()
    cat_dag = p_dag[3].item()  # cat = index 3
    print(f"λ={lam}, α={alpha:.1f}: H(p†)={H_dag:.4f}, cat%={cat_dag:.4f}")

# 예상 출력 (정확한 값은 실행 시 확인):
# λ=1.2, α=6.0:  H(p†) ≈ 2.22~2.25, cat% ≈ 0.20+ (peaked)
# λ=1.5, α=3.0:  H(p†) ≈ 2.27~2.28, cat% ≈ 0.15
# λ=2.0, α=2.0:  H(p†) ≈ 2.296,     cat% ≈ 0.132
# λ=3.0, α=1.5:  H(p†) ≈ 2.298,     cat% ≈ 0.118

# λ ≤ 1: collapse 예측 (vertex solution)
# λ=0.5: p† = δ_{argmax π} = δ_cat → H(p†) = 0
# λ=0.8: 동일
# λ=1.0: 동일
```

## 4. Phase B: I_batch 통제 실험 (원인 2 제거)

### 목적
Phase A에서 π 고정만으로 이론과 실험이 충분히 일치하지 않을 경우, I_batch를 추가로 통제하여 Theorem 3의 exact cone compression 가정에 더 가까운 조건을 만듦.

### 방법: Temperature scaling으로 I_batch 감소

```python
"""
핵심 아이디어: logit을 더 sharp하게 만들면 모든 sample의 prediction이 비슷해짐.
이건 cone compression을 인위적으로 강화하는 것과 같은 효과.

방법: logit에 추가 temperature scaling을 적용.
z_scaled = z * gamma,  gamma > 1

gamma가 크면 softmax가 더 peaked → 모든 sample이 top-1 class에 집중
→ prediction이 동질화 → I_batch → 0
"""

GAMMA_VALUES = [1.0, 2.0, 5.0, 10.0]  # 1.0 = 기본값 (Phase A와 동일)

# Phase A의 best λ (예: λ=2.0) 고정, π도 고정
lam = 2.0

for gamma in GAMMA_VALUES:
    for step in range(n_steps):
        # forward pass
        img_feats = model.encode_image(images)
        txt_feats = model.encode_text(text_tokens)
        
        # 기존 logit
        logits_raw = model.logit_scale.exp() * img_feats @ txt_feats.T  # (B, K)
        
        # Temperature scaling으로 I_batch 통제
        logits_scaled = logits_raw * gamma
        probs = F.softmax(logits_scaled, dim=1)
        
        # π 고정
        pi = PI_FIXED.to(device)
        
        # Loss 계산 (scaled logits 기반)
        p_bar = probs.mean(dim=0)
        L_ent = -(probs * probs.log()).sum(dim=1).mean()
        L_kl = F.kl_div(p_bar.log(), pi, reduction='sum')
        loss = L_ent + lam * L_kl
        
        # Backward는 logits_raw에 대해 (모델 파라미터를 업데이트하려면)
        # gamma는 상수이므로 gradient는 자연스럽게 전파됨
        
        # 로깅: I_batch 포함
        I_batch = H_pbar - mean_H_pi
```

### 대안 방법: Vanilla로 I_batch ≈ 0 상태 만든 후 KL 적용

```python
"""
대안: 더 자연스럽게 I_batch ≈ 0을 만드는 방법.

Step 1: Vanilla entropy minimization을 ~15 step 돌려서 I_batch를 낮춤
        (Exp 6에서 vanilla step=15일 때 I_batch=0.262)
Step 2: 그 상태에서 model을 저장 (checkpoint)
Step 3: Checkpoint에서 시작하여 KL을 λ=2.0으로 켜고 추가 adaptation
Step 4: 이때의 H(p̄)가 이론 예측과 일치하는지 확인

장점: 인위적 temperature 없이 자연스러운 I_batch ≈ 0 상태
단점: Vanilla가 이미 collapse 방향으로 갔으므로 starting point가 biased
"""

# Step 1: Vanilla warm-up (I_batch 감소용)
VANILLA_WARMUP_STEPS = [5, 10, 15]  # 여러 warm-up 길이 시도

for warmup_steps in VANILLA_WARMUP_STEPS:
    # vanilla (kl_lam=0)로 warmup_steps만큼 adaptation
    model_copy = copy.deepcopy(model)
    optimizer_copy = create_optimizer(model_copy)
    
    for step in range(warmup_steps):
        # L_ent만으로 adaptation
        loss = L_ent  # KL 없음
        loss.backward()
        optimizer_copy.step()
    
    # 이 시점의 I_batch 측정
    # Exp 6 참조: step=5 → I_batch≈0.81, step=10 → 0.49, step=15 → 0.26
    
    # Step 2: 여기서부터 KL(π 고정, λ=2.0)로 전환
    for step in range(warmup_steps, warmup_steps + 30):
        pi = PI_FIXED.to(device)
        loss = L_ent + 2.0 * L_kl
        loss.backward()
        optimizer_copy.step()
        
        # 로깅: H(p̄), I_batch, cat%, online_acc
```
