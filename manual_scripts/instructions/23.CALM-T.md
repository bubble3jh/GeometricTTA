# Instruction 23: CALM-T (Text-Aware Anisotropic Shrinkage) 실험

## 1. 목적

1. **Phase 1:** CALM의 π 계산에 text geometry를 반영한 CALM-T의 기본 작동 검증 (gaussian noise sev=5)
2. **Phase 2:** CLIP-specificity 증명 ablation — semantic graph vs random/identity/dense
3. **Phase 3:** Best config로 15-corruption single-config sweep
4. **Fallback:** CALM-T가 성능을 깨면, 현재 CALM 유지 + ResNet 비교 실험으로 CLIP-specificity를 analysis-only로 방어

---

## 2. 배경: 왜 이 실험을 하는가

### 문제
현재 CALM의 모든 component가 generic:
- L_ent: general entropy minimization
- Evidence 수집: prediction ranking + 1/rank (어떤 classifier든 가능)
- π 계산: (s+α)^β (general smoothing)
- KL(p̄‖π): general distribution regularization

→ 리뷰어 공격: "이건 CLIP 특화가 아니다. ResNet에서도 작동할 것 같은데?"

### 해결 방향
π 계산만 CLIP-aware로 교체. L_ent, KL 구조, detach 전부 유지.
- 현재: scalar β로 모든 class 차이를 동일 비율로 압축 (isotropic)
- 수정: text similarity graph로 near-collinear class pair는 더 세게 압축 (anisotropic)

### 이론적 근거
- CLIP에서 classifier = text embedding의 cosine similarity
- Text embedding이 near-collinear (rank≈2, pairwise cos≈0.84)
- cat-dog (cos=0.95)의 evidence 차이는 confusion일 가능성 높음 → 더 세게 압축
- cat-ship (cos=0.75)의 evidence 차이는 real signal일 가능성 높음 → 덜 압축
- 이건 isotropic SNR shrinkage를 covariance-aware shrinkage로 generalize한 것

### 4.2 Graph 구성 방법 (여러 variant, Phase 2에서 비교)

```python
K = 10

# === (a) Raw cosine matrix ===
cos_matrix = T @ T.T  # (K, K), 대각선 = 1.0

# === (b) Centered local affinity (추천) ===
row_mean = cos_matrix.mean(dim=1, keepdim=True)
A_centered = (cos_matrix - row_mean).clamp(min=0)  # negative는 0으로
A_centered.fill_diagonal_(0)  # self-loop 제거

# === (c) k-NN graph ===
def make_knn_graph(cos_matrix, k=2):
    A = torch.zeros_like(cos_matrix)
    for i in range(K):
        vals, idx = cos_matrix[i].topk(k+1)  # +1 for self
        for j in idx:
            if j != i:
                A[i, j] = cos_matrix[i, j]
                A[j, i] = cos_matrix[j, i]  # symmetric
    return A

A_knn = make_knn_graph(cos_matrix, k=2)

# === (d) Random graph (같은 sparsity, random weight) ===
def make_random_graph(A_template):
    """A_template과 같은 sparsity pattern, random positive weights"""
    mask = (A_template > 0).float()
    rand_weights = torch.rand_like(A_template) * mask
    rand_weights = (rand_weights + rand_weights.T) / 2  # symmetric
    return rand_weights

A_random = make_random_graph(A_centered)

# === (e) Dense uniform graph ===
A_dense = torch.ones(K, K) / (K - 1)
A_dense.fill_diagonal_(0)

# === (f) Identity (= 원래 CALM, L_T = 0) ===
# L_T = 0이면 B_T = βI, 원래 CALM과 동치
```

### 4.3 Laplacian 계산

```python
def compute_laplacian(A, normalized=True):
    """Unnormalized or normalized Laplacian"""
    D = A.sum(dim=1)
    if normalized:
        D_inv_sqrt = torch.diag(1.0 / (D.sqrt() + 1e-8))
        L = torch.eye(A.shape[0], device=A.device) - D_inv_sqrt @ A @ D_inv_sqrt
    else:
        L = torch.diag(D) - A
    return L

# Phase 2에서 각 graph variant마다 L_T를 만듦
```

### Run A: 현재 CALM (baseline, 재현 확인용)

```python
def compute_pi_A(logits, L_T, alpha=0.1, beta=0.3):
    """현재 CALM: harmonic simplex + isotropic β"""
    B, K = logits.shape
    ranks = logits.argsort(dim=1, descending=True).argsort(dim=1).float() + 1
    weights = 1.0 / ranks
    weights = weights / weights.sum(dim=1, keepdim=True)
    c = weights.sum(dim=0)
    s = c / B
    pi = (s + alpha).pow(beta)
    pi = pi / pi.sum()
    return pi
```

HP: α=0.1, β=0.3, λ=2.0
기대: online ≈ 0.677 (Inst 22 재현)

### Run B: CALM-T (self-tuned η)

```python
def compute_pi_B(logits, L_T, alpha=0.1, beta=0.3):
    """CALM-T: text-aware anisotropic shrinkage, self-tuned η"""
    B, K = logits.shape
    
    # 1. Harmonic simplex evidence (현재와 동일)
    ranks = logits.argsort(dim=1, descending=True).argsort(dim=1).float() + 1
    weights = 1.0 / ranks
    weights = weights / weights.sum(dim=1, keepdim=True)
    c = weights.sum(dim=0)
    s = c / B  # (K,)
    
    # 2. Centered log-odds
    g = torch.log(s + alpha)
    g = g - g.mean()  # center (K,)
    
    # 3. Self-tuned η (evidence의 graph inconsistency 측정)
    gLg = g @ L_T @ g      # scalar: evidence가 text graph와 얼마나 불일치하는지
    gg = g @ g + 1e-8       # scalar
    eta = gLg / gg
    
    # 4. Anisotropic shrinkage matrix
    #    B_T = β (I + η L_T)^{-1}
    M = torch.eye(K, device=logits.device) + eta * L_T  # (K, K)
    B_T_inv_g = torch.linalg.solve(M, g)  # (I + ηL_T)^{-1} g, (K,)
    
    # 5. π
    pi = F.softmax(beta * B_T_inv_g, dim=0)
    return pi
```

HP: α=0.1, β=0.3, λ=2.0, η=self-tuned
기대: online ≈ 0.677 ± 0.005

### Run C~H: η 고정값 sweep

Run B와 동일한 코드, η만 고정:

```python
def compute_pi_fixed_eta(logits, L_T, alpha=0.1, beta=0.3, eta_fixed=1.0):
    """CALM-T with fixed η"""
    B, K = logits.shape
    ranks = logits.argsort(dim=1, descending=True).argsort(dim=1).float() + 1
    weights = 1.0 / ranks
    weights = weights / weights.sum(dim=1, keepdim=True)
    c = weights.sum(dim=0)
    s = c / B
    
    g = torch.log(s + alpha)
    g = g - g.mean()
    
    M = torch.eye(K, device=logits.device) + eta_fixed * L_T
    B_T_inv_g = torch.linalg.solve(M, g)
    
    pi = F.softmax(beta * B_T_inv_g, dim=0)
    return pi
```

## 10. 중요 구현 주의사항

### 주의 1: g의 centering

```python
g = torch.log(s + alpha)
g = g - g.mean()  # 반드시 center

# 이유: centering 안 하면 g의 common mode가 커서
# L_T에 의한 anisotropic 효과가 묻힘.
# centered g는 "class 간 차이"만 남기므로 L_T의 역할이 명확해짐.
```

### 주의 2: Numerical stability

```python
# torch.linalg.solve는 직접 inverse보다 안정적
# K=10이면 사실 둘 다 문제없지만, 습관적으로 solve 사용

B_T_inv_g = torch.linalg.solve(
    torch.eye(K, device=device) + eta * L_T, 
    g
)
# 이게 (I + ηL_T)^{-1} g
```

### 주의 3: L_T의 common mode 처리

```python
# Text embedding의 pairwise cosine이 평균 0.84로 매우 높음
# Raw cosine matrix를 A로 쓰면 거의 모든 entry가 비슷
# → L_T가 almost zero → anisotropic 효과 없음

# 반드시 centered affinity 사용:
row_mean = cos_matrix.mean(dim=1, keepdim=True)
A = (cos_matrix - row_mean).clamp(min=0)

# 또는 k-NN graph로 sparse하게 만들기
```

### 주의 4: π detach 확인

```python
pi = F.softmax(beta * B_T_inv_g, dim=0)
pi = pi.detach()  # 반드시!

# L_T는 frozen이고 g도 logits에서 detach해서 만드므로
# 원래 gradient가 안 흘러야 하지만, 
# torch autograd가 예상치 못하게 연결할 수 있으므로 명시적 detach.
```

### 주의 5: Phase 1 Run C (η=0) = Run A 확인

```python
# η=0이면 B_T = βI이므로 π = softmax(β·g) = softmax(β·(log(s+α) - mean))
# 이건 현재 CALM의 π_k ∝ (s_k+α)^β와 수학적으로 동치여야 함.
# 
# 확인: 둘의 π가 numerical precision 내에서 같은지 출력해서 비교.
# 만약 다르면 구현 버그.
#
# 증명: π_k ∝ (s_k+α)^β = exp(β·log(s_k+α))
#        softmax(β·g) = softmax(β·(log(s+α) - mean))
#                      = softmax(β·log(s+α) - β·mean)
#                      = softmax(β·log(s+α))  (constant shift는 softmax에서 소거)
#                      ∝ exp(β·log(s+α)) = (s+α)^β  ✓
```
