# CALM v2: Phase 0 진단 실험 — Indicator 구분력 측정

**날짜:** 2026-03-09
**목적:** I2T prototype cleansing을 위한 후보 indicator들의 정오답 구분력을 AUC로 측정
**원칙:** 방법론 구현 전에 전제를 검증. AUC가 낮으면 해당 indicator 기각.
**선행 기각:** adaptive λ (Report 21), cos(f_i, t_k) filtering (softmax와 동일 정보원)

---

## Part 1: 배경 — 왜 이 진단이 필요한가

### 1.1 문제

CALM v1의 I2T prototype이 오분류 샘플에 의해 오염된다.
I2T on/off가 corruption별로 갈리는 이유: prototype 오염 정도가 다르기 때문.

### 1.2 기존 시도의 실패

CLIP logit에서 파생되는 모든 신호 (softmax, cos, margin, entropy)는 동일 정보원.
"cos(f_i, t_cat)이 높다 = softmax가 cat에 높다" — 같은 말.
따라서 logit 기반 신호로는 overconfident-wrong을 구분할 수 없다.

### 1.3 이 진단의 목적

Logit과 **독립적인** 새로운 축의 정보가 존재하는지 확인한다.
후보 indicator들이 실제로 정오답을 구분하는지 AUC로 측정한다.
True label을 사용하지만, 이는 진단 목적이지 방법론에 쓰는 것이 아니다.

## Part 3: 측정할 Indicator 목록

### 3.1 Pairwise Coherence — c_ik (GPT-Pro 제안, 최우선)

**직관:** class k로 예측된 다른 이미지들과 feature가 얼마나 비슷한가.
진짜 같은 클래스 이미지들은 서로 coherent하지만, 오분류로 들어온 이미지들은 서로 incoherent.

**이것이 logit과 다른 이유:** logit은 image-text 관계. 이건 image-image 관계.

**수식:**
```
c_ik = Σ_j q_jk · max(0, f_i^T f_j) / Σ_j q_jk
```
- f_i, f_j: L2-normalized image features
- q_jk: 이미지 j의 class k softmax probability
- max(0, ...): 음의 cosine은 무시 (반대 방향 이미지는 기여하지 않도록)

**구현:**
```python
def compute_pairwise_coherence(img_features, q):
    """
    Args:
        img_features: (B, D) L2-normalized
        q: (B, K) softmax probabilities
    Returns:
        c: (B, K) pairwise coherence score per (sample, class)
    """
    B, K = q.shape
    # image-image similarity matrix
    sim_matrix = img_features @ img_features.T  # (B, B)
    sim_matrix = torch.clamp(sim_matrix, min=0)  # ReLU: 음의 cosine 제거

    c = torch.zeros(B, K, device=q.device)
    for k in range(K):
        # q[:, k]를 weight로 사용한 가중 평균 similarity
        # c_ik = Σ_j q_jk · sim(i,j) / Σ_j q_jk
        weights = q[:, k]  # (B,)
        weighted_sim = sim_matrix * weights.unsqueeze(0)  # (B, B)
        c[:, k] = weighted_sim.sum(dim=1) / (weights.sum() + 1e-8)

    return c  # (B, K)
```

**비용:** B×B 행렬곱 1회 (200×200). CLIP forward 대비 무시 가능.
**추가 하이퍼파라미터:** 없음.

### 3.2 Subspace Projection Score — S_geo (Gemini-Research 제안)

**직관:** image feature가 text embedding들이 만드는 subspace 안에 얼마나 있는가.
Corruption이 feature를 text subspace 밖으로 밀어내면, 오답 샘플의 S_geo가 낮아진다.

**이것이 logit과 다른 이유:** logit은 subspace 내에서의 상대 위치 (어느 text에 가장 가까운가).
S_geo는 subspace 안/밖 (text들이 설명할 수 있는 방향인가 아닌가).

**수식:**
```
T = [t_1, ..., t_K]^T ∈ R^{K×D}       (text features, K=10, D=512)
Q_T, R_T = QR(T^T)                      (T의 column space orthonormal basis)
basis = Q_T[:, :K]                       (D×K orthonormal basis)

proj_i = f_i @ basis @ basis^T           (f_i를 text subspace에 투영)
S_geo(i) = ||proj_i|| / ||f_i||         (투영 에너지 비율)
```

**구현:**
```python
def compute_subspace_projection(img_features, text_features):
    """
    Args:
        img_features: (B, D) L2-normalized
        text_features: (K, D) L2-normalized
    Returns:
        s_geo: (B,) subspace projection score
    """
    # text subspace의 orthonormal basis 계산 (한 번만 하면 됨)
    # text_features: (K, D) -> transpose -> (D, K)
    Q, R = torch.linalg.qr(text_features.T)  # Q: (D, K) orthonormal
    basis = Q  # (D, K)

    # 각 image feature를 text subspace에 투영
    proj = img_features @ basis  # (B, K) — subspace 좌표
    proj_reconstructed = proj @ basis.T  # (B, D) — 복원
    proj_norm = proj_reconstructed.norm(dim=1)  # (B,)
    feat_norm = img_features.norm(dim=1)  # (B,)

    s_geo = proj_norm / (feat_norm + 1e-8)  # (B,)
    return s_geo
```

**비용:** QR decomposition 1회 (고정, 매 배치 안 해도 됨) + 행렬곱 2회. 무시 가능.
**추가 하이퍼파라미터:** 없음.

**주의:** ViT-B-16의 LayerNorm이 ||f_i||를 거의 일정하게 만들기 때문에,
S_geo의 변동이 작을 수 있음. 이 경우 AUC가 낮게 나올 수 있음.
→ 이를 확인하는 것이 이 진단의 핵심 목적 중 하나.

### 3.3 Prompt Variance — p_ik (GPT-Pro 2순위)

**직관:** 여러 text template에 대해 cosine이 일관적이면 진짜, 들쭉날쭉하면 가짜.

**이것이 logit과 다른 이유:** 현재 t_k = 평균(t_{k,1}, ..., t_{k,R}).
평균 cosine이 같아도 분산은 다를 수 있음.

**수식:**
```
templates[k] = [t_{k,1}, ..., t_{k,R}]  (R개 text template)
sims_{i,k} = [f_i · t_{k,1}, ..., f_i · t_{k,R}]
p_ik = exp(-Var(sims_{i,k}))
```

**Template 예시 (CIFAR-10, R=7):**
```python
templates = [
    "a photo of a {}",
    "a blurry photo of a {}",
    "a photograph of a {}",
    "a picture of a {}",
    "an image of a {}",
    "a drawing of a {}",
    "a rendition of a {}",
]
# 각 class name을 {}에 넣어서 R×K개의 text embedding 사전 계산
```

**구현:**
```python
def compute_prompt_variance(img_features, template_features):
    """
    Args:
        img_features: (B, D) L2-normalized
        template_features: (K, R, D) — K classes × R templates × D dims
    Returns:
        p: (B, K) prompt consistency score
    """
    B = img_features.shape[0]
    K, R, D = template_features.shape

    p = torch.zeros(B, K, device=img_features.device)
    for k in range(K):
        # (B, R) cosine similarities with each template
        sims = img_features @ template_features[k].T  # (B, R)
        var = sims.var(dim=1)  # (B,)
        p[:, k] = torch.exp(-var)

    return p  # (B, K) — 높을수록 일관적
```

**비용:** K×R개의 text embedding 사전 계산 (1회) + B×R 내적 K회. 매우 가벼움.
**추가 하이퍼파라미터:** template 선택.

**CIFAR-10 제약:** 클래스명이 단순 ("cat", "dog")해서 template 간 차이가 작을 수 있음.
효과가 ImageNet보다 제한적일 가능성. → AUC로 확인.

## Part 4: 측정 프로토콜

### 4.1 AUC 측정 방법

각 배치에서:
1. CALM v1 forward → logits, img_features, q
2. 각 indicator 계산
3. True label과 비교하여 정답/오답 구분
4. Indicator 값으로 정답/오답을 얼마나 잘 구분하는지 ROC-AUC 계산

**Sample-level indicator** (S_geo, confidence, margin):
```python
# 이미지 i가 정답이면 label=1, 오답이면 label=0
correct = (pred == true_label).float()
auc = sklearn.metrics.roc_auc_score(correct, indicator_values)
```

**(Sample, class)-level indicator** (c_ik, p_ik):
```python
# 이미지 i가 class k에 대해: 진짜 class k이면 label=1, 아니면 label=0
# 예측이 class k인 샘플들만 대상으로 측정
for k in range(K):
    mask = (pred == k)
    if mask.sum() < 10:
        continue
    true_k = (true_label[mask] == k).float()
    auc_k = roc_auc_score(true_k, indicator[mask, k])
# class별 AUC의 가중 평균 (class k 예측 수로 가중)
```

### 4.2 독립성 측정

AUC가 비슷해도 같은 정보일 수 있음. Pearson 상관계수로 독립성 확인:

```python
# sample-level에서
corr_c_vs_conf = pearsonr(c_ik_scores, confidence_scores)
corr_sgeo_vs_conf = pearsonr(s_geo_scores, confidence_scores)
corr_c_vs_sgeo = pearsonr(c_ik_scores, s_geo_scores)
```

상관 < 0.5이면서 AUC > 0.6이면 → **독립적인 새 신호 확인.**

### 5.2 run_diagnostic_phase0.py 흐름

```python
# 1. CALM v1 모델 초기화 (기존 코드에서 가져옴)
model = load_clip_model(...)
setup_calm_v1(model, lambda_mi=2.0, ...)

# 2. Template text embeddings 사전 계산
template_features = precompute_templates(model, class_names, templates)
# shape: (K, R, D) = (10, 7, 512)

# 3. Text subspace basis 사전 계산
text_features = model.get_text_features(...)  # (K, D)
basis = compute_text_subspace_basis(text_features)  # (D, K)

# 4. 배치 루프
all_indicators = []
for step, (images, true_labels) in enumerate(dataloader):
    # CALM v1 forward + update
    logits, img_features = model(images)
    q = softmax(logits)
    pred = logits.argmax(dim=1)

    # Indicator 계산
    c_ik = compute_pairwise_coherence(img_features, q)        # (B, K)
    s_geo = compute_subspace_projection(img_features, basis)   # (B,)
    p_ik = compute_prompt_variance(img_features, template_features)  # (B, K)
    conf = q.max(dim=1).values                                 # (B,)
    margin = q.topk(2, dim=1).values[:, 0] - q.topk(2, dim=1).values[:, 1]

    # True label로 정오답 구분
    correct = (pred == true_labels)

    # 저장
    all_indicators.append({
        "c_ik": c_ik, "s_geo": s_geo, "p_ik": p_ik,
        "conf": conf, "margin": margin,
        "pred": pred, "true_labels": true_labels, "correct": correct,
    })

    # CALM v1 loss 계산 + LayerNorm 업데이트 (기존 로직 그대로)
    loss = calm_v1_loss(logits, img_features, ...)
    loss.backward()
    optimizer.step()

# 5. 전체 AUC 및 상관 계산
compute_and_save_diagnostics(all_indicators, output_path)
```

### 5.4 주의사항

1. **img_features는 L2-normalized여야 함.** CLIP의 image encoder 출력을 L2 normalize.
   open_clip에서 이미 normalized일 수 있음 → 코드에서 확인.

2. **c_ik 계산 시 자기 자신 제외.** sim_matrix의 대각선을 0으로 만들어야 함.
   자기 자신과의 cosine은 항상 1이라 신호를 왜곡.
   ```python
   sim_matrix.fill_diagonal_(0)
   ```

3. **S_geo 계산 시 text_features의 rank 확인.** K=10이면 rank 10 subspace.
   만약 text features 간 cosine이 매우 높으면 effective rank가 10보다 작을 수 있음.

4. **Template text features 계산.** open_clip의 tokenizer와 text encoder를 사용.
   CLIP 원래 80개 template을 쓸 수도 있고, 7개 선택본을 쓸 수도 있음.
   처음에는 7개로 시작하고, AUC가 유의미하면 80개로 확대.

5. **Per-batch AUC vs 전체 AUC.** Per-batch AUC는 배치 크기 200에서 불안정할 수 있음.
   전체 N=10K에서의 AUC를 primary metric으로, per-batch는 참고용.

## Part 6: 의사결정 트리

```
진단 결과 시나리오:

Case A: c_ik AUC > 0.65 AND corr(c_ik, confidence) < 0.5
  → Pairwise coherence는 독립적이고 유의미한 신호!
  → P1: c_ik를 I2T prototype weight에 적용
  → w_ik = q_ik · c_ik
  → 가장 이상적인 시나리오

Case B: c_ik AUC > 0.65 BUT corr(c_ik, confidence) > 0.7
  → 구분력은 있지만 confidence와 중복
  → c_ik 단독은 약함. confidence와 조합 시 추가 이득이 있는지 확인
  → c_ik × confidence가 confidence 단독보다 AUC 높은지 테스트

Case C: S_geo AUC > 0.65 AND corr(S_geo, confidence) < 0.5
  → Subspace projection이 독립적 신호!
  → S_geo를 sample-level filter로 적용 가능
  → 하지만 (sample, class)-level이 아닌 sample-level이라 I2T 특화는 아님
  → c_ik와 조합: w_ik = q_ik · c_ik · S_geo(i)

Case D: c_ik AUC > 0.65 AND S_geo AUC > 0.65 AND corr(c_ik, S_geo) < 0.5
  → 두 독립 축 발견! 최고의 시나리오
  → 두 신호를 조합한 I2T weight 설계

Case E: p_ik AUC > 0.60 (다른 것들보다 낮더라도)
  → Prompt variance는 보조 신호로 활용 가능
  → 비용이 0이므로 다른 indicator와 조합

Case F: 모든 indicator AUC < 0.55
  → 단일 forward pass로는 정보 부족 확정
  → 선택지:
    a) Augmentation consistency로 전환 (추가 forward 2-4회)
    b) Robust estimation만으로 가기 (vMF shrinkage, 추가 정보 없이)
    c) I2T를 포기하고 H(p̄)만으로 논문 작성 (novelty 약함)
  → a가 가장 유망하지만 비용 증가. b는 비용 없지만 효과 불확실.

Case G: c_ik AUC가 gaussian_noise에서는 높고 brightness에서는 낮음
  → corruption severity에 따라 c_ik의 유용성이 다름
  → 이건 오히려 좋은 신호: 오염이 심할 때만 작동 = 필요할 때만 작동
  → 오염이 약한 corruption에서는 어차피 I2T가 깨끗하니까 filter 불필요

Case H: D3 (BATCLIP, no adapt)에서 c_ik AUC가 D1 (CALM v1)보다 현저히 낮음
  → H(p̄)가 collapse를 줄여야 c_ik가 작동한다는 가설 확인
  → CALM v1의 H(p̄)와 c_ik 기반 I2T가 시너지를 이룬다는 스토리
```

### 각 Case에서의 다음 행동 요약

| Case | c_ik | S_geo | 행동 |
|---|---|---|---|
| A | 높음, 독립 | — | → P1: c_ik weighted I2T |
| B | 높음, 중복 | — | → 조합 테스트 |
| C | — | 높음, 독립 | → S_geo + c_ik 조합 |
| D | 둘 다 높음, 독립 | 둘 다 높음, 독립 | → 최적 조합 설계 |
| E | — | — | → p_ik 보조 추가 |
| F | 전부 낮음 | 전부 낮음 | → augmentation 또는 robust estimation |
| G | corruption 의존 | — | → adaptive weight (corruption 심하면 활성) |

## Part 9: 성공 시 다음 실험 (Preview)

Phase 0에서 c_ik의 AUC가 유의미하면 (Case A 또는 D), 다음 실험은:

### P1: c_ik Weighted I2T

```python
# CALM v1 loss 구조 유지, I2T prototype만 변경
w_ik = q_ik * c_ik  # (B, K) — coherence-gated weight
v_bar_k = (w_ik.T @ img_features) / (w_ik.sum(dim=0, keepdim=True).T + 1e-8)
v_hat_k = F.normalize(v_bar_k, dim=1)  # (K, D)
L_i2t = (v_hat_k * text_features).sum(dim=1).mean()

L = L_ent - lambda_mi * H_pbar - w_i2t * L_i2t
```

**비교:** CALM v1 (I2T off) vs CALM v1 (I2T uniform) vs CALM v2 (I2T c_ik weighted)
**기대:** CALM v2가 두 baseline 모두를 초과. 특히 I2T on/off 혼재 해소.

### P2: Text Shrinkage 추가

```python
# prototype이 불안정한 class는 text로 fallback
n_k = w_ik.sum(dim=0)  # (K,) 유효 샘플 수
mu_k = F.normalize(kappa_0 * text_features + w_ik.T @ img_features, dim=1)
# n_k가 작으면 자연스럽게 text 지배, n_k가 크면 data 지배
```

## 부록 A: Pairwise Coherence의 가설과 한계

### A.1 핵심 가설

"같은 클래스의 진짜 이미지들은 feature space에서 coherent cluster를 형성하고,
 다른 클래스에서 오분류되어 들어온 이미지들은 서로 incoherent하다."

이 가설이 맞을 조건:
- 진짜 class k 이미지들의 feature가 서로 비슷함 (intra-class coherence)
- 오분류 이미지들이 여러 다른 클래스에서 왔으므로 서로 다름 (inter-source diversity)
- H(p̄)가 이미 collapse를 줄여서, class k 안에 소수라도 진짜 샘플이 남아있음

### A.2 가설이 틀릴 수 있는 경우

- 오분류 이미지들이 특정 corruption pattern을 공유하여 서로 coherent할 때
  (예: gaussian noise가 모든 이미지를 비슷한 방향으로 밀어서, 오분류 dog들끼리도 비슷)
- 이 경우 c_ik가 오분류에도 높게 나와서 구분 불가
- → AUC 측정으로 확인

### A.3 D3 (BATCLIP, 적응 전)과의 비교 의의

BATCLIP 상태에서 c_ik AUC가 낮고, CALM v1 상태에서 높으면:
"H(p̄)가 collapse를 줄여줘야 비로소 coherent core가 드러나고, c_ik가 작동한다"
→ H(p̄)와 c_ik의 시너지를 보여주는 증거
→ 논문 스토리 강화
