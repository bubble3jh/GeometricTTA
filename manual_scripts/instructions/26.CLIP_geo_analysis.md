## 1. 동기

### 문제
CALM의 KL regularization은 성능이 좋지만 CLIP-specific하지 않다. NeurIPS 리뷰어의 예상 공격:
- "이건 generic entropy method, CLIP과 무관하다"
- "text embedding을 왜 안 쓰나, CLIP TTA답지 않다"

### 이전 시도의 실패 원인
CALM-T, CALM-AV class gate, CALM-AV sample gate 모두 **text-text cosine similarity**를 활용하려 했으나, K=10에서 text embedding near-collinearity (pairwise cosine ~0.84, effective rank=2.03)로 인해 신호가 죽음.

### 이번 진단의 핵심 전환
Text-text 관계가 아닌 **image-text 사이의 구조 (modality gap)**를 분석. Modality gap은 K에 무관한 CLIP-specific 구조이므로, K=10 collinearity 문제를 우회할 수 있음.

### 이미 알려진 사실
- d_eff: 512D → 1.21D under gaussian_noise sev=5 (Report 11)
- 오분류 52.8%가 class 3 (cat)으로 집중 (Report 11)
- Overconfident-wrong rate: 25-27% (Report 6)
- Corruption creates directional bias, not isotropic noise (multiple reports)
- Modality gap severity 변화: ±1.1% (Report 6) — batch 평균 기준

### 빠져있는 분석
- Gap vector와 collapse 방향의 관계
- Per-class gap 변화
- Overconfident-wrong 샘플의 gap 내 위치
- Adaptation 중 gap dynamics
- Cone 구조 변형

전체 구조: **Block A** (static, 모델 실행 없음) → **Block B** (frozen model + corrupted data) → **Block C** (adaptation 중 dynamics)

## 4. Block A: Static Geometry (모델 실행 없음, 코드만)

### 목적

### A1: Gap Vector 기본 측정

```python
"""
frozen CLIP에서 CIFAR-10 class text embeddings와
clean CIFAR-10 test images의 기본 geometry를 측정.

필요한 것:
- text embeddings: T = (K, D) = (10, 512)
  prompt: "a photo of a {class_name}" → CLIP text encoder → L2 normalize
- clean image embeddings: F_clean = (N, D)
  CIFAR-10 test set (clean, no corruption) → CLIP image encoder → L2 normalize
  N = 10000
"""

import torch
import torch.nn.functional as F

# === 계산할 양 ===

# 1. Modality gap vector
mean_img = F_clean.mean(dim=0)       # (D,)
mean_txt = T.mean(dim=0)             # (D,)
gap_vector = mean_txt - mean_img     # (D,) text 방향이 positive
gap_magnitude = gap_vector.norm().item()
gap_direction = F.normalize(gap_vector, dim=0)  # unit vector

# 2. Gap magnitude (cosine version)
gap_cosine = F.cosine_similarity(mean_img.unsqueeze(0), mean_txt.unsqueeze(0)).item()

# 3. Per-class gap
# class별 clean image mean vs 해당 text anchor
for k in range(K):
    mask_k = (labels == k)
    mean_img_k = F_clean[mask_k].mean(dim=0)
    gap_k_cos = F.cosine_similarity(mean_img_k.unsqueeze(0), T[k].unsqueeze(0)).item()
    gap_k_euclid = (mean_img_k - T[k]).norm().item()
    # log: class_name, gap_k_cos, gap_k_euclid

# 4. Image cone statistics
pairwise_img_cos = F_clean @ F_clean.T  # (N, N)
# 상삼각만 사용
triu_mask = torch.triu(torch.ones(N, N, dtype=torch.bool), diagonal=1)
img_cone_mean_cos = pairwise_img_cos[triu_mask].mean().item()
img_cone_std_cos = pairwise_img_cos[triu_mask].std().item()

# 5. Text cone statistics
pairwise_txt_cos = T @ T.T  # (K, K)
triu_mask_txt = torch.triu(torch.ones(K, K, dtype=torch.bool), diagonal=1)
txt_cone_mean_cos = pairwise_txt_cos[triu_mask_txt].mean().item()
txt_cone_std_cos = pairwise_txt_cos[triu_mask_txt].std().item()

# 6. Effective rank (image features)
F_centered = F_clean - mean_img  # (N, D)
# SVD on centered features
U, S, V = torch.linalg.svd(F_centered, full_matrices=False)
p = S / S.sum()
eff_rank_clean = torch.exp(-(p * p.log()).sum()).item()

# 7. Cross-modal alignment per class
# 각 class의 image feature들이 해당 text anchor와 얼마나 aligned인지
for k in range(K):
    mask_k = (labels == k)
    cos_to_own_text = F.cosine_similarity(F_clean[mask_k], T[k].unsqueeze(0)).mean().item()
    cos_to_mean_text = F.cosine_similarity(F_clean[mask_k], mean_txt.unsqueeze(0)).mean().item()
    # log: class_name, cos_to_own_text, cos_to_mean_text
```

## 5. Block B: Corruption Effect on Geometry (Frozen Model)

### 목적
Corruption이 modality gap, cone, feature 분포를 어떻게 변형하는지 측정.
**모델은 frozen (adaptation 없음).** Pure corruption effect만 분리.

### Corruptions to test

### B1: Gap Vector vs Collapse Direction Alignment (H1 — 최우선)

```python
"""
핵심 가설: corruption에 의한 feature collapse 방향이 gap vector와 aligned인가?

이것이 전체 진단의 go/no-go를 결정함.
"""

# F_corr = corrupted image features (N, D), frozen model
# F_clean = clean image features (N, D), frozen model
# gap_direction = A1에서 계산한 unit gap vector

# 1. Corrupted feature의 principal component (collapse 방향)
F_corr_centered = F_corr - F_corr.mean(dim=0)
U_corr, S_corr, V_corr = torch.linalg.svd(F_corr_centered, full_matrices=False)
PC1_corr = V_corr[0]  # (D,) first principal component of corrupted features

# 2. Per-sample shift vector
delta = F_corr - F_clean  # (N, D) corruption에 의한 shift
mean_delta = delta.mean(dim=0)
mean_delta_dir = F.normalize(mean_delta, dim=0)

# PC1 of shift vectors
delta_centered = delta - mean_delta
U_d, S_d, V_d = torch.linalg.svd(delta_centered, full_matrices=False)
PC1_delta = V_d[0]

# 3. Alignment 측정
cos_PC1corr_gap = F.cosine_similarity(PC1_corr.unsqueeze(0), gap_direction.unsqueeze(0)).item()
cos_meandelta_gap = F.cosine_similarity(mean_delta_dir.unsqueeze(0), gap_direction.unsqueeze(0)).item()
cos_PC1delta_gap = F.cosine_similarity(PC1_delta.unsqueeze(0), gap_direction.unsqueeze(0)).item()

# 4. Cat text anchor와의 alignment
t_cat = T[3]  # class 3 = cat
t_cat_dir = F.normalize(t_cat, dim=0)
cos_PC1corr_cat = F.cosine_similarity(PC1_corr.unsqueeze(0), t_cat_dir.unsqueeze(0)).item()
cos_meandelta_cat = F.cosine_similarity(mean_delta_dir.unsqueeze(0), t_cat_dir.unsqueeze(0)).item()

# 5. 삼각 관계: gap, PC1_corr, t_cat 사이의 pairwise cosine
cos_gap_cat = F.cosine_similarity(gap_direction.unsqueeze(0), t_cat_dir.unsqueeze(0)).item()

print(f"cos(PC1_corr, gap): {cos_PC1corr_gap:.4f}")
print(f"cos(mean_delta, gap): {cos_meandelta_gap:.4f}")
print(f"cos(PC1_delta, gap): {cos_PC1delta_gap:.4f}")
print(f"cos(PC1_corr, t_cat): {cos_PC1corr_cat:.4f}")
print(f"cos(mean_delta, t_cat): {cos_meandelta_cat:.4f}")
print(f"cos(gap, t_cat): {cos_gap_cat:.4f}")
```

**Go/No-Go 기준:**
```
|cos(PC1_corr, gap)| > 0.3  OR  |cos(mean_delta, gap)| > 0.3
  → Gap 방향과 collapse 방향이 관련 있음 → Block B 나머지 + Block C 진행

둘 다 < 0.15
  → Gap은 collapse와 무관 → Gap 기반 방법론 포기, cone 분석(B5)으로 전환
```

### B2: Per-Class Gap Change under Corruption (H2)

```python
"""
Corruption이 class별 modality gap을 어떻게 변형하는지.
Cat class의 gap이 비정상적으로 변하는가?
"""

CIFAR10_CLASSES = ['airplane', 'automobile', 'bird', 'cat', 'deer',
                   'dog', 'frog', 'horse', 'ship', 'truck']

for k in range(K):
    mask_k = (labels == k)
    
    # Clean
    mean_clean_k = F_clean[mask_k].mean(dim=0)
    cos_clean_k = F.cosine_similarity(mean_clean_k.unsqueeze(0), T[k].unsqueeze(0)).item()
    L2_clean_k = (mean_clean_k - T[k]).norm().item()
    
    # Corrupted
    mean_corr_k = F_corr[mask_k].mean(dim=0)
    cos_corr_k = F.cosine_similarity(mean_corr_k.unsqueeze(0), T[k].unsqueeze(0)).item()
    L2_corr_k = (mean_corr_k - T[k]).norm().item()
    
    # Change
    delta_cos_k = cos_corr_k - cos_clean_k
    delta_L2_k = L2_corr_k - L2_clean_k
    
    # Corrupted features → wrong text anchor 거리
    # 각 class k의 corrupted features가 다른 class text anchor와 얼마나 가까운지
    cos_to_all_text = (F_corr[mask_k] @ T.T).mean(dim=0)  # (K,)
    most_confused_class = cos_to_all_text.argmax().item()
    
    print(f"{CIFAR10_CLASSES[k]}: clean_cos={cos_clean_k:.4f} → corr_cos={cos_corr_k:.4f} "
          f"(Δ={delta_cos_k:+.4f}), most_confused={CIFAR10_CLASSES[most_confused_class]}")
```

### B3: Overconfident-Wrong 샘플의 Gap 위치 (H4, H5)

```python
"""
Overconfident-wrong 샘플이 gap 방향으로 특정 패턴을 보이는가?
"""

# Frozen model로 prediction
logits = (F_corr @ T.T) * tau  # tau = CLIP temperature (100.0 for ViT-B/16)
probs = F.softmax(logits, dim=1)
preds = logits.argmax(dim=1)
max_prob = probs.max(dim=1).values

# 분류
correct_mask = (preds == labels)
wrong_mask = ~correct_mask
overconf_wrong_mask = wrong_mask & (max_prob > 0.5)  # high confidence + wrong
underconf_correct_mask = correct_mask & (max_prob < 0.5)  # low confidence + correct

# Gap projection
gap_proj = F_corr @ gap_direction  # (N,) — gap 방향으로의 projection

# 그룹별 gap projection 분포
groups = {
    'correct': gap_proj[correct_mask],
    'wrong': gap_proj[wrong_mask],
    'overconf_wrong': gap_proj[overconf_wrong_mask],
    'underconf_correct': gap_proj[underconf_correct_mask],
}

for name, vals in groups.items():
    print(f"{name}: n={len(vals)}, mean_gap_proj={vals.mean():.4f}, "
          f"std={vals.std():.4f}")

# 통계적 차이
from scipy import stats
t_stat, p_val = stats.ttest_ind(
    gap_proj[correct_mask].cpu().numpy(),
    gap_proj[overconf_wrong_mask].cpu().numpy()
)
print(f"t-test correct vs overconf_wrong: t={t_stat:.4f}, p={p_val:.6f}")

# Gap parallel vs perpendicular 분해
gap_parallel = (F_corr @ gap_direction).unsqueeze(1) * gap_direction.unsqueeze(0)  # (N, D)
gap_perp = F_corr - gap_parallel  # (N, D)

# 각 성분에서의 wrong-class margin
for name, mask in [('correct', correct_mask), ('overconf_wrong', overconf_wrong_mask)]:
    f_par = gap_parallel[mask]   # (n, D)
    f_perp = gap_perp[mask]      # (n, D)
    
    pred_k = preds[mask]
    true_k = labels[mask]
    
    # Parallel에서의 cosine to pred vs true text
    cos_par_pred = F.cosine_similarity(f_par, T[pred_k]).mean().item()
    cos_par_true = F.cosine_similarity(f_par, T[true_k]).mean().item()
    
    # Perpendicular에서의 cosine
    cos_perp_pred = F.cosine_similarity(f_perp, T[pred_k]).mean().item()
    cos_perp_true = F.cosine_similarity(f_perp, T[true_k]).mean().item()
    
    print(f"{name}: par(pred={cos_par_pred:.4f}, true={cos_par_true:.4f}), "
          f"perp(pred={cos_perp_pred:.4f}, true={cos_perp_true:.4f})")
```

### B4: Cat Sink 방향 삼각 관계 (H6)

```python
"""
세 벡터 간의 관계: gap_direction, PC1_corr, t_cat
이미 B1에서 pairwise cosine을 계산했으므로 여기서는 추가 분석.

질문: collapse 방향이 "gap을 따라 cat 쪽으로" 가는 것인가,
      아니면 gap과 무관하게 cat 쪽으로 가는 것인가?
"""

# gap_direction에서 t_cat 성분을 빼면?
t_cat_dir = F.normalize(T[3], dim=0)
gap_sans_cat = gap_direction - (gap_direction @ t_cat_dir) * t_cat_dir
gap_sans_cat = F.normalize(gap_sans_cat, dim=0)

# PC1_corr가 gap_sans_cat과도 aligned이면
# → collapse는 gap의 cat 성분뿐 아니라 gap 자체를 따름
cos_PC1_gap_sans_cat = F.cosine_similarity(
    PC1_corr.unsqueeze(0), gap_sans_cat.unsqueeze(0)
).item()

print(f"cos(PC1, gap):          {cos_PC1corr_gap:.4f}")
print(f"cos(PC1, t_cat):        {cos_PC1corr_cat:.4f}")
print(f"cos(PC1, gap_sans_cat): {cos_PC1_gap_sans_cat:.4f}")
print(f"cos(gap, t_cat):        {cos_gap_cat:.4f}")

# 해석:
# cos(PC1, gap) 높고 cos(PC1, gap_sans_cat) 낮으면
#   → collapse는 gap의 cat 성분만 따름
# cos(PC1, gap) 높고 cos(PC1, gap_sans_cat)도 높으면
#   → collapse는 gap 방향 전체를 따름 (cat은 부수적)
```

### B5: Cone 구조 변형 (H7, H8)

```python
"""
Corruption이 image cone을 어떻게 변형하는가.
"""

# 1. Pairwise cosine (cone 좁기 측정)
# Clean
# 전체 N=10000 pairwise는 메모리 부담 → 1000 sample만
sample_idx = torch.randperm(N)[:1000]

F_clean_s = F_clean[sample_idx]
F_corr_s = F_corr[sample_idx]

pw_clean = (F_clean_s @ F_clean_s.T)
triu = torch.triu(torch.ones(1000, 1000, dtype=torch.bool), diagonal=1)
cone_clean_mean = pw_clean[triu].mean().item()
cone_clean_std = pw_clean[triu].std().item()

pw_corr = (F_corr_s @ F_corr_s.T)
cone_corr_mean = pw_corr[triu].mean().item()
cone_corr_std = pw_corr[triu].std().item()

# 2. Effective rank
F_corr_centered = F_corr - F_corr.mean(dim=0)
U_c, S_c, V_c = torch.linalg.svd(F_corr_centered, full_matrices=False)
p_c = S_c / S_c.sum()
eff_rank_corr = torch.exp(-(p_c * p_c.log()).sum()).item()

# 3. Cone direction shift
cone_shift = F.cosine_similarity(
    F_clean.mean(dim=0).unsqueeze(0),
    F_corr.mean(dim=0).unsqueeze(0)
).item()

# 4. Top-5 singular value ratio (concentration)
sv_ratio_clean = (S_clean[:5].sum() / S_clean.sum()).item()  # S_clean from A1
sv_ratio_corr = (S_c[:5].sum() / S_c.sum()).item()

print(f"Cone mean cos: clean={cone_clean_mean:.4f}, corr={cone_corr_mean:.4f}")
print(f"Cone std cos:  clean={cone_clean_std:.4f}, corr={cone_corr_std:.4f}")
print(f"Eff rank:      clean={eff_rank_clean:.2f}, corr={eff_rank_corr:.2f}")
print(f"Cone shift:    cos(mean_clean, mean_corr)={cone_shift:.4f}")
print(f"Top-5 SV ratio: clean={sv_ratio_clean:.4f}, corr={sv_ratio_corr:.4f}")
```

```python
with torch.no_grad():
    # 현재 step의 image features (batch)
    img_feats = model.encode_image(images)  # (B, D)
    img_feats = F.normalize(img_feats, dim=1)
    
    # 현재 step의 text features (adapted text LN 포함)
    text_feats = model.encode_text(text_tokens)  # (K, D)
    text_feats = F.normalize(text_feats, dim=1)
    
    # 1. Batch-level gap
    batch_mean_img = img_feats.mean(dim=0)  # (D,)
    mean_txt = text_feats.mean(dim=0)       # (D,)
    
    gap_magnitude = (mean_txt - batch_mean_img).norm().item()
    gap_cosine = F.cosine_similarity(
        batch_mean_img.unsqueeze(0), mean_txt.unsqueeze(0)
    ).item()
    
    # 2. Gap 방향 변화 (초기 gap 대비)
    current_gap_dir = F.normalize(mean_txt - batch_mean_img, dim=0)
    # gap_direction_init은 step 0에서 저장해둔 것
    gap_dir_stability = F.cosine_similarity(
        current_gap_dir.unsqueeze(0), gap_direction_init.unsqueeze(0)
    ).item()
    
    # 3. Image cone 좁기 (batch 내)
    pw = img_feats @ img_feats.T  # (B, B)
    triu_b = torch.triu(torch.ones(B, B, dtype=torch.bool, device=pw.device), diagonal=1)
    batch_cone_mean = pw[triu_b].mean().item()
    
    # 4. Per-class gap (상위 3 class만 — cat, dog, 가장 큰 class)
    preds_batch = (img_feats @ text_feats.T).argmax(dim=1)
    for k in [3]:  # cat = class 3
        mask_k = (preds_batch == k)
        if mask_k.sum() > 5:
            mean_k = img_feats[mask_k].mean(dim=0)
            cat_text_cos = F.cosine_similarity(
                mean_k.unsqueeze(0), text_feats[3].unsqueeze(0)
            ).item()
        else:
            cat_text_cos = float('nan')
    
    # 5. Effective rank of batch (cheap version: top-5 SV ratio)
    F_batch_c = img_feats - batch_mean_img
    try:
        S_batch = torch.linalg.svdvals(F_batch_c)
        p_batch = S_batch / S_batch.sum()
        eff_rank_batch = torch.exp(-(p_batch * p_batch.log()).sum()).item()
    except:
        eff_rank_batch = float('nan')

# Step log에 추가:
step_log.append({
    'step': step,
    'gap_magnitude': gap_magnitude,
    'gap_cosine': gap_cosine,
    'gap_dir_stability': gap_dir_stability,
    'batch_cone_mean_cos': batch_cone_mean,
    'cat_text_cos': cat_text_cos,
    'eff_rank_batch': eff_rank_batch,
    # 기존: online_acc, cat_pct, mean_entropy
})
```

### C3: Mean-Centering 효과 테스트 (H12 — 탐색적)

```python
"""
SpLiCE에서 영감.
매 batch에서 image features를 batch mean으로 centering + re-normalize 한 뒤
logit을 계산하면 성능이 달라지는가?

이건 adaptation이 아니라 inference-time preprocessing.
H2 위에 추가할 수 있는 CLIP-specific component 후보.

중요: 이건 Block C의 optional 실험. B1이 strong positive일 때만 실행.
"""

# Option 1: Batch mean centering (매 batch)
img_feats_centered = img_feats - img_feats.mean(dim=0, keepdim=True)
img_feats_centered = F.normalize(img_feats_centered, dim=1)
logits_centered = img_feats_centered @ text_feats.T * tau

# Option 2: Running mean centering (exponential moving average)
if step == 0:
    running_mean = img_feats.mean(dim=0)
else:
    running_mean = 0.9 * running_mean + 0.1 * img_feats.mean(dim=0)
img_feats_centered = img_feats - running_mean
img_feats_centered = F.normalize(img_feats_centered, dim=1)
logits_centered = img_feats_centered @ text_feats.T * tau

# 비교: centering 전후 accuracy
# 이건 loss에 영향 없이 prediction만 바꾸는 것
# H2 adaptation은 원래 logits로 하되, 최종 prediction만 centered logits로 할 수도 있음
```

## 9. 결과 해석 가이드

### 시나리오 1: Gap-Collapse Alignment이 강함 (|cos| > 0.5)

**의미:** Corruption이 image feature를 modality gap 방향으로 collapse시킴.
**논문 활용:**
- Analysis section: "CLIP의 modality gap이 corruption-induced collapse의 기하학적 원인"
- Motivation: "Collapse 방향이 gap 방향이므로, gap-aware regularization이 필요"
- 방법론 확장 가능: H13 (gap-aware entropy weighting) 또는 H14 (gap preservation)

### 시나리오 2: Gap-Collapse Alignment이 약하지만 Cat-Collapse는 강함

**의미:** Collapse는 cat text anchor 방향이지만 gap 방향은 아님.
**논문 활용:**
- "Collapse가 특정 class의 text anchor 방향으로 발생" — 이것도 CLIP-specific
- CLIP에서 text anchor가 classifier head이므로, collapse가 text geometry를 따름
- Gap 기반 방법론은 약하지만, text-anchor 기반 분석은 가능

### 시나리오 3: Cone 변형이 가장 두드러진 신호

**의미:** Corruption이 image cone을 극적으로 좁히고 (eff_rank ↓↓), 이게 collapse의 주 원인.
**논문 활용:**
- "Corruption이 CLIP image embedding의 cone을 1D로 collapse"
- Mean-centering이나 whitening 같은 geometric correction이 방법론이 될 수 있음

### 시나리오 4: Gap dynamics가 adaptation 중 크게 변함

**의미:** Adaptation이 gap 구조를 바꿈.
**논문 활용:**
- "KL regularization이 modality gap을 보존하면서 adaptation한다"
- H14 (gap preservation) 방향이 자연스러움
