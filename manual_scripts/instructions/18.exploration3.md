# Instruction 18: Centered Contrastive Relational Adaptation — Exploration Sweep

## 1. 배경 및 목적

### 연구 배경 요약

CLIP ViT-B-16의 online TTA에서 gaussian noise sev=5 시 53%의 예측이 cat (class 3)으로 collapse하는 "cat sink" 문제가 있음.

**기존 방법들의 결과 (72-run comprehensive sweep + Run 1-3 진단):**

| Method | Loss | gaussian_noise Acc | Cat% | 비고 |
|---|---|---|---|---|
| Frozen zero-shot | — | 0.3796 | 53.0% | baseline |
| BATCLIP | L_ent + L_i2t | 0.6060 | ~27% | ICCV 2025 baseline |
| CALM v1 | L_ent - 2·H(p̄) | 0.6458 | ~13% | H(p̄) = uniform 강제, skew에서 -7.67pp |
| H2 (KL evidence) | L_ent + KL(p̄∥π_evid) | 0.6734 | 12.9% | Best acc. 하지만 β=0.3 ≈ uniform, skew에서 BATCLIP 이하 |
| J3 (Rel only) | L_rel | 0.5370 (online), 0.6002 (offline) | 14.6% | Collapse 방지 O, sharpening X |

**Run 1-3 진단 결과:**
- **Run 1:** J3의 offline acc=0.60 (representation은 BATCLIP급), mean entropy=0.98 (prediction 매우 soft), LN delta=4.09 (gradient 충분). H2의 offline acc=0.715, mean entropy=0.149. → J3의 병목은 sharpness.
- **Run 2:** L_rel + 0.2·L_ent → step 10에서 cat=69.1%, step 50에서 89.9%. 즉시 collapse. → Rel은 L_ent의 collapse를 못 막음. Rel의 gradient가 prototype을 경유하는 간접 경로(2-hop)라 L_ent의 직접 경로(1-hop)보다 약함.
- **Run 3:** J3 frozen logit에서 neighbor vote rerank → 0.5793 (J3 offline 0.60보다 -2.1pp 악화). → Soft prediction 위에서는 post-hoc 교정 불가.

**핵심 진단:** J3은 "방향은 맞지만 확신이 없는" 상태. L_ent 수준의 sharpening이 필요하지만 L_ent를 쓰면 collapse. 다른 방식의 sharpening 필요.

### 문헌에서 얻은 insight

1. **CLIPTTA (NeurIPS 2025):** Entropy minimization은 CLIP의 contrastive pretraining과 misaligned. Soft contrastive loss가 batch-aware로 collapse를 막으면서 sharpening. CIFAR-10-C에서 +20.5pp.
2. **Modality gap / cone effect (Liang 2022):** Image와 text가 narrow cone에 분포. Text effective rank=2.03 (우리 측정). Centering이 common-mode를 제거.
3. **TCA (CVPR 2025):** Inter-class topology 보존만으로는 부족, self-training (L_SCE)을 같이 씀. 우리 J3과 동일한 문제.
4. **GPT-Pro 분석:** H2의 진짜 메커니즘은 "uniform 강제"가 아니라 "common-mode batch logit drift 억제". 이걸 분포 가정 없이 L_cm = ||U^T f̄||²로 대체 가능.

### 이번 실험의 목적

"분포 가정 없이, CLIP의 image-text structure를 활용하여 sharpening + anti-collapse를 동시에 달성하는 method의 핵심 component를 탐색." 최고 성능이 아니라 **방향성 확인**이 목표.

## 3. 공통 구현 요소

### 3.1 Common-mode direction U 계산 (한 번만, adaptation 전에)

```python
import torch
import torch.nn.functional as F

# text_features: (K, D) — frozen CLIP text embeddings, K=10
# 방법 1: text mean direction (rank 1)
t_mean = text_features.mean(dim=0)
u0 = F.normalize(t_mean, dim=0)  # (D,)
U_rank1 = u0.unsqueeze(1)  # (D, 1)

# 방법 2: text top-2 PC (rank 2) — 우리 setting에서 effective rank=2.03이므로
# text_features를 centering 후 SVD
t_centered = text_features - t_mean.unsqueeze(0)  # (K, D)
_, S, Vh = torch.linalg.svd(t_centered, full_matrices=False)
U_rank2 = Vh[:2].T  # (D, 2) — top-2 right singular vectors

# 이번 실험에서는 U_rank1을 기본으로 사용.
# 실험 B에서 rank 1 vs rank 2 비교.
U = U_rank1  # (D, 1)
```

### 3.2 Centering / Common-mode 제거

```python
def remove_common_mode(features, U):
    """features: (B, D), U: (D, r) — common-mode directions"""
    projection = features @ U @ U.T  # (B, D)
    return features - projection

def compute_centered_logits(image_features, text_features, U, tau=1.0):
    """Common-mode를 제거한 centered logit 계산"""
    f_hat = remove_common_mode(image_features, U)  # (B, D)
    t_hat = remove_common_mode(text_features, U)    # (K, D)
    # Normalize after projection
    f_hat = F.normalize(f_hat, dim=1)
    t_hat = F.normalize(t_hat, dim=1)
    logits = f_hat @ t_hat.T / tau  # (B, K)
    return logits
```

### 3.3 Soft Contrastive Loss (CLIPTTA-style)

```python
def soft_contrastive_loss(logits, tau_q=1.0, tau_c=0.5):
    """
    Batch-aware soft contrastive loss.
    logits: (B, K) — centered logits
    tau_q: temperature for soft pseudo-label generation
    tau_c: temperature for contrastive loss computation
    
    Returns: scalar loss
    
    핵심 메커니즘:
    - L_i2t: 각 이미지를 가까운 text에 정렬 (sharpening 효과)
    - L_t2i: 각 text를 관련 이미지에 정렬 (batch-aware anti-collapse)
      → 한 class로 몰리면 softmax(logits[:,k])가 uniform에 가까워져서
        q와의 cross-entropy가 커짐 → collapse에 벌점
    """
    B, K = logits.shape
    
    # Soft pseudo-label (detach — gradient 안 흐름)
    q = F.softmax(logits / tau_q, dim=1).detach()  # (B, K)
    
    # Image → Text direction
    log_p_i2t = F.log_softmax(logits / tau_c, dim=1)  # (B, K)
    L_i2t = -(q * log_p_i2t).sum(dim=1).mean()
    
    # Text → Image direction
    log_p_t2i = F.log_softmax(logits.T / tau_c, dim=1)  # (K, B)
    q_t2i = q.T  # (K, B) — 각 class에 대한 image별 soft assignment
    # Normalize per class (각 class에 속하는 image들의 weight 합 = 1)
    q_t2i = q_t2i / (q_t2i.sum(dim=1, keepdim=True) + 1e-8)
    L_t2i = -(q_t2i * log_p_t2i).sum(dim=1).mean()
    
    return (L_i2t + L_t2i) / 2
```

### 3.4 Common-mode Penalty Loss

```python
def common_mode_penalty(image_features, U):
    """
    배치 평균 feature가 common-mode direction으로 drift하면 벌점.
    H2의 KL evidence를 분포 가정 없이 대체하는 역할.
    
    image_features: (B, D)
    U: (D, r)
    """
    f_mean = image_features.mean(dim=0)  # (D,)
    drift = U.T @ f_mean  # (r,)
    return drift.pow(2).sum()
```

### 3.5 Relational Loss (J3에서 가져옴, centered)

```python
def relational_loss(image_features, text_features, logits, U, tau_rel=1.0):
    """
    J3의 L_rel. Prototype 간 관계가 text 간 관계와 같아야 한다.
    Centered space에서 작동.
    
    image_features: (B, D)
    text_features: (K, D)
    logits: (B, K) — 현재 model의 raw logits (centered가 아닌 원본)
    U: (D, r)
    """
    K = text_features.shape[0]
    
    # Soft assignment
    q = F.softmax(logits, dim=1)  # (B, K)
    
    # Prototypes
    m = (q.T @ image_features) / (q.sum(0).unsqueeze(1) + 1e-8)  # (K, D)
    
    # Center both
    t_hat = remove_common_mode(text_features, U)
    m_hat = remove_common_mode(m, U)
    
    # Normalize
    Delta_t = F.normalize(t_hat - t_hat.mean(0), dim=1)  # (K, D)
    Delta_m = F.normalize(m_hat - m_hat.mean(0), dim=1)  # (K, D)
    
    # Text relational structure (target, 고정)
    r = F.softmax(Delta_t @ Delta_t.T / tau_rel, dim=1)  # (K, K)
    
    # Prototype relational structure (변화)
    p = F.softmax(Delta_m @ Delta_t.T / tau_rel, dim=1)  # (K, K)
    
    # KL divergence per class, averaged
    loss = 0
    for k in range(K):
        loss += F.kl_div(p[k].log(), r[k], reduction='sum')
    return loss / K
```

### 3.6 Far-Negative Loss

```python
def far_negative_loss(centered_logits, text_features_centered, R=3):
    """
    Top-R candidate 밖의 class를 text 관계 기반으로 누르기.
    "이건 절대 아니다"를 학습.
    
    centered_logits: (B, K) — centered space logits
    text_features_centered: (K, D) — centered & normalized text features
    R: candidate set size
    """
    B, K = centered_logits.shape
    q = F.softmax(centered_logits, dim=1)  # (B, K)
    
    # Per-sample candidate set
    _, topR_idx = centered_logits.topk(R, dim=1)  # (B, R)
    
    # Text similarity matrix
    text_sim = text_features_centered @ text_features_centered.T  # (K, K)
    
    loss = 0
    for i in range(B):
        S_i = topR_idx[i]  # (R,)
        for k in range(K):
            if k not in S_i:
                # Weight: text상 candidate와 먼 class일수록 높은 weight
                max_sim_to_candidate = text_sim[k][S_i].max()
                w_ik = 1.0 - max_sim_to_candidate.item()
                loss += w_ik * (-torch.log(1 - q[i, k] + 1e-8))
    return loss / B

# 주의: 위 구현은 이해용. 실제로는 vectorize해서 속도를 올려야 함:
def far_negative_loss_vectorized(centered_logits, text_features_centered, R=3):
    B, K = centered_logits.shape
    q = F.softmax(centered_logits, dim=1)
    
    _, topR_idx = centered_logits.topk(R, dim=1)  # (B, R)
    text_sim = text_features_centered @ text_features_centered.T  # (K, K)
    
    # Candidate mask: (B, K), True if k is in top-R
    mask = torch.zeros(B, K, dtype=torch.bool, device=centered_logits.device)
    mask.scatter_(1, topR_idx, True)
    
    # For each sample, max text similarity between each non-candidate class and candidate classes
    # text_sim[k, S_i].max() for each k not in S_i
    # Approximate: for each sample i, gather text_sim rows for S_i, take max over S_i dimension
    candidate_text_sim = text_sim.unsqueeze(0).expand(B, K, K)  # (B, K, K)
    # Mask: only keep columns that are in S_i
    candidate_mask_expanded = mask.unsqueeze(1).expand(B, K, K)  # (B, K, K)
    candidate_text_sim_masked = candidate_text_sim.masked_fill(~candidate_mask_expanded, -1e9)
    max_sim = candidate_text_sim_masked.max(dim=2).values  # (B, K)
    
    weights = (1.0 - max_sim).clamp(min=0)  # (B, K)
    neg_loss = -torch.log(1 - q + 1e-8)  # (B, K)
    
    # Only for non-candidate classes
    non_candidate_mask = ~mask
    loss = (weights * neg_loss * non_candidate_mask.float()).sum() / B
    return loss
```

### 3.7 Candidate-Pool Margin Loss

```python
def candidate_pool_margin_loss(centered_logits, R=3, margin=1.0):
    """
    Top-R candidate로 질량을 모으되 winner를 hard하게 고르지 않는 loss.
    "top-R face 안에 있어야 한다"만 강제.
    
    centered_logits: (B, K)
    """
    _, topR_idx = centered_logits.topk(R, dim=1)  # (B, R)
    
    # Candidate mask
    mask = torch.zeros_like(centered_logits, dtype=torch.bool)
    mask.scatter_(1, topR_idx, True)
    
    # LogSumExp of candidate vs non-candidate
    large_neg = -1e9
    in_logits = centered_logits.masked_fill(~mask, large_neg)
    out_logits = centered_logits.masked_fill(mask, large_neg)
    
    lse_in = torch.logsumexp(in_logits, dim=1)    # (B,)
    lse_out = torch.logsumexp(out_logits, dim=1)   # (B,)
    
    # Margin loss: candidate logits should be higher than non-candidate by margin
    loss = torch.log(1 + torch.exp(margin - lse_in + lse_out)).mean()
    return loss
```


**판단 기준:**
- D1 > D2 (+1pp 이상): Rel이 contrastive 위에서 도움 → method에 포함
- D1 ≈ D2: Rel이 contrastive와 redundant → Rel은 analysis contribution만

### 실험 E: Gradient coherence 분석 (진단/분석용)

**핵심 질문:** 왜 Rel은 L_ent를 못 이기고, H2/contrastive는 이기는가?

**이 실험은 adaptation을 하지 않음.** Frozen 또는 초기 상태에서 첫 몇 배치의 gradient를 분석.

```python
# 첫 5개 배치에서 측정
# 각 loss의 per-sample gradient를 구해서 batch 내 coherence 측정

# 방법: 각 sample i에 대해 loss_i를 구하고, grad(loss_i, theta)를 계산
# theta = LayerNorm parameters (flatten해서 하나의 vector로)

from torch.autograd import grad

def compute_gradient_coherence(model, images, loss_fn, norm_params):
    """
    loss_fn: function(model, images_single) -> scalar loss for one sample
    Returns: mean pairwise cosine of per-sample gradients
    """
    B = images.shape[0]
    grads = []
    for i in range(B):
        model.zero_grad()
        loss_i = loss_fn(model, images[i:i+1])
        g = grad(loss_i, norm_params, create_graph=False, retain_graph=False)
        g_flat = torch.cat([gi.flatten() for gi in g])
        grads.append(g_flat)
    
    grads = torch.stack(grads)  # (B, P)
    grads_norm = F.normalize(grads, dim=1)
    
    # Pairwise cosine
    sim_matrix = grads_norm @ grads_norm.T  # (B, B)
    mask = torch.triu(torch.ones(B, B, dtype=torch.bool), diagonal=1)
    coherence = sim_matrix[mask].mean().item()
    return coherence

# 측정할 loss들:
# C(L_ent): entropy gradient coherence
# C(L_rel): relational gradient coherence
# C(L_contra): contrastive gradient coherence
# C(L_cm): common-mode penalty gradient coherence

# 추가: cat-predicted sample subset에서
# cos(g_rel_mean, g_ent_mean)
# → 음수면 반대 방향 (좋은 anti-collapse), 양수면 같은 방향 (나쁨), 0이면 무관
```

**주의:** Per-sample gradient 계산이 느릴 수 있음. B=200이면 200번 backward. 첫 2-3 배치만으로 충분.

## 7. 주의사항

1. **모든 run 시작 전 model reset.** 매 run은 원래 CLIP weight에서 시작.

2. **Prediction은 항상 raw logits로.** Loss는 centered logits로 계산하지만, accuracy 측정은 `model(images)`의 원래 logits로. 이렇게 해야 기존 baseline과 공정 비교.

3. **U는 adaptation 전에 한 번만 계산.** Text features는 frozen이므로 U도 고정.

4. **Soft contrastive의 tau_q는 1.0으로 고정.** tau_c만 sweep. tau_q를 바꾸면 변수가 너무 많아짐.

5. **Far-negative와 pool margin에서 R=3.** 우리 top-3 recall=65.2% finding에 기반.

6. **Gradient coherence 실험 (E)은 adaptation 없이 frozen model에서 첫 3 배치만.** 목적이 "왜 gradient가 다른가"의 진단이지 adaptation 효과가 아님.

7. **조기 종료: step 20에서 cat% > 80%이면 collapse 확정, 해당 run 종료.**

8. **offline_acc 측정: 각 run 끝난 후 최종 model로 전체 10K를 다시 forward해서 accuracy 측정.** Online acc와 offline acc를 모두 기록.
