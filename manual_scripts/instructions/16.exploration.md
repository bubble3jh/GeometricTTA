# CALM 방향 탐색: 8개 방향 Gate 실험 일괄 설계

**목적:** 살아있는 모든 방향을 빠르게 가설검증하여 dead-end를 조기에 기각하고, 유망한 방향만 남기기.
**원칙:** 각 방향당 1-2 run. 성능보다 "이 방향이 살아있는가?"만 판단. 총 ~15 runs, ~75분.

---

## 공통 설정

- Corruption: **gaussian_noise sev=5** (가장 hard, cat sink 발생)
- Dataset: CIFAR-10-C, N=10,000, B=200, seed=1
- Backbone: ViT-B-16 (OpenAI CLIP, QuickGELU), open_clip 2.20.0
- Metric: overall accuracy (total_correct / total_samples)
- 참조 baseline:
  - BATCLIP (L_ent + L_i2t): 0.6060 (no adapt에서) → adaptation 후 약 0.61
  - CALM v1 (L_ent + H(p̄), λ=2, I2T=off): 0.6458
  - CALM v2.2 (L_ent + H(p̄) + centered_NCE): 0.6695

---

## 방향 1: Centered NCE 단독 (H(p̄) 제거)

### 질문
H(p̄) 없이 L_ent + L_centered_NCE만으로 collapse가 방지되는가?

### 실험

| Run | Loss | 설명 |
|---|---|---|
| E1-a | L_ent only | Pure entropy minimization (collapse 예상) |
| E1-b | L_ent + L_centered_NCE (w=1.0, τ=1.0) | **핵심: NCE가 anti-collapse 역할을 하는가?** |
| E1-c | L_ent + L_centered_NCE (w=2.0, τ=1.0) | NCE weight 높여서 더 강한 pressure |

```python
# E1-a: L_ent only
loss = L_ent

# E1-b, E1-c: L_ent + centered NCE (H(p̄) 없음!)
t_bar = text_features.mean(0)
Delta_t = F.normalize(text_features - t_bar, dim=1)

m_k = (q.T @ f) / (q.sum(0, keepdim=True).T + 1e-8)
m_bar = m_k.mean(0)  # 단순 평균
Delta_m = F.normalize(m_k - m_bar, dim=1)

sim = Delta_m @ Delta_t.T / tau_nce
L_nce = F.cross_entropy(sim, torch.arange(K))

loss = L_ent + w_nce * L_nce  # H(p̄) 없음!
```

### 판단
- E1-a < 0.50 → collapse 확인 (expected)
- E1-b > 0.55 → NCE가 collapse를 어느 정도 방지 → 유망
- E1-b < 0.50 → NCE 단독으로 collapse 못 막음 → 추가 메커니즘 필요
- E1-b의 cat prediction % 모니터링 (>40%면 collapse 진행 중)

## 방향 2: Relational Anchor (centered KL matching)

### 질문
Text의 centered relational structure를 prototype에 KL match하면 더 강한가?

### 실험

| Run | Loss | 설명 |
|---|---|---|
| E2-a | L_ent + H(p̄) + L_rel_anchor (w=1.0) | H(p̄)와 함께 (기존 대비 I2T 교체) |
| E2-b | L_ent + L_rel_anchor (w=1.0) | H(p̄) 없이 relational anchor만 |

```python
# Text relational target (한 번만 계산)
t_bar = text_features.mean(0)
Delta_t = F.normalize(text_features - t_bar, dim=1)
tau_t = 1.0  # text-side temperature (별도 설정 가능)
r_k = F.softmax(Delta_t @ Delta_t.T / tau_t, dim=1)  # (K, K) text relational structure

# 매 배치:
m_k = (q.T @ f) / (q.sum(0, keepdim=True).T + 1e-8)
m_bar = m_k.mean(0)
Delta_m = F.normalize(m_k - m_bar, dim=1)

# Prototype의 relational structure
p_k = F.softmax(Delta_m @ Delta_t.T / tau_nce, dim=1)  # (K, K)

# KL matching: prototype의 relational 구조가 text의 relational 구조와 같아야
L_rel = sum(F.kl_div(p_k[k].log(), r_k[k], reduction='sum') for k in range(K)) / K

loss = L_ent + w_rel * L_rel  # E2-b: H(p̄) 없이
# 또는
loss = L_ent - lambda_mi * H_pbar + w_rel * L_rel  # E2-a: H(p̄)와 함께
```

### 판단
- E2-a vs v2.2 (0.6695): relational anchor가 centered NCE보다 나은가?
- E2-b > 0.55: relational anchor가 H(p̄) 없이도 collapse 방지 → 매우 유망
- E2-b < 0.50: relational anchor도 단독으로 collapse 못 막음

## 방향 3: Output-space correction (Adaptation-free)

### 질문
Parameter 업데이트 없이, frozen CLIP의 logit만 후처리해서 accuracy를 올릴 수 있는가?

### 실험

| Run | Method | 설명 |
|---|---|---|
| E3-a | Frozen CLIP zero-shot (raw logit) | baseline (0.3796 예상) |
| E3-b | Frozen CLIP + centered logit | logit에서 mean 빼기 |
| E3-c | Frozen CLIP + LAME-style correction | batch affinity 기반 label propagation |

```python
# E3-a: raw zero-shot
logits = f @ text_features.T * logit_scale
preds = logits.argmax(1)

# E3-b: centered logit (adaptation 없음)
t_bar = text_features.mean(0)
Delta_t = F.normalize(text_features - t_bar, dim=1)
centered_logits = f @ Delta_t.T * logit_scale  # centered text로 분류
preds = centered_logits.argmax(1)

# E3-c: LAME-style (batch-level label propagation, adaptation 없음)
# 배치 내 affinity matrix로 soft label을 전파
# W_ij = exp(cos(f_i, f_j) / sigma)
# 반복: q_new = normalize(W @ q_old)
# 최종: preds = q_new.argmax(1)

W = f @ f.T  # (B, B) affinity
W = F.softmax(W / sigma, dim=1)  # sigma = temperature
q = F.softmax(logits, dim=1)  # initial soft labels
for _ in range(num_iters):  # 3-5 iterations
    q = F.normalize(W @ q, p=1, dim=1)  # label propagation
preds = q.argmax(1)
```

**주의:** E3는 adaptation 없음. Forward만. 배치 단위로 독립 처리.

### 판단
- E3-b > E3-a: centering이 zero-shot에서도 도움 → common-mode rejection이 근본적으로 유효
- E3-b ≈ E3-a: centering은 adaptation과 함께해야만 의미
- E3-c > E3-a: output correction이 parameter adaptation 없이도 가능 → LAME 방향 유망

## 방향 4: Augmentation Consistency

### 질문
Flip consistency가 collapse 방지 또는 accuracy 개선에 기여하는가?

### 실험

| Run | Loss | 설명 |
|---|---|---|
| E4-a | L_ent + L_flip (w=1.0) | H(p̄) 없이 flip consistency만 |
| E4-b | L_ent + H(p̄) + L_flip (w=1.0) | 기존 CALM v1에 flip 추가 |

```python
# Flip consistency
with torch.no_grad():
    logits_flip = model(torch.flip(images, dims=[3]))  # horizontal flip
    q_flip = F.softmax(logits_flip, dim=1)

L_flip = F.kl_div(
    F.log_softmax(logits, dim=1),
    q_flip.detach(),
    reduction='batchmean'
)

loss = L_ent + w_flip * L_flip  # E4-a: H(p̄) 없이
```

**비용:** 추가 forward 1회 (flip). Gradient는 원본에서만.

### 판단
- E4-a > 0.55: flip consistency가 어느 정도 anti-collapse → 유망
- E4-a < 0.50: flip-invariant corruption에서 무의미 (예상)
- E4-b > CALM v1 (0.6458): flip이 H(p̄)와 함께 추가 이득 → 보조 항으로 가치

## 방향 5: Adaptive Prior (soft H(p̄))

### 질문
Uniform 대신 running EMA prior를 target으로 하면 skew에서도 작동하는가?

### 실험

| Run | Loss | 설명 |
|---|---|---|
| E5-a | L_ent + KL(p̄ ∥ π̂), π̂ from frozen teacher EMA | Adaptive prior |

```python
# Frozen teacher의 prediction으로 running prior 추정
with torch.no_grad():
    logits_teacher = frozen_model(images)
    q_teacher = F.softmax(logits_teacher, dim=1)
    p_bar_teacher = q_teacher.mean(0)  # (K,) 배치 내 class 분포

# Running prior (EMA)
pi_hat = 0.9 * pi_hat + 0.1 * p_bar_teacher
pi_hat = pi_hat / pi_hat.sum()  # normalize

# KL to adaptive prior
p_bar = q.mean(0)  # student의 배치 marginal
L_kl_prior = F.kl_div(p_bar.log(), pi_hat.detach(), reduction='sum')

loss = L_ent + lambda_kl * L_kl_prior
```

**주의:** frozen_model = adaptation 전의 CLIP (원본 weight 복사). 추가 forward 1회.
**초기화:** π̂_0 = uniform (1/K).

### 판단
- E5-a > CALM v1 (0.6458): adaptive prior가 더 나음 → 유망
- E5-a ≈ CALM v1: teacher가 오염되어서 uniform과 큰 차이 없음
- E5-a << CALM v1: teacher prior가 collapse를 허용 (GPT-Pro 비판대로)
- **추가:** 이걸 skewed distribution에서도 돌려야 진짜 판단 가능 (하지만 gate에서는 gaussian만)

## 방향 6: Active-set 진단 (TopR에 true class가 살아있는가?)

### 질문
Cat sink에서 top-1=cat인 샘플의 top-3, top-5에 true class가 얼마나 남아있는가?

### 실험

| Run | Method | 설명 |
|---|---|---|
| E6 | 진단 측정 (adaptation 없음) | Frozen CLIP forward + TopR 분석 |

```python
# Frozen CLIP (adaptation 없음)으로 gaussian_noise forward
logits = model(images)  # (B, K)
probs = F.softmax(logits, dim=1)

# Centered logits도 측정
t_bar = text_features.mean(0)
Delta_t = F.normalize(text_features - t_bar, dim=1)
centered_logits = f @ Delta_t.T * logit_scale

for R in [2, 3, 5]:
    # Raw logit TopR
    topR_raw = logits.topk(R, dim=1).indices  # (N, R)
    true_in_topR_raw = (topR_raw == labels.unsqueeze(1)).any(dim=1).float().mean()

    # Centered logit TopR
    topR_cen = centered_logits.topk(R, dim=1).indices
    true_in_topR_cen = (topR_cen == labels.unsqueeze(1)).any(dim=1).float().mean()

    print(f"R={R}: raw={true_in_topR_raw:.3f}, centered={true_in_topR_cen:.3f}")

# Cat-specific: top-1=cat인 샘플에서 true class가 top-R에 있는 비율
cat_mask = (logits.argmax(1) == 3)  # cat으로 예측된 샘플
for R in [2, 3, 5]:
    topR = logits[cat_mask].topk(R, dim=1).indices
    true_labels_cat = labels[cat_mask]
    true_in_topR = (topR == true_labels_cat.unsqueeze(1)).any(dim=1).float().mean()
    print(f"Cat-predicted, R={R}: true_in_topR={true_in_topR:.3f}")
```

### 판단
- Top-3에 true class가 >60%: active-set / TopR 방향 유망
- Top-3에 true class가 <40%: TopR도 오염, active-set 방향 약함
- Centered logit의 TopR > raw logit의 TopR: centering이 TopR도 개선

## 방향 7: Weight Anchor (parameter regularization)

### 질문
LayerNorm이 원래에서 멀어지지 않게 L2 제한하면 collapse가 줄어드는가?

### 실험

| Run | Loss | 설명 |
|---|---|---|
| E7-a | L_ent + L2_anchor (w=0.1) | 약한 weight anchor |
| E7-b | L_ent + L2_anchor (w=1.0) | 강한 weight anchor |

```python
# 원래 LayerNorm parameters 저장
theta_0 = {}
for name, param in model.named_parameters():
    if param.requires_grad:  # LayerNorm만
        theta_0[name] = param.data.clone()

# Adaptation 중:
L2_anchor = sum(
    (param - theta_0[name]).pow(2).sum()
    for name, param in model.named_parameters()
    if name in theta_0
)

loss = L_ent + w_anchor * L2_anchor
```

### 판단
- E7 > E1-a (L_ent only): weight anchor가 collapse를 줄임 → 보조 항으로 가치
- E7 ≈ E1-a: weight anchor만으로는 collapse 못 막음
- E7 > 0.55: weight anchor 단독으로 어느 정도 작동 → 다른 방향과 조합 가치

### 로깅 (모든 run 공통)

```python
result = {
    "run_id": str,
    "direction": str,
    "loss_components": [str],
    "overall_acc": float,
    "pred_distribution": [float] * 10,  # 각 class 예측 비율
    "cat_fraction": float,              # cat 예측 비율 (sink 모니터링)
    "mean_entropy": float,              # 배치 평균 entropy
    "H_pbar_final": float,              # 최종 marginal entropy
}
```

### 주의사항

1. **E3 (output correction)은 adaptation 없음.** Model parameter 변경 금지. Forward만.
   배치 단위로 독립 처리. 전체 10K에 대한 overall accuracy.

2. **E5 (adaptive prior)는 frozen teacher 필요.** Adaptation 전 model의 사본.
   `frozen_model = copy.deepcopy(model); frozen_model.eval()` — VRAM 주의.
   대안: frozen model 대신 adaptation 전 1회 forward해서 logits 저장 후 사용.
   → 메모리 효율적. `teacher_logits = model(all_images).detach()` 한 번에.
   → 하지만 online setting에서는 미래 배치를 볼 수 없으므로,
   → **각 배치에서 adaptation 전의 현재 model로 teacher prediction 추출.**
   → 사실상 stop-gradient self-distillation.

3. **E6 (active-set 진단)은 true label 사용.** 이건 진단이니까 OK. 실제 method에서는 true label 없음.

4. **E1-a (L_ent only)는 collapse할 것.** 이건 "collapse가 진짜 일어나는가" 확인용.
   중간에 acc가 10% 이하로 떨어지면 조기 종료해도 됨.

5. **E3-c (LAME)의 sigma와 iteration.**
   sigma: affinity matrix의 temperature. {0.1, 0.5} 시도.
   iteration: label propagation 반복 횟수. 5회면 충분.

6. **E2 (relational anchor)에서 tau_t와 tau_nce.**
   tau_t = 1.0 (text relational structure 계산용)
   tau_nce = 1.0 (prototype relational structure 계산용, Gate B best와 동일)
