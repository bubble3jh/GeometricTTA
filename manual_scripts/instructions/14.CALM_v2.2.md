# CALM v2.2: Centered Proto-NCE 실험 설계서

**날짜:** 2026-03-10
**목적:** I2T의 공통 방향 문제를 centering으로 해결하여, 단일 config에서 I2T가 일관되게 도움이 되도록 함
**선행 기각:** v2 (c_ik filtering, AUC=0.43), v2.1 (raw text-subspace projection, effective rank=2.03)
**원칙:** Gate A → B → C → D 순서. 각 gate 실패 시 다음 gate로. Gate A 실패 시 전체 방향 기각.

---

## Part 0: 이전 실패와 이번의 차이

### v2.1이 실패한 이유
```
Raw text SVD:
  Singular values: [2.926, 0.611, 0.462, ...]
  S_0이 전체 에너지의 85.6% → "a photo of a" 공통 방향이 지배
  Effective rank = 2.03 → 사실상 2차원
  Pairwise cosine mean = 0.84 → 10개 class가 near-collinear

P_T projection → 모든 feature를 rank-2 공간으로 압축
→ cos(prototype, text) ≈ 0.95 → gradient ≈ 0 → I2T가 LayerNorm 업데이트에 기여 안 함
```

### 이번 접근이 다른 이유
```
Centering: t_k → Δt_k = t_k - t̄

공통 방향 (S_0 = 2.926)을 빼면:
→ 남는 건 class-discriminative 차이만
→ effective rank가 올라갈 것 (Gate A에서 확인)
→ projection이 아니라 subtraction이므로 정보 손실 없음

추가로 image prototype도 centering:
→ Δm_k = m_k - m̄
→ 공통 corruption shift가 m̄에 흡수되어 상쇄
→ Δm_k에는 class-discriminative 차이만 남음
```

**핵심 메커니즘 (common-mode rejection):**
```
corruption이 모든 feature를 같은 방향으로 밀면:
  m_k ≈ m_k_clean + corruption_shift
  m̄  ≈ m̄_clean  + corruption_shift
  Δm_k = m_k - m̄ ≈ m_k_clean - m̄_clean   ← corruption 상쇄!
```

## Part 1: Gate A — Centered Text SVD (생사 판정)

### 목적
Mean-centering 후 text embedding의 effective rank를 측정.
이것이 올라가지 않으면 centering 방향 전체가 무의미.

### 구현

```python
import torch
import torch.nn.functional as F

def gate_a_centered_text_svd(text_features):
    """
    Args:
        text_features: (K, D) L2-normalized text embeddings, K=10, D=512
    Prints:
        Raw vs Centered SVD comparison
    """
    K, D = text_features.shape

    # === Raw (기존, 비교용) ===
    U_raw, S_raw, V_raw = torch.linalg.svd(text_features, full_matrices=False)
    eff_rank_raw = (S_raw.sum() ** 2) / (S_raw ** 2).sum()
    energy_raw = (S_raw ** 2).cumsum(0) / (S_raw ** 2).sum()
    cos_raw = (text_features @ text_features.T)
    cos_raw_mean = (cos_raw.sum() - cos_raw.trace()) / (K * (K - 1))

    # === Centered ===
    t_bar = text_features.mean(dim=0)  # (D,)
    T_centered = text_features - t_bar  # (K, D)

    U_cen, S_cen, V_cen = torch.linalg.svd(T_centered, full_matrices=False)
    eff_rank_cen = (S_cen.sum() ** 2) / (S_cen ** 2).sum()
    energy_cen = (S_cen ** 2).cumsum(0) / (S_cen ** 2).sum()

    # Centered pairwise cosine (re-normalize 후)
    T_cen_norm = F.normalize(T_centered, dim=1)
    cos_cen = (T_cen_norm @ T_cen_norm.T)
    cos_cen_mean = (cos_cen.sum() - cos_cen.trace()) / (K * (K - 1))

    print("=== Raw Text SVD ===")
    print(f"  Singular values: {S_raw[:5].tolist()}")
    print(f"  Effective rank: {eff_rank_raw:.2f}")
    print(f"  Energy @1: {energy_raw[0]:.3f}, @3: {energy_raw[2]:.3f}, @5: {energy_raw[4]:.3f}")
    print(f"  Pairwise cosine mean: {cos_raw_mean:.3f}")

    print("\n=== Centered Text SVD ===")
    print(f"  Singular values: {S_cen[:5].tolist()}")
    print(f"  Effective rank: {eff_rank_cen:.2f}")
    print(f"  Energy @1: {energy_cen[0]:.3f}, @3: {energy_cen[2]:.3f}, @5: {energy_cen[4]:.3f}")
    print(f"  Pairwise cosine mean: {cos_cen_mean:.3f}")

    return eff_rank_cen
```

### 판단 기준

| Centered Effective Rank | 판단 | 다음 행동 |
|---|---|---|
| > 5 | Centering이 효과적. Class-discriminative 구조 존재 | → Gate B |
| 3 ~ 5 | 부분적. 시도 가치 있음 | → Gate B (cautious) |
| < 3 | Centering도 불충분. CIFAR-10 text가 근본적으로 약함 | → 방향 기각 |

**비용:** 코드 실행 ~1분. GPU 불필요 (CPU에서 SVD 가능).

## Part 2: Gate B — Centered I2T 실험 (핵심)

### 전제: Gate A 통과 (centered effective rank > 3)

### 실험할 I2T 변형 4가지

#### B0: CALM v1 I2T=off (baseline)
```python
L = L_ent - lambda_mi * H_pbar
# I2T 없음
```

#### B1: CALM v1 I2T=uniform raw (기존)
```python
v_bar_k = (q.T @ f) / (q.sum(0, keepdim=True).T + 1e-8)
v_hat_k = F.normalize(v_bar_k, dim=1)
L_i2t = (v_hat_k * text_features).sum(dim=1).mean()
L = L_ent - lambda_mi * H_pbar - w_i2t * L_i2t
```

#### B2: Centered Cosine I2T (centering만, loss는 cosine)
```python
# text centering (한 번)
t_bar = text_features.mean(dim=0)
Delta_t = F.normalize(text_features - t_bar, dim=1)  # (K, D)

# prototype centering (매 배치)
m_k = (q.T @ f) / (q.sum(0, keepdim=True).T + 1e-8)   # (K, D)
m_bar = (q.sum(0) @ m_k) / (q.sum() + 1e-8)            # (D,) weighted mean
Delta_m = F.normalize(m_k - m_bar, dim=1)               # (K, D)

# centered cosine I2T
L_i2t_centered = (Delta_m * Delta_t).sum(dim=1).mean()
L = L_ent - lambda_mi * H_pbar - w_i2t * L_i2t_centered
```

#### B3: Centered Proto-NCE (centering + contrastive)
```python
# text centering (한 번)
t_bar = text_features.mean(dim=0)
Delta_t = F.normalize(text_features - t_bar, dim=1)

# prototype centering (매 배치)
m_k = (q.T @ f) / (q.sum(0, keepdim=True).T + 1e-8)
m_bar = (q.sum(0) @ m_k) / (q.sum() + 1e-8)
Delta_m = F.normalize(m_k - m_bar, dim=1)

# Proto-NCE: 각 prototype이 자기 text에 다른 text보다 가까운 순서 유지
sim = Delta_m @ Delta_t.T / tau  # (K, K)
L_proto_nce = F.cross_entropy(sim, torch.arange(K, device=sim.device))
L = L_ent - lambda_mi * H_pbar - w_i2t * L_proto_nce
```

### 판단 기준

**성공 조건 (둘 다 만족):**
- gaussian_noise: B2 또는 B3 > B0 (I2T=off) AND > B1 (raw I2T)
- brightness: B2 또는 B3 ≥ B1 (기존 I2T on보다 같거나 나음)

**부분 성공:**
- gaussian에서만 개선, brightness에서 동등 → 채택 (gaussian이 hard case)
- brightness에서만 개선, gaussian에서 동등 → 의미 제한적

**실패:**
- B2, B3 모두 ≤ B0 → centering 방향 기각. Gate D (nuisance subtraction) 검토
- B2, B3가 B1과 동등 → centering이 효과 없음 (공통 방향 제거가 도움 안 됨)

```python
step_log["cos_centered_prototype_text"] = (Delta_m * Delta_t).sum(1).mean().item()
step_log["cos_raw_prototype_text"] = (v_hat_k * text_features).sum(1).mean().item()
```

**핵심 확인:** cos_centered_prototype_text가 0.9 이상으로 saturate하지 않는지.
v2.1에서는 cos_raw=0.95로 saturate → gradient 소실. Centered에서도 이러면 실패.

### 주의사항

1. **Δt_k의 re-normalization.** t_k - t̄를 normalize해야 unit sphere 위에서 비교 가능.
   Normalize 안 하면 class별 Δt_k 크기가 달라서 불균형.

2. **Δm_k의 zero-vector 방지.** 만약 모든 prototype이 같은 방향이면 m̄ = m_k → Δm_k = 0.
   이건 collapse 상황에서 발생 가능. Fallback: Δm_k norm < ε이면 raw m_k 사용.

3. **m̄ 계산 방식.** 단순 평균 vs class mass 가중 평균.
   가중: m̄ = Σ_k (Σ_i q_ik) · m_k / Σ_i,k q_ik. Cat이 많으면 m̄가 cat 쪽으로 기울어.
   단순: m̄ = (1/K) Σ_k m_k. Class mass 무관.
   **단순 평균을 권장** — class mass를 무시하면 sink class의 영향을 줄일 수 있어.

4. **Proto-NCE의 tau.** v2.1의 raw projection에서는 cos ≈ 0.95라 tau=0.01에서 explode.
   Centered에서는 cos가 더 낮을 것이므로 tau를 높게 잡아야 할 수 있음.
   Sweep: {0.1, 0.5, 1.0}. CLIP 원래 tau (0.01)는 너무 sharp할 수 있음.

5. **w_i2t (I2T loss weight).** 기존 CALM v1에서 w_i2t=1.0. Proto-NCE의 scale이 다를 수 있으므로
   w_i2t ∈ {0.5, 1.0, 2.0} sweep 필요할 수 있음. 하지만 Gate B에서는 w_i2t=1.0 고정.

6. **Gradient flow.** Δm_k는 q와 f를 통해 differentiable.
   m̄도 q를 통해 differentiable.
   Δt_k는 frozen (stop-gradient).
   → gradient: L → Δm_k → m_k → q, f → logits → LayerNorm. 정상 작동.

## Part 3: Gate C — Streaming Prototype (Gate B 성공 시)

### 목적
Per-batch prototype → streaming EMA prototype으로 변경하여 variance 감소.

```python
class StreamingCenteredPrototype:
    def __init__(self, K, D, momentum=0.9):
        self.running_sum = torch.zeros(K, D)    # Σ q_ik · f_i 누적
        self.running_count = torch.zeros(K)      # Σ q_ik 누적
        self.momentum = momentum

    def update(self, f, q, text_features):
        K = q.shape[1]

        # EMA update
        batch_sum = q.T @ f                      # (K, D)
        batch_count = q.sum(dim=0)               # (K,)
        self.running_sum = self.momentum * self.running_sum + (1 - self.momentum) * batch_sum
        self.running_count = self.momentum * self.running_count + (1 - self.momentum) * batch_count

        # prototype
        m_k = self.running_sum / (self.running_count.unsqueeze(1) + 1e-8)  # (K, D)

        # centering
        m_bar = m_k.mean(dim=0)                  # 단순 평균
        Delta_m = F.normalize(m_k - m_bar, dim=1)

        t_bar = text_features.mean(dim=0)
        Delta_t = F.normalize(text_features - t_bar, dim=1)

        return Delta_m, Delta_t
```

### 실험

| # | Corruption | Momentum | 비교 |
|---|---|---|---|
| C1 | gaussian_noise | 0.9 | vs Gate B best |
| C2 | gaussian_noise | 0.7 | vs Gate B best |
| C3 | gaussian_noise | 0.5 | vs Gate B best |
| C4 | brightness | (best momentum) | floor check |

### 판단
Streaming > per-batch이면 채택. 아니면 per-batch 유지.

## Part 4: Gate D — Nuisance Subtraction (Gate B에서 gaussian만 실패 시)

### 목적
Corruption direction을 직접 추정하여 feature에서 제거.
Panda (ECCV 2024) 스타일의 negative augmentation 활용.

### 방법

```python
def create_negative_images(images, block_size=8):
    """
    이미지의 8×8 블록을 랜덤 셔플하여 semantic 파괴, corruption pattern 유지.
    CIFAR-10 32×32 → 4×4 grid of 8×8 blocks.
    """
    B, C, H, W = images.shape
    blocks_h = H // block_size  # 4
    blocks_w = W // block_size  # 4
    n_blocks = blocks_h * blocks_w  # 16

    # block 단위로 분할
    blocks = images.unfold(2, block_size, block_size).unfold(3, block_size, block_size)
    blocks = blocks.contiguous().view(B, C, n_blocks, block_size, block_size)

    # 각 이미지 내에서 block 순서를 랜덤 셔플
    idx = torch.stack([torch.randperm(n_blocks) for _ in range(B)])
    shuffled = blocks[torch.arange(B).unsqueeze(1), :, idx]

    # 다시 이미지로 조합
    shuffled = shuffled.view(B, C, blocks_h, blocks_w, block_size, block_size)
    shuffled = shuffled.permute(0, 1, 2, 4, 3, 5).contiguous()
    neg_images = shuffled.view(B, C, H, W)
    return neg_images


# 사용:
x_neg = create_negative_images(images, block_size=8)
with torch.no_grad():
    f_neg = model.encode_image(x_neg)  # frozen forward (gradient 불필요)
nu = f_neg.mean(dim=0)  # batch-shared corruption direction (D,)

# feature에서 corruption 방향 제거
f_clean = f - beta * (f @ nu).unsqueeze(1) * nu.unsqueeze(0)  # projection 제거
f_clean = F.normalize(f_clean, dim=1)

# 이후 centering + Proto-NCE는 f_clean으로 계산
```

### 실험

| # | Corruption | beta | 비교 |
|---|---|---|---|
| D1 | gaussian_noise | 0.5 | vs Gate B best |
| D2 | gaussian_noise | 1.0 | vs Gate B best |
| D3 | gaussian_noise | 2.0 | vs Gate B best |
| D4 | brightness | (best beta) | floor check |

### 비용
추가 forward 1회 (negative images). Gradient 불필요라 메모리 부담 적음.
32×32 이미지에서 8×8 block shuffle은 semantic을 효과적으로 파괴.

### 판단
D > Gate B best이면 채택. 아니면 불필요.

## Part 5: 전체 의사결정 트리

```

  ├─ Centered eff rank 3~5 → Gate B (cautious)
  └─ Centered eff rank < 3 → 방향 기각

Gate B: Centered I2T 실험 (~40분)
  ├─ B2/B3 > B0 AND > B1 (both corruptions) → Gate C (대성공)

  ├─ B2/B3 > B0 BUT ≤ B1 → centering 도움, contrastive 불필요
  ├─ B2/B3 ≈ B0 (gaussian), B2/B3 ≥ B1 (brightness) → centering 효과 없음
  └─ B2/B3 < B0 → centering 해로움 → Gate D 또는 기각

Gate C: Streaming Prototype (~20분)
  ├─ Streaming > per-batch → 채택
```

```python
#
# 핵심 차이 (vs CALM v1):
#   1. Text centering: Δt_k = normalize(t_k - t̄)
#      → 공통 text 방향 ("a photo of a") 제거
#      → class-discriminative 방향만 남김
#
#   2. Prototype centering: Δm_k = normalize(m_k - m̄)
#      → 공통 corruption shift 상쇄
#      → class-discriminative 차이만 남김
#
#   3. Proto-NCE loss: cross_entropy(Δm @ Δt.T / τ, arange(K))
#      → absolute alignment 대신 relative ranking
#      → prototype이 약간 오염되어도 순서가 맞으면 loss ≈ 0
#
# 추가 하이퍼파라미터: τ (NCE temperature), 1개

#   backbone = ViT-B-16 (OpenAI, QuickGELU)
#   trainable = LayerNorm only (~65K params)
#   optimizer = AdamW(lr=1e-3, wd=0.01)
#   batch_size = 200
#   λ = 2.0 (H(p̄) weight)
```

### CLI
```bash
python run_calm_v2.2.py \
    --cfg cfgs/cifar10_c/soft_logit_tta.yaml \
    DATA_DIR ./data \
    --corruption gaussian_noise \
    --lambda_mi 2.0 \
    --i2t_mode centered_nce \    # off / uniform_raw / centered_cosine / centered_nce
    --tau 0.5 \                  # NCE temperature
    --streaming off \            # off / on
    --momentum 0.9 \             # streaming momentum (streaming=on 시)
    --nuisance off \             # off / on
    --nuisance_beta 1.0 \        # nuisance subtraction strength
    --output_dir experiments/runs/calm_v2.2/
```
