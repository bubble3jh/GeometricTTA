# CAMA Experiment Instructions (Code Agent)

## 배경

CAMA는 CLIP 기반 Test-Time Adaptation 방법론입니다. Corruption된 이미지에 대해 CLIP 모델의 LayerNorm 파라미터만 adaptation합니다.

### 핵심 Loss
```
L_solver = (1/B) Σ H(p_i) + λ · KL(p̄ || π)
```
- `H(p_i)`: 개별 prediction의 entropy (낮을수록 discriminative)
- `KL(p̄ || π)`: batch marginal p̄와 corruption reference π 사이의 KL divergence
- `λ`: 두 loss 간 balance (자동 결정)

### 2-Phase 구조
- **Phase 1 (Calibration)**: N_cal batches를 forward하여 π 추정 + λ 측정. 모델 업데이트 없음.
- **Phase 2 (Adaptation)**: 매 batch마다 L_solver로 1 step gradient update.

## 코드 수정 사항 (3가지)

### 수정 A: λ Gradient Accumulation

**문제**: 현재 코드는 λ₀ = ||g_E|| / ||g_K||를 1 batch에서만 backward하여 계산합니다.
논문에는 "N_cal calibration batches 전체에서 측정"으로 기술되어 있으므로, 코드를 논문에 맞춥니다.

**원리**: λ₀는 entropy loss와 KL loss의 gradient scale 비율입니다.
- g_E = ∇θ[(1/N_cal·B) Σ_all H(p_i)]: entropy gradient
- g_K = ∇θ[KL(p̄_pooled || π)]: KL gradient
- λ₀ = ||g_E|| / ||g_K||

1 batch (200 samples)로 측정하면 noisy합니다. 특히 K=1000, B=64일 때 1 batch = 64 samples로 1000 classes의 gradient ratio를 추정하는 셈입니다.

**수정 방법**: PyTorch의 gradient accumulation을 사용합니다. `zero_grad()` 없이 `backward()`를 반복하면 gradient가 자동 누적됩니다.

```python
# ===== 현재 코드 (1 batch) =====
imgs_b0 = next(iter(loader_lam))[0]  # 1 batch만
logits = model(imgs_b0)
p = softmax(logits / tau)

# g_E
l_ent = -p.log().mul(p).sum(-1).mean()  # (1/B) Σ H(p_i)
l_ent.backward()
g_E = torch.cat([param.grad.flatten() for param in adapted_params]).norm()
model.zero_grad()

# g_K
p_bar = p.mean(0)
kl = (p_bar * (p_bar.log() - pi.log())).sum()
kl.backward()
g_K = torch.cat([param.grad.flatten() for param in adapted_params]).norm()
model.zero_grad()

lambda_0 = (g_E / g_K).item()


# ===== 수정 코드 (N_cal batches gradient accumulation) =====
# cal_batches: Phase 1에서 π 추정에 사용한 N_cal개 batch 리스트
# 이미 forward하여 logits를 얻은 것과 동일한 batches를 재사용

# g_E: entropy gradient 누적
model.zero_grad()
for batch_imgs in cal_batches:
    logits = model(batch_imgs)
    p = softmax(logits / tau)
    l_ent = -p.log().mul(p).sum(-1).mean() / N_cal  # N_cal로 나눠서 평균
    l_ent.backward()  # zero_grad 안 함 → gradient 누적
# 이 시점: param.grad = (1/N_cal) Σ_batches ∇H
g_E = torch.cat([param.grad.flatten() for param in adapted_params]).norm()

# g_K: KL gradient (surrogate method — memory-efficient, mathematically exact)
#
# 문제: KL(p̄_pooled || π)를 N_cal batches 전체에 대해 backward하려면
#       N_cal*B*K tensor를 computation graph에 유지해야 → K=1000, N_cal=16에서 VRAM 초과
#
# 해법: chain rule ∇_θ KL = (dKL/dp̄) · (dp̄/dθ) 를 수동 분해
#       Step 1: p̄_pooled를 no_grad로 계산
#       Step 2: dKL/dp̄ 계산 (K-dim vector, 상수)
#       Step 3: 각 batch에서 surrogate loss backward → gradient 누적
#
# 수학적 검증:
#   surrogate 누적 gradient = Σ_k (dKL/dp̄_k) · ∂p̄_pooled_k/∂θ = ∇_θ KL(p̄_pooled || π)
#
# ⚠️ Naive 대안 (per-batch KL 평균)은 틀립니다:
#   (1/N_cal) Σ_b ∇KL(p̄_batch_b || π) ≠ ∇KL(p̄_pooled || π)
#   KL이 nonlinear이므로 Jensen's inequality에 의해 gradient가 과대추정됩니다.

model.zero_grad()

# Step 1: p̄_pooled 계산 (no grad, 메모리 K-dim만 사용)
with torch.no_grad():
    p_bar_accum = torch.zeros(K, device=device)
    for batch_imgs in cal_batches:
        logits = model(batch_imgs)
        p = softmax(logits / tau)
        p_bar_accum += p.mean(0)
    p_bar_pooled = p_bar_accum / N_cal  # (K,)

# Step 2: dKL/dp̄ at p̄_pooled (K-dim constant)
dKL_dp = p_bar_pooled.log() - pi.log() + 1  # (K,)

# Step 3: surrogate backward (1 batch씩, gradient 누적)
for batch_imgs in cal_batches:
    logits = model(batch_imgs)  # WITH grad
    p = softmax(logits / tau)
    p_bar_batch = p.mean(0)  # (K,)
    surrogate = (dKL_dp.detach() * p_bar_batch).sum() / N_cal
    surrogate.backward()  # zero_grad 안 함 → gradient 누적

g_K = torch.cat([param.grad.flatten() for param in adapted_params]).norm()
model.zero_grad()

lambda_0 = (g_E / g_K).item()
```

**주의사항**:
- `cal_batches`는 π 추정에 사용한 것과 동일한 데이터여야 합니다.
- g_E와 g_K는 반드시 별도의 backward pass (사이에 zero_grad)로 수집해야 합니다.
- `l_ent`를 N_cal로 나눈 뒤 backward하면 누적 gradient가 자동으로 평균이 됩니다.
- g_K의 surrogate method는 메모리 1 batch 분량만 사용하면서 exact gradient를 계산합니다.

**⚠️ 이전 코드의 N_cal 불일치 (K=1000)**:
- 이전 코드: K=1000에서 π 추정에 **1 batch (64 samples)**만 사용
- 논문 수식: N_cal = max(3, ⌈K/B⌉) = 16 → **16 batches (1024 samples)**
- K=10/100에서는 이전에도 N_cal=3을 사용 (정상)
- 이번 수정에서 K=1000의 π 추정도 16 batches로 수정합니다
- 따라서 K=1000의 이전 9개 결과는 전부 폐기, 15개 전체 재실행

### 수정 B: λ Scaling 공식 변경

**문제**: 현재 `min(1, B/K)` → `B/(B+K-1)`로 변경

**원리**: 매 adaptation step에서 p̄ = (1/B)Σp_i는 finite sample estimate입니다.
KL(p̄||π)의 expected bias ≈ (K-1)/(2B) (chi-squared approximation).
K=1000, B=64일 때 이 bias = 7.8로, KL signal보다 noise가 큽니다.
B/(B+K-1)는 Dirichlet shrinkage에서 유도된 first-order correction입니다.

```python
# 현재
lambda_eff = 1 + (lambda_0 - 1) * min(1, B / K)

# 수정
lambda_eff = 1 + (lambda_0 - 1) * (B / (B + K - 1))
```

**영향 범위**:
```
K=10,  B=200: 1.0   → 0.957  (미미한 차이)
K=100, B=200: 1.0   → 0.669  (유의미한 변화!)
K=1000, B=64: 0.064 → 0.060  (거의 동일)
```
K=100 결과가 가장 크게 바뀔 것으로 예상됩니다.

ALL_CORRUPTIONS = ['gaussian_noise', 'shot_noise', 'impulse_noise',
    'defocus_blur', 'glass_blur', 'motion_blur', 'zoom_blur',
    'snow', 'frost', 'fog', 'brightness',
    'contrast', 'elastic_transform', 'pixelate', 'jpeg_compression']

all_results = []

for corruption in ALL_CORRUPTIONS:
    # === Skip gaussian_noise if already completed (PC only) ===
    if corruption == 'gaussian_noise' and os.path.exists(f'outputs/main_table/gaussian_noise.json'):
        print("gaussian_noise already completed, loading from previous run")
        all_results.append(load_json('outputs/main_table/gaussian_noise.json'))
        continue

    # === Model reset (episodic) ===
    model = load_model()  # pre-trained weights로 매 corruption마다 reset
    configure_model(model)  # LayerNorm만 adapt
    optimizer = make_optimizer(model)  # 매 corruption마다 새로 생성
    # === Phase 1: Calibration (model 업데이트 없음) ===
    cal_batches = load_calibration_data(corruption, N_cal)
    pi, lambda_0, lambda_eff, I_batch_0 = calibrate(model, cal_batches, ...)

    # === Phase 2: Adaptation ===
    adapt_loader = make_loader(corruption, BS, N_TOTAL)
    
    # step-wise logging은 gaussian_noise에서만 활성화
    do_step_log = (corruption == 'gaussian_noise')
    trajectory = []  # gaussian_noise일 때만 사용
    
    running_correct, running_total = 0, 0
    
    for step, (images, labels) in enumerate(adapt_loader):
        logits = model(images)
        p = softmax(logits / tau)
        
        # --- Loss 계산 + backward + optimizer step (매 batch) ---
        loss = compute_cama_loss(p, pi, lambda_eff)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        
        # --- Online accuracy (매 batch) ---
        preds = logits.argmax(-1)
        running_correct += (preds == labels).sum().item()
        running_total += len(labels)
        online_acc = running_correct / running_total
        
        # --- Step-wise logging (gaussian_noise에서만) ---
        if do_step_log:
            p_bar = p.mean(0)
            H_pbar = -(p_bar * p_bar.log()).sum().item()
            I_batch = H_pbar - (-p.log().mul(p).sum(-1)).mean().item()
            cos = compute_pairwise_cos(model.encode_image(images))

            # 매 10 step: 추가 offline metrics
            if step % 10 == 0 or step == 49:

                row.update({'offline_acc': off_acc, 
                            'overconf_wrong': overconf, 'ece': ece})
            
            trajectory.append(row)
    
    # === Phase 2 완료 후 ===
    offline_acc = offline_eval(model, corruption)
    
    result = {
        'corruption': corruption,
        'lambda_0': lambda_0,
        'lambda_eff': lambda_eff,
        'online_acc': online_acc,
        'offline_acc': offline_acc,
    }
    all_results.append(result)
    
    # gaussian_noise: trajectory 저장 + analysis features 저장
    if corruption == 'gaussian_noise':
        save_csv(trajectory, 'outputs/figure2/trajectory_CAMA.csv')
        save_embeddings(model, corruption, 'outputs/analysis/')
    
    # Analysis metrics (K=10: 모든 corruption)
    collect_analysis_metrics(model, corruption, ...)
    
    # Per-corruption JSON (crash recovery)
    save_json(result, f'outputs/main_table/{corruption}.json')
    
    # model은 다음 corruption에서 reset됨 (episodic)

save_csv(all_results, 'outputs/main_table/k10_results.csv')
```

H_pbar = -(p_bar * p_bar.log()).sum().item()

pairwise_cos = compute_pairwise_cos(model.encode_image(batch))

# 매 10 step에서 추가 (offline eval):
# 전체 test set forward
all_logits = model_forward_all(test_set)
all_p = softmax(all_logits / tau)
preds = all_p.argmax(-1)
labels = test_labels

offline_acc = (preds == labels).float().mean().item()

# Overconfident wrong (threshold 0.9)
max_conf = all_p.max(-1).values
wrong = (preds != labels)
overconf_wrong = ((max_conf > 0.9) & wrong).float().mean().item()

# ECE (10 bins)
ece = compute_ece(max_conf, (preds == labels).float(), n_bins=10)
```

**ECE 계산 함수**:
```python
def compute_ece(confidences, accuracies, n_bins=10):
    bin_boundaries = torch.linspace(0, 1, n_bins + 1)
    ece = 0.0
    for i in range(n_bins):
        mask = (confidences > bin_boundaries[i]) & (confidences <= bin_boundaries[i+1])
        if mask.sum() > 0:
            avg_conf = confidences[mask].mean()
            avg_acc = accuracies[mask].mean()
            ece += mask.float().mean() * abs(avg_conf - avg_acc)
    return ece.item()
```

## 검증 체크리스트

실행 후 확인사항:

```
□ Episodic 확인: 각 corruption의 lambda_0 값이 독립적인지 
  → non-episodic이면 후반 corruption의 λ₀가 이전 adaptation 영향을 받음
  → gaussian_noise(첫 번째)와 비슷한 범위인지 확인

□ lambda_0 값이 기존과 다른지 확인 (grad accumulation 효과)
  → K=10에서는 큰 차이 없을 수 있음 (B=200 >> K=10)
  → K=100/1000에서 차이 확인

□ lambda_eff 값이 기존과 다른지 확인 (B/(B+K-1) 효과)
  → K=10: 기존 λ₀ vs 새 λ₀×0.957 → 거의 동일 예상
  → K=100: 기존 λ₀ vs 새 λ₀×0.669 → 유의미한 차이 예상

□ online_acc가 기존 (이전 episodic 결과)과 비교하여 ±2pp 이내인지 확인
  → 크게 달라지면 코드 버그 가능성 점검

□ 모든 corruption에서 collapse 없음 확인
  → cat_pct (또는 max class %) < 15% for K=10
  → cat_pct < 5% for K=100

□ cos_corrupt > cos_clean 확인 (corruption이 cone을 압축)
  → noise 계열(gauss, shot, impulse)에서 특히 분명해야 함
  → blur/weather에서는 덜 할 수 있음

□ cos_adapted < cos_corrupt 확인 (CAMA가 cone을 재개방)
  → 15/15 corruption 모두

□ Spearman r > 0.8 확인 (모든 corruption)
```
