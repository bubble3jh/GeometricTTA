## 1. 배경 및 동기

GPT-Pro는 CALM-AV를 제안했다. 핵심은 두 개의 detached gate:

- **Class gate** `q_k`: soft prototype `μ_k`의 prototype-anchor margin `m_k`로 evidence `s_k`를 reweight → `π_k ∝ (q_k · s_k + α)^β`
- **Sample gate** `a_i`: excess img-txt alignment으로 `L_ent`의 per-sample magnitude 조절

이 제안의 타당성은 두 가지 empirical 전제에 의존한다.

**전제 1 (Class gate):** collapse 진행 중 sink class의 `s_k`가 커질 때, `m_k` (prototype-anchor margin)는 커지지 않거나 오히려 떨어진다. 즉 `s_k`와 `m_k`가 dissociate한다.

**전제 2 (Sample gate):** excess alignment `u_i = [cos(f_i, t̃_i) − mean_j cos(f_i, t_j)]₊`가 misclassified / corrupted sample에서 낮다. 즉 `a_i`가 실제로 신뢰도를 구별한다.

**CALM-T가 이미 알려준 것:**
- `A_kj` 행렬은 실제로 sparse하고 의미 있는 구조를 가짐 (cat-dog 0.9176, auto-truck 0.9165). 이 부분은 검증 완료.
- 그러나 K=10에서 text graph 구조를 `π`에 개입시키면 semantic vs random 차이가 0.0006pp에 불과. 신호가 너무 약함.
- CALM-AV의 `A_kj`는 CALM-T와 동일한 centered cosine 구조를 쓴다. 따라서 `A_kj` 자체의 재검증은 불필요.

**Phase 0의 질문은 단 하나다:**
> `m_k`와 `a_i`가 collapse 신호를 실제로 감지하는가?

## 2. Falsifiable Claims

| ID | Claim | 측정 방법 | Go threshold |
|----|-------|-----------|--------------|
| C1 | Collapse 진행 중 sink class에서 `s_k ↑`이지만 `m_k`는 `s_k`보다 늦게 오르거나 떨어진다 | step별 `(s_k, m_k)` trajectory, sink class vs non-sink | `corr(s_sink, m_sink) < corr(s_nonsink, m_nonsink)` 또는 `m_sink` 후반 감소 |
| C2 | `q_k` variance가 의미 있다 (모든 class에서 거의 1이 아니다) | step별 `std(q_k)` | `mean std(q_k) > 0.05` |
| C3 | `a_i`가 correct vs misclassified sample 간에 차이가 있다 | correct / misclassified 분리 후 `a_i` 분포 비교 | `mean(a_i | correct) > mean(a_i | misclassified)` by ≥ 0.05 |
| C4 | `C_collapse = max_k s_k(1 − q_k)`가 collapse와 함께 증가한다 | step별 trajectory | collapse step에서 단조 증가 또는 상관 ≥ 0.3 |

**C1 + C2 모두 통과 → Phase 1 (class gate) 진행**
**C3 통과 → Phase 2 (sample gate) 진행**
**C4는 참고용 (hard gate 아님)**

## 3. Method

### 3.1 계산할 quantities
```
# 기존 CALM (H2 C-variant) 그대로 실행하면서 아래를 로그

# --- Class gate diagnostics ---

# Soft prototype (harmonic weight)
μ_k = Σ_i w_ik * f̂_i / (Σ_i w_ik + ε),  f̂_i = f_i / ||f_i||

# Text neighbor structure (CALM-T와 동일한 A_kj 재사용)
c_kj = cos(t_k, t_j)
c̄_k  = mean_{j≠k} c_kj
A_kj = max(0, c_kj − c̄_k) * 1[j≠k]
A_kj = A_kj / (Σ_{ℓ≠k} max(0, c_kℓ − c̄_k) + ε)   # row-normalize

# Prototype-anchor margin
m_k = cos(μ_k, t_k) − Σ_{j≠k} A_kj * cos(μ_k, t_j)

# Class trust
q_k = exp(m_k) / (mean_ℓ exp(m_ℓ))   # detach, no gradient

# Collapse precursor
C_collapse = max_k [ s_k * (1 − q_k) ]

# --- Sample gate diagnostics ---

# Harmonic text mixture
t̃_i = Σ_k w_ik * t_k
t̃_i = t̃_i / (||t̃_i|| + ε)

# Excess alignment
u_i = max(0, cos(f_i, t̃_i) − (1/K) Σ_j cos(f_i, t_j))

# Mean-normalized sample trust
a_i = (u_i + ε) / (mean_j (u_j + ε))   # detach, no gradient
```

### 3.2 로깅 단위

- **step-level:** `s_k`, `m_k`, `q_k`, `std(q_k)`, `C_collapse`, `mean(a_i)`, `std(a_i)`, `cat%`, `mean_ent`
- **sample-level (마지막 10 step):** `(a_i, correct_i)` 쌍 → correct/misclassified 분리 분석용

### 3.3 실험 조건

| 항목 | 값 |
|---|---|
| Base method | CALM H2 C-variant (Inst 22 Phase 3) — 파라미터 변경 없음 |
| Loss | `L_ent + 2.0 · KL(p̄ ‖ π)` — 변경 없음 |
| α, β, λ | 0.1, 0.3, 2.0 — 변경 없음 |
| Dataset | CIFAR-10-C |
| Corruption | gaussian_noise sev=5 (primary), impulse_noise sev=5, glass_blur sev=5 (secondary) |
| N, batch, steps | 10,000 / 200 / 50 |
| Seed | 1 |
| 모델 변경 | 없음. μ_k, m_k, q_k, a_i는 전부 detached 계산만 |

gaussian_noise sev=5를 primary로 쓰는 이유: CALM-T와 동일 조건이므로 baseline trajectory가 이미 알려져 있어 비교가 용이하다.

## 5. 분석 계획

### 5.1 C1: s_k vs m_k dissociation (핵심)

step 1–50에 걸쳐 sink class (cat, class 3)와 non-sink class 대표 (ship, class 8)에 대해:

- `s_k` trajectory plot
- `m_k` trajectory plot
- `corr(s_k, m_k)` over steps: sink vs non-sink 비교

기대 패턴 (go):
```
s_cat: 0.13 → 0.3+ (상승)
m_cat: 정체 또는 하강
→ dissociation 확인
```

실패 패턴 (no-go):
```
m_cat이 s_cat과 같이 상승
→ q_k가 s_k를 그대로 따라가므로 class gate 의미 없음
```

### 5.2 C2: q_k variance

step별 `std(q_k)` 계산. 전 구간 평균이 0.05 미만이면 q_k가 사실상 uniform → class gate가 π에 영향을 못 줌.

CALM-T 교훈 대입: CALM-T에서 semantic vs random이 0.0006pp 차이였던 원인 중 하나가 "K=10에서 log-odds vector g가 Laplacian 방향에 anisotropic하게 로드되지 않음"이었다. q_k variance가 작으면 같은 문제가 CALM-AV에서도 반복된다.

### 5.3 C3: a_i discrimination

마지막 10 step의 sample-level 로그에서:
```python
correct_mask = (pred == label)
a_correct = a_i[correct_mask].mean()
a_wrong   = a_i[~correct_mask].mean()
gap = a_correct - a_wrong
```

gap ≥ 0.05 → sample gate가 실질적 신뢰도 구분 가능.

추가로 `a_i` 전체 분포의 histogram을 확인. bimodal이면 gate가 의미 있음. uniform에 가까우면 의미 없음.

### 5.4 C4: C_collapse trajectory (참고)
```
C_collapse = max_k s_k * (1 - q_k)
```

collapse가 진행되는 step 구간에서 이 값이 증가하면, "evidence는 크지만 trust는 낮다"는 신호가 실제로 발생하는 것. Phase 1에서 개입 지점으로 쓸 수 있음.

## 8. 실패 시 해석

### C1 실패 (m_k가 s_k와 함께 상승)

soft prototype μ_k가 이미 collapse된 feature를 반영하므로, μ_k 자체가 sink class 방향으로 오염되어 있을 가능성. 즉 prototype이 trust를 측정하는 게 아니라 collapse를 증폭하는 방향으로 반응하는 것.

→ hard argmax prototype을 써야 하는지, 또는 prototype-based trust 자체가 K=10에서 무의미한지 판단 필요.

### C2 실패 (std(q_k) < 0.05)

CALM-T의 "semantic = random at K=10" 문제가 다른 경로로 재현되는 것. text embedding near-collinearity가 margin m_k 자체를 죽임.

→ GPT-Pro의 핵심 우려 ("pairwise cosine=0.84이면 m_k가 너무 작아진다")가 현실화된 것. CALM-AV class gate 방향 자체를 재고해야 함.

### C3 실패 (a_i gap < 0.05)

excess alignment가 corrupted sample을 구별하지 못함. collapse 중 cat 방향 f_i는 오히려 t̃_i와의 alignment가 높게 나올 수 있음 (sink class 방향으로 쏠린 feature가 cat text와 잘 맞으므로).

→ sample gate는 포기. class gate만으로 Phase 1 설계.
