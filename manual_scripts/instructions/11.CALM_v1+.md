## Part 1: 현재 확정 방법론 (CALM v1 — Baseline)

### 1.1 손실 함수

```
L = L_ent - λ · H(p̄) - w_i2t · L_i2t
```

| 항 | 수식 | 역할 | 방향 |
|---|---|---|---|
| L_ent | -(1/B) Σ_i Σ_k q_ik log q_ik | 각 샘플 예측을 sharp하게 | minimize |
| -λ·H(p̄) | λ · Σ_k p̄_k log p̄_k, p̄_k = (1/B) Σ_i q_ik | 배치 주변 분포를 uniform으로 | maximize H(p̄) |
| -w_i2t·L_i2t | 이미지 프로토타입-텍스트 정렬 | CLIP 공간 유지 | maximize L_i2t |

### 1.2 확정된 하이퍼파라미터

```python
lambda_mi       = 2.0     # H(p̄) 가중치 (14/15 corruption에서 overall 기준 최적)
w_i2t           = 1.0     # I2T 정렬 강도
use_uniform_i2t = True    # I2T 프로토타입 계산 시 균등 가중치 (MAD 스케일 아님)
w_cov           = 0.0     # L_cov OFF (15/15 corruption에서 해로움 확정)
tau_inf         = 0.0     # 추론 시 조정 없음

# 모델/최적화
backbone        = "ViT-B-16"  # OpenAI CLIP, QuickGELU
trainable       = "LayerNorm only"  # ~65K params
optimizer       = "AdamW"  # lr=1e-3, beta1=0.9, wd=0.01
batch_size      = 200
grad_scaler     = "GradScaler(init_scale=1000)"
```

### 1.3 예측

```python
pred = argmax(raw_logits)  # 추론 시 추가 조정 없음
```

## Part 2: Adaptive λ 설계 (CALM v2)

### 2.1 동기

Fixed λ=2는 모든 corruption에서 동일한 강도로 collapse를 교정한다.
그러나 collapse severity는 corruption마다, 배치마다 다르다:
- gaussian_noise: cat sink 53% → collapse 심각 → 더 강한 교정 필요
- brightness: 거의 균등 → 약한 교정이면 충분

**핵심 아이디어:** collapse가 심할 때만 λ를 올리고, 이미 균등하면 λ를 낮춰서 불필요한 교정을 피한다.

### 2.2 Collapse Indicator

```python
# KL(p̄ || uniform) = log(K) - H(p̄)
# K=10일 때 log(K) = log(10) ≈ 2.3026
# 완전 균등: KL = 0
# 완전 collapse (한 클래스): KL = log(K)
collapse_score = torch.log(torch.tensor(K)) - H_pbar  # = KL(p̄ || uniform)
```

이 값은 매 배치 forward에서 이미 계산되는 H(p̄)에서 바로 나옴. 추가 연산 비용 없음.

### 2.3 Adaptive λ 후보 (3가지)

모든 후보는 **floor guarantee**를 만족: adaptive 부분이 0일 때 λ_t = λ_base = 2.0 (현재 best).

#### Variant A: Linear scaling (추가 하이퍼파라미터 0개)

```python
lambda_t = lambda_base * (1 + collapse_score / log(K))
```

- collapse_score = 0 (균등) → λ_t = λ_base = 2.0
- collapse_score = log(K) (완전 collapse) → λ_t = 2 * λ_base = 4.0
- **추가 하이퍼파라미터: 없음**
- 범위: [λ_base, 2·λ_base] = [2.0, 4.0]

#### Variant B: Scaled linear (추가 하이퍼파라미터 1개: α)

```python
lambda_t = lambda_base + alpha * collapse_score
```

- collapse_score = 0 → λ_t = λ_base = 2.0
- α로 adaptive 강도 조절
- **추가 하이퍼파라미터: α** (sweep: {1, 2, 3, 5})
- 범위: [λ_base, λ_base + α·log(K)]

#### Variant C: Sigmoid gating (추가 하이퍼파라미터 2개: λ_adapt, τ_collapse)

```python
gate = sigmoid((collapse_score - tau_collapse) * 10)
lambda_t = lambda_base + lambda_adapt * gate
```

- collapse_score < τ_collapse → gate ≈ 0 → λ_t ≈ λ_base (교정 안 함)
- collapse_score > τ_collapse → gate ≈ 1 → λ_t ≈ λ_base + λ_adapt
- **추가 하이퍼파라미터: λ_adapt, τ_collapse**
- 가장 유연하지만 가장 복잡

### 2.4 실험 우선순위

**Variant A를 먼저 테스트한다.** 이유:
1. 추가 하이퍼파라미터 0개 → 논문 스토리가 깔끔 ("parameter-free adaptive extension")
2. Floor guarantee 자동 만족
3. 안 되면 Variant B로 확장, 그래도 안 되면 Variant C

### 2.5 π_ref 확장 (Framework 차원, 실험은 후순위)

현재 KL(p̄ || uniform)을 쓰지만, 일반적으로 KL(p̄ || π_ref)로 확장 가능.
- π_ref = uniform → 현재 설정 (balanced dataset)
- π_ref = estimated target prior → non-balanced dataset 지원

**이번 실험에서는 π_ref = uniform 고정.** Non-uniform 확장은 논문의 "future work" 또는 "framework generality" 섹션에서 언급.

### 3.5 구현 가이드

#### 핵심 코드 변경 (pseudo-code)

기존 CALM v1 forward에서 λ 계산 부분만 변경:

```python
# === CALM v1 (fixed lambda) ===
# lambda_t = self.lambda_mi  # 항상 2.0

# === CALM v2 Variant A (adaptive, parameter-free) ===
K = logits.shape[1]  # 10
log_K = torch.log(torch.tensor(K, dtype=torch.float32, device=logits.device))
H_pbar = -(p_bar * torch.log(p_bar + 1e-8)).sum()
collapse_score = log_K - H_pbar  # KL(p̄ || uniform)
lambda_t = self.lambda_base * (1.0 + collapse_score / log_K)

# === CALM v2 Variant B (adaptive, with alpha) ===
# lambda_t = self.lambda_base + self.alpha * collapse_score

# 나머지는 동일:
loss = L_ent - lambda_t * H_pbar - self.w_i2t * L_i2t
```

**주의사항:**
1. `collapse_score`는 `H_pbar`에서 파생되므로 추가 forward 없음
2. `lambda_t`는 스칼라 — backward에서 H_pbar의 gradient에 collapse_score가 곱해짐
3. `lambda_t`를 detach할지 말지는 중요한 설계 선택:
   - **detach (권장):** `lambda_t = lambda_t.detach()` → λ_t는 상수 취급, H_pbar gradient만 scale 변경
   - **no detach:** λ_t 자체도 θ에 대해 미분됨 → 2차 효과 발생, 불안정 가능성

**→ Phase 1에서는 detach로 시작. 안정성 확인 후 no-detach 테스트는 optional.**

## 부록: 이론적 배경

### A.1 Fixed λ의 한계

KL(p̄ || π_ref) = log(K) - H(p̄) 일 때, 현재 loss의 H(p̄) 항은:

```
-λ · H(p̄) = λ · KL(p̄ || uniform) - λ · log(K)
```

Fixed λ는 KL이 크든 작든 동일한 가중치를 부여한다.
이는 다음 문제를 야기한다:
- **Collapse가 심할 때 (KL 큼):** λ=2가 부족할 수 있음 (gaussian_noise에서 λ=5가 better)
- **Collapse가 약할 때 (KL ≈ 0):** λ=2가 과도할 수 있음 (불필요한 uniform 압력)

### A.2 Adaptive λ의 효과

Variant A: λ_t = λ_base · (1 + KL/log(K))

이를 loss에 대입하면:
```
-λ_t · H(p̄) = -λ_base · (1 + KL/log(K)) · H(p̄)
             = -λ_base · H(p̄) - (λ_base/log(K)) · KL · H(p̄)
```

두 번째 항 `-(λ_base/log(K)) · KL · H(p̄)`는 KL이 클 때 H(p̄) 최대화를 더 강하게 한다.
이는 "collapse가 심할수록 더 강하게 교정"하는 효과.

### A.3 Floor Guarantee 증명

collapse_score = log(K) - H(p̄) ≥ 0 (항상, H(p̄) ≤ log(K)이므로)

따라서:
- Variant A: λ_t = λ_base · (1 + collapse_score/log(K)) ≥ λ_base · 1 = λ_base ✓
- Variant B: λ_t = λ_base + α · collapse_score ≥ λ_base (α ≥ 0일 때) ✓

λ_t ≥ λ_base가 항상 보장되므로, adaptive가 fixed보다 약한 교정을 하는 경우는 없다.

> **주의:** 이것이 "성능이 항상 ≥ baseline"을 보장하지는 않는다.
> λ가 더 크다고 항상 좋은 건 아님 (λ=10이 λ=2보다 나쁜 것처럼).
> Floor guarantee는 "교정 강도의 하한"이지 "성능의 하한"이 아님.
> 따라서 실험적 검증이 필수.
