### Run A: 현재 H2 (baseline)

```python
def compute_evidence_A(logits, R=5, alpha=0.1, beta=0.3):
    """현재 H2: top-R binary indicator + power transform"""
    B, K = logits.shape
    topR = logits.topk(R, dim=1).indices  # (B, R)
    e = torch.zeros(K, device=logits.device)
    for k in range(K):
        e[k] = (topR == k).any(dim=1).float().mean()
    pi = (e + alpha).pow(beta)
    pi = pi / pi.sum()
    return pi
```

HP: α=0.1, β=0.3, R=5, λ=2.0

### Run B: Harmonic Raw

```python
def compute_evidence_B(logits, alpha=0.1, beta=0.3):
    """Harmonic rank weight, no per-sample normalization.
    
    e_k = (1/B) Σ_i (1/rank_ik)
    
    rank_ik = sample i에서 class k의 순위 (1=best, K=worst)
    모든 순위가 기여하되 낮은 순위는 자연스럽게 적은 weight.
    R parameter 없음.
    """
    B, K = logits.shape
    
    # 각 sample에서 class별 순위 계산 (1-indexed, 1=best)
    # argsort(descending=True) → 가장 큰 logit이 index 0
    # 다시 argsort → 각 class의 순위
    order = logits.argsort(dim=1, descending=True)  # (B, K)
    ranks = torch.zeros_like(logits)
    for i in range(B):
        ranks[i, order[i]] = torch.arange(1, K+1, dtype=logits.dtype, device=logits.device)
    # ranks[i][k] = sample i에서 class k의 순위 (1=best)
    
    # 더 효율적인 방법:
    # ranks = logits.argsort(dim=1, descending=True).argsort(dim=1).float() + 1
    
    # Harmonic weight
    weights = 1.0 / ranks  # (B, K), 1등:1.0, 2등:0.5, ..., 10등:0.1
    
    # Evidence: 배치 평균
    e = weights.mean(dim=0)  # (K,)
    
    # Power transform (현재 H2와 동일)
    pi = (e + alpha).pow(beta)
    pi = pi / pi.sum()
    return pi
```

HP: α=0.1, β=0.3, λ=2.0 (R 없음)

### Run C: Harmonic Simplex

```python
def compute_evidence_C(logits, alpha=0.1, beta=0.3):
    """Harmonic rank weight with per-sample simplex normalization.
    
    w_ik = (1/rank_ik) / Σ_k (1/rank_ik)   → Σ_k w_ik = 1 per sample
    c_k = Σ_i w_ik                           → Σ_k c_k = B
    s_k = c_k / B                            → evidence distribution on simplex
    
    R parameter 없음. Σ c_k = B이므로 Dirichlet 해석과 연결 가능.
    """
    B, K = logits.shape
    
    # 순위 계산 (1-indexed)
    ranks = logits.argsort(dim=1, descending=True).argsort(dim=1).float() + 1  # (B, K)
    
    # Harmonic weight
    weights = 1.0 / ranks  # (B, K)
    
    # Per-sample simplex normalization
    weights = weights / weights.sum(dim=1, keepdim=True)  # (B, K), 각 row 합=1
    
    # Soft count
    c = weights.sum(dim=0)  # (K,), Σ c_k = B
    
    # Evidence distribution
    s = c / B  # (K,), simplex 위
    
    # Power transform
    pi = (s + alpha).pow(beta)
    pi = pi / pi.sum()
    return pi
```

HP: α=0.1, β=0.3, λ=2.0 (R 없음)

**주의:** s가 이미 simplex 위 (Σ=1)이므로, (s+α)^β 후 normalize하면 현재 H2의 (e+α)^β와 유사한 구조. α의 상대적 크기가 s의 scale에 맞는지 확인 필요. s_k ∈ [0.05, 0.3] 정도 범위일 것이므로 α=0.1은 적절.

### Run D1: Rank-power Unified (c=1.5, β=1)

```python
def compute_evidence_D(logits, c=1.5):
    """Rank-power unified: R과 β를 c 하나로 통합.
    
    w_r = r^{-c} / Σ_m m^{-c}
    e_k = (1/B) Σ_i w_{rank_ik}
    π = e / e.sum()   ← β=1 (tempering 없음, c가 대신)
    
    c가 크면 → top-heavy → 강한 shrinkage 효과 (β<1에 대응)
    c가 작면 → flat → 약한 shrinkage (β≈1에 대응)
    """
    B, K = logits.shape
    
    # 순위 계산
    ranks = logits.argsort(dim=1, descending=True).argsort(dim=1).float() + 1  # (B, K)
    
    # Rank-power weight
    weights = ranks.pow(-c)  # (B, K)
    
    # Normalize per sample (simplex)
    weights = weights / weights.sum(dim=1, keepdim=True)
    
    # Evidence
    e = weights.sum(dim=0) / B  # (K,), simplex
    
    # No tempering (β=1), just normalize
    pi = e / e.sum()
    
    # α smoothing을 넣으려면: pi = (e + alpha) / (e + alpha).sum()
    # 하지만 D는 HP 최소화가 목적이므로 α도 안 넣음
    return pi
```

HP: c=1.5, λ=2.0만 (α 없음, β 없음, R 없음)

## 7. 주의사항

1. **모든 corruption 시작 전 model reset.** 각 corruption은 독립. 이전 corruption의 adapted weight를 이어받지 않음.

2. **Phase 1의 rank 계산 효율성.** `ranks = logits.argsort(dim=1, descending=True).argsort(dim=1).float() + 1`이 가장 효율적. For loop 사용 시 느려질 수 있음.

3. **D1/D2에서 α smoothing 없음.** Evidence e가 0인 class가 있으면 π_k=0이 되어 KL에서 문제. 매우 작은 ε를 추가하거나:
```python
pi = e + 1e-8
pi = pi / pi.sum()
```

4. **Phase 2/3에서 offline evaluation.** 각 corruption의 adaptation이 끝난 후 같은 corruption의 전체 10K를 재평가. Dataloader를 두 번 순회해야 함 (1회: adaptation, 2회: offline eval).

5. **Phase 3 variant 결정은 자동화 가능:**
```python
# Phase 1 결과 로드
phase1 = load_json('phase1_summary.json')
h2_online = phase1['A']['online_acc']

# Best R-free variant 선정
candidates = ['C', 'B', 'D1', 'D2']  # 우선순위 순
best_variant = None
for name in candidates:
    if abs(phase1[name]['online_acc'] - h2_online) <= 0.01:  # ±1pp
        best_variant = name
        break

if best_variant is None:
    print("No R-free variant matches H2. Phase 3 = skip (use Phase 2 results).")
else:
    print(f"Phase 3 variant: {best_variant}")
```
