# Instruction 20: J3 Text LN 설정 비교 + 정밀 진단

## 1. 배경 및 목적

### 발견된 문제

J3 (L_rel only)의 구현을 분석한 결과, **text encoder의 LayerNorm도 함께 학습**되고 있었음 (BATCLIP baseline과 동일 설정). 하지만 J3의 L_rel에서 사용하는 text relational target (r_k)과 centered text (Delta_t)는 **adaptation 전 초기 text로 한 번만 계산하고 고정**되어 있음.

```
현재 J3의 동작:
  - r_k, Delta_t: 초기 text encoder에서 한 번 계산 → 고정 (anchor 역할)
  - model의 text LN: 매 step adaptation으로 변화
  - prediction: model(images) → adapted image encoder @ adapted text encoder
  
문제:
  - L_rel은 "초기 text 공간"에서 prototype을 정렬
  - Prediction은 "adapted text 공간"에서 계산
  - 두 공간이 step이 진행되면서 점점 달라짐 (drift)
  - L_rel이 열심히 최적화하는 방향과 실제 prediction이 이득 보는 방향이 다름
```

이 drift가 J3의 acc가 0.537에 머무는 주요 원인일 수 있음.

### 이번 실험의 목적

1. **Text LN 설정 3가지를 비교**하여 drift가 J3의 성능에 미치는 영향을 확인.
2. **각 설정에서 정밀 진단**을 실행하여 J3의 진짜 병목이 무엇인지 특정.
3. 진단 결과에 따라 다음 방향 (head 교체, loss 수정, H2 방향 전환 등)을 결정.

## 3. Part 1: Text LN 설정 3가지 비교

### Run X1: Text LN Freeze + r_k 고정 (옵션 A)

**개념:** Text encoder를 완전히 고정. Image encoder LN만 adaptation. r_k와 Delta_t는 초기 frozen text에서 계산하고 고정. Prediction도 frozen text 사용.

**모든 것이 같은 text 공간에서 작동 → drift 없음.**

```python
# === configure_model 수정 ===
# 기존 BATCLIP/J3: model 전체에서 LayerNorm을 찾아 requires_grad=True
# X1: image encoder의 LayerNorm만 requires_grad=True, text encoder는 전부 freeze

def configure_model_image_only(model):
    """Image encoder LN만 학습, text encoder 전체 freeze."""
    model.eval()
    
    # 먼저 전부 freeze
    for param in model.parameters():
        param.requires_grad = False
    
    # Image encoder (model.visual)의 LayerNorm만 unfreeze
    for name, module in model.visual.named_modules():
        if isinstance(module, nn.LayerNorm):
            module.requires_grad_(True)
            # train mode로 (BN 통계 업데이트는 없지만 LN은 상관없음)
            module.train()
    
    # 학습 파라미터 수집
    params = []
    for name, param in model.visual.named_parameters():
        if param.requires_grad:
            params.append(param)
    
    return model, params

# === text features 계산 (한 번만, frozen) ===
with torch.no_grad():
    text_features = model.encode_text(tokenized_prompts)  # (K=10, D=512)
    text_features = F.normalize(text_features, dim=1)

# === L_rel target 계산 (한 번만, frozen text 기준) ===
t_bar = text_features.mean(dim=0)
Delta_t = F.normalize(text_features - t_bar, dim=1)  # (K, D)
tau_t = 1.0
r = F.softmax(Delta_t @ Delta_t.T / tau_t, dim=1)    # (K, K)

# === Adaptation loop ===
for step, (images, labels) in enumerate(batches):
    image_features = model.encode_image(images)          # adapted image encoder
    image_features = F.normalize(image_features, dim=1)  # (B, D)
    
    # Prediction: frozen text 사용
    logits = image_features @ text_features.T * logit_scale  # (B, K)
    # 여기서 text_features는 위에서 계산한 frozen text
    # logit_scale은 model.logit_scale (frozen)
    
    preds = logits.argmax(dim=1)  # 이걸로 accuracy 계산
    
    # === L_rel 계산 ===
    q = F.softmax(logits, dim=1)  # (B, K)
    
    # Prototype
    m = (q.T @ image_features) / (q.sum(0).unsqueeze(1) + 1e-8)  # (K, D)
    m_bar = m.mean(dim=0)
    Delta_m = F.normalize(m - m_bar, dim=1)  # (K, D)
    
    tau_nce = 1.0
    p = F.softmax(Delta_m @ Delta_t.T / tau_nce, dim=1)  # (K, K)
    
    L_rel = 0
    for k in range(K):
        L_rel += F.kl_div(p[k].log(), r[k], reduction='sum')
    L_rel = L_rel / K
    
    loss = L_rel
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

**핵심 주의사항:**
- `model.encode_text()`를 adaptation loop 안에서 호출하지 않음 (frozen이니까 한 번만)
- `text_features`는 loop 밖에서 한 번 계산하고 `.detach()`로 사용
- Prediction logits도 이 frozen text_features로 계산
- `model.logit_scale`도 freeze 확인 (requires_grad=False)

### Run X2: Text LN 학습 유지 + r_k 매 step 재계산 (옵션 B)

**개념:** 기존과 동일하게 image + text LN 모두 학습. 하지만 r_k와 Delta_t를 **매 배치마다 현재 adapted text로 재계산.** 이러면 Loss 공간 = Prediction 공간 (항상 현재 text).

```python
# === configure_model은 기존 J3과 동일 ===
# model 전체에서 LayerNorm을 찾아 requires_grad=True (image + text 모두)
model, params = configure_model(model)  # 기존 함수 그대로

# === text prompts tokenize (한 번만) ===
tokenized_prompts = tokenize(["a photo of a " + c for c in class_names]).to(device)

# === Adaptation loop ===
for step, (images, labels) in enumerate(batches):
    # 현재 adapted text encoder로 text features 재계산
    text_features = model.encode_text(tokenized_prompts)  # (K, D) — 매 step 변화
    text_features = F.normalize(text_features, dim=1)
    
    # 현재 text로 Delta_t, r_k 재계산
    t_bar = text_features.mean(dim=0)
    Delta_t = F.normalize(text_features - t_bar, dim=1)
    tau_t = 1.0
    r = F.softmax(Delta_t @ Delta_t.T / tau_t, dim=1)  # (K, K) — 매 step 변화
    # 중요: r은 detach하지 않음. text LN을 통해 gradient가 흐름.
    # 또는 r.detach()로 target을 고정하고 싶으면 그렇게 해도 됨.
    # 일단 detach 없이 (gradient 허용) 실행.
    
    # Image features
    image_features = model.encode_image(images)
    image_features = F.normalize(image_features, dim=1)
    
    # Prediction logits: 현재 adapted text 사용
    logits = image_features @ text_features.T * model.logit_scale  # (B, K)
    preds = logits.argmax(dim=1)
    
    # === L_rel 계산 ===
    q = F.softmax(logits, dim=1)  # (B, K)
    
    m = (q.T @ image_features) / (q.sum(0).unsqueeze(1) + 1e-8)  # (K, D)
    m_bar = m.mean(dim=0)
    Delta_m = F.normalize(m - m_bar, dim=1)
    
    tau_nce = 1.0
    p = F.softmax(Delta_m @ Delta_t.T / tau_nce, dim=1)  # (K, K)
    
    L_rel = 0
    for k in range(K):
        L_rel += F.kl_div(p[k].log(), r[k], reduction='sum')
    L_rel = L_rel / K
    
    loss = L_rel
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

**핵심 주의사항:**
- `model.encode_text(tokenized_prompts)`를 **매 배치 loop 안에서** 호출
- text_features가 매 step 달라짐 → Delta_t, r_k도 달라짐
- Prediction logits도 현재 text_features로 계산 → drift 없음
- text encode는 K=10이라 비용 무시 가능 (~0.5ms)
- **r을 detach할지 말지:** 일단 detach 없이. Text LN에도 L_rel gradient가 흐르게.

### 진단 5: Feature space Fisher criterion (global + pairwise)

```python
# Class-conditional statistics in feature space
features = model.encode_image(all_images)  # (N, D)
features = F.normalize(features, dim=1)

centroids = []
intra_vars = []
class_features_dict = {}
for k in range(K):
    mask = (labels == k)
    class_features = features[mask]  # (n_k, D)
    centroid_k = class_features.mean(dim=0)  # (D,)
    centroids.append(centroid_k)
    class_features_dict[k] = class_features
    
    # Intra-class variance: 각 샘플과 centroid 간 거리의 평균
    intra_var = (class_features - centroid_k).pow(2).sum(dim=1).mean().item()
    intra_vars.append(intra_var)

centroids = torch.stack(centroids)  # (K, D)

# Inter-class distance: centroid 간 pairwise distance
inter_dists = []
for i in range(K):
    for j in range(i+1, K):
        dist = (centroids[i] - centroids[j]).pow(2).sum().item()
        inter_dists.append(dist)

mean_intra_var = sum(intra_vars) / K
mean_inter_dist = sum(inter_dists) / len(inter_dists)
fisher_ratio = mean_inter_dist / (mean_intra_var + 1e-8)

# 추가: Pairwise Fisher (GPT-Pro 권고)
# 특정 class 쌍에서의 separation 품질을 개별적으로 확인
pairwise_fisher = {}
for a in range(K):
    for b in range(a+1, K):
        inter = (centroids[a] - centroids[b]).pow(2).sum().item()
        intra = intra_vars[a] + intra_vars[b]
        F_ab = inter / (intra + 1e-8)
        pairwise_fisher[(a, b)] = F_ab
        if F_ab < 1.0:  # 약한 separation
            print(f"  Weak pair: {class_names[a]}-{class_names[b]}: F={F_ab:.3f}")
```

### 진단 6: Nearest-Centroid Acc (split-half, 핵심 분기점)

```python
# Split-half NC: data를 반으로 나눠서 한쪽으로 centroid, 다른 쪽으로 평가
# Same-data NC보다 낙관적이지 않은 추정치

features = model.encode_image(all_images)  # (N, D)
features = F.normalize(features, dim=1)

# Split indices
N = len(labels)
perm = torch.randperm(N)
half = N // 2
split_A = perm[:half]
split_B = perm[half:]

# Split A로 centroid 계산, Split B로 평가
centroids_A = []
for k in range(K):
    mask = (labels[split_A] == k)
    if mask.sum() > 0:
        centroids_A.append(features[split_A][mask].mean(dim=0))
    else:
        centroids_A.append(torch.zeros(features.shape[1], device=features.device))
centroids_A = torch.stack(centroids_A)
centroids_A = F.normalize(centroids_A, dim=1)

nc_logits_B = features[split_B] @ centroids_A.T
nc_preds_B = nc_logits_B.argmax(dim=1)
nc_acc_half1 = (nc_preds_B == labels[split_B]).float().mean().item()

# 반대: Split B로 centroid, Split A로 평가
centroids_B = []
for k in range(K):
    mask = (labels[split_B] == k)
    if mask.sum() > 0:
        centroids_B.append(features[split_B][mask].mean(dim=0))
    else:
        centroids_B.append(torch.zeros(features.shape[1], device=features.device))
centroids_B = torch.stack(centroids_B)
centroids_B = F.normalize(centroids_B, dim=1)

nc_logits_A = features[split_A] @ centroids_B.T
nc_preds_A = nc_logits_A.argmax(dim=1)
nc_acc_half2 = (nc_preds_A == labels[split_A]).float().mean().item()

nc_acc = (nc_acc_half1 + nc_acc_half2) / 2

# Same-data NC도 같이 보기 (상한)
centroids_all = []
for k in range(K):
    mask = (labels == k)
    centroids_all.append(features[mask].mean(dim=0))
centroids_all = torch.stack(centroids_all)
centroids_all = F.normalize(centroids_all, dim=1)
nc_logits_all = features @ centroids_all.T
nc_acc_same = (nc_logits_all.argmax(1) == labels).float().mean().item()

# 핵심 해석:
# nc_acc (split-half) >> offline_acc: text head가 bottleneck, feature는 좋음
# nc_acc ≈ offline_acc: feature 자체가 한계
```

### 진단 7: Text head vs Oracle centroid head 상세 비교

```python
# CLIP text head로 prediction
text_logits = features @ text_features.T  # (N, K)
# 여기서 text_features:
#   X1: frozen text features (adaptation 전)
#   X2: 최종 adapted text features
#   X3: 최종 adapted text features

    mask = (labels == k)

    nc_recall_k = (nc_preds[mask] == k).float().mean().item()
    print(f"Class {k} ({class_names[k]}): text_head={text_recall_k:.3f}, nc_head={nc_recall_k:.3f}, diff={nc_recall_k - text_recall_k:+.3f}")

# 핵심 해석:
# text_acc < nc_acc인 class들 → text embedding이 이 class들에서 약함
# 특히 text near-collinearity로 인해 혼동되는 class 쌍에서 nc가 나을 것
```

### 진단 8: Text drift 측정 (X3 전용, 추가)

```python
# X3에서만: 초기 text와 adapted text의 drift를 직접 측정
# adaptation 전 text features (이미 저장됨)
text_features_init = initial_text_features  # (K, D)

# adaptation 후 text features
with torch.no_grad():
    text_features_final = model.encode_text(tokenized_prompts)  # (K, D)
    text_features_final = F.normalize(text_features_final, dim=1)

# Cosine similarity between initial and final text
for k in range(K):
    cos_drift = F.cosine_similarity(text_features_init[k:k+1], text_features_final[k:k+1]).item()
    print(f"Class {k} ({class_names[k]}): text cos(init, final) = {cos_drift:.4f}")

# Text relational structure drift
Delta_t_init = F.normalize(text_features_init - text_features_init.mean(0), dim=1)
Delta_t_final = F.normalize(text_features_final - text_features_final.mean(0), dim=1)
r_init = F.softmax(Delta_t_init @ Delta_t_init.T / 1.0, dim=1)
r_final = F.softmax(Delta_t_final @ Delta_t_final.T / 1.0, dim=1)

# KL between initial and final relational structure
rel_drift = 0
for k in range(K):
    rel_drift += F.kl_div(r_final[k].log(), r_init[k], reduction='sum').item()
rel_drift /= K
print(f"Mean relational structure drift (KL): {rel_drift:.4f}")
```

### 진단 9: Deconvolution head (GPT-Pro 제안, 추가)

```python
# Text Gram matrix의 inverse로 cross-talk 제거
# J3의 "correlated codebook" 문제를 직접 해결하는 진단

# Text Gram matrix
# X1: frozen text, X2: final adapted text, X3: final adapted text
G = text_features @ text_features.T  # (K, K)

# Ridge regularized inverse
lambda_ridge = 0.1  # HP
G_inv = torch.linalg.inv(G + lambda_ridge * torch.eye(K, device=G.device))  # (K, K)

# Deconvolved logits
raw_logits = features @ text_features.T  # (N, K) — 기존 logits
deconv_logits = raw_logits @ G_inv  # (N, K) — deconvolved

# Deconvolution head prediction
deconv_preds = deconv_logits.argmax(dim=1)
deconv_acc = (deconv_preds == labels).float().mean().item()

# Top-3 restricted deconvolution (J3의 candidate set 활용)
# 각 sample의 top-3 candidate에서만 deconvolution
for i in range(len(labels)):
    S_i = raw_logits[i].topk(3).indices  # top-3 candidate
    G_sub = G[S_i][:, S_i]  # (3, 3) sub-matrix
    G_sub_inv = torch.linalg.inv(G_sub + lambda_ridge * torch.eye(3, device=G.device))
    z_sub = raw_logits[i, S_i]  # (3,)
    a_sub = G_sub_inv @ z_sub  # (3,) deconvolved coefficients
    deconv_preds_restricted[i] = S_i[a_sub.argmax()]

deconv_restricted_acc = (deconv_preds_restricted == labels).float().mean().item()

# 핵심 해석:
# deconv_acc >> offline_acc: text codebook cross-talk가 주요 병목 
# deconv_restricted_acc >> deconv_acc: top-3 restriction이 추가 도움
# deconv_acc ≈ offline_acc: cross-talk가 주요 문제가 아님

# 여러 lambda 시도
for lam in [0.01, 0.05, 0.1, 0.5, 1.0]:
    G_inv = torch.linalg.inv(G + lam * torch.eye(K, device=G.device))
    deconv_logits = raw_logits @ G_inv
    deconv_acc = (deconv_logits.argmax(1) == labels).float().mean().item()
    print(f"lambda={lam}: deconv_acc={deconv_acc:.4f}")
```

### 진단 10: Prototype purity (J3 특화)

```python
# J3의 prototype m_k가 실제로 어떤 true class의 mixture인지 측정.
# q가 soft하면 (entropy 0.98) prototype이 여러 class의 mixture일 수 있음.
# Purity가 낮으면 → J3이 class를 배우는 게 아니라 mixture를 정렬하는 것.

# 최종 모델로 전체 10K forward
logits = model(all_images)  # 각 run의 prediction 방식에 맞게
q = F.softmax(logits, dim=1)  # (N, K) soft assignment

# A_k,c = class c가 prototype k에 기여하는 비율
# = Σ_i q_ik * 1[y_i = c] / Σ_i q_ik
A = torch.zeros(K, K)  # (K, K): row=prototype, col=true class
for k in range(K):
    q_k = q[:, k]  # (N,) — 각 sample이 prototype k에 속하는 weight
    total_weight = q_k.sum() + 1e-8
    for c in range(K):
        mask_c = (labels == c).float()
        A[k, c] = (q_k * mask_c).sum() / total_weight

# Purity: 각 prototype에서 가장 큰 true class 비율
purity = A.max(dim=1).values  # (K,)
mean_purity = purity.mean().item()

# 출력
print(f"Mean prototype purity: {mean_purity:.3f}")
for k in range(K):
    dominant_class = A[k].argmax().item()
    print(f"  Proto {k} ({class_names[k]}): purity={purity[k]:.3f}, "
          f"dominant_true_class={class_names[dominant_class]} ({A[k, dominant_class]:.3f})")
print(f"\nFull A matrix (row=proto, col=true_class):")
print(A)

# 핵심 해석:
# mean_purity > 0.7: prototype이 대체로 한 class에 대응 → J3이 class를 배우고 있음
# mean_purity < 0.5: prototype이 mixture → J3의 training signal이 오염됨
# 특정 prototype의 purity가 매우 낮으면 → 그 class에서 confusion 예상
```

### 진단 11: Relational target identifiability

```python
# r_a와 r_b가 거의 같으면 → J3이 그 pair를 원리적으로 구분 못함.
# Text near-collinearity (rank=2.03) 때문에 일부 class 쌍의 r이 거의 동일할 수 있음.

# r_k는 adaptation 전에 계산된 text relational target (고정)
# 이미 위에서 r = softmax(Delta_t @ Delta_t.T / tau_t) 계산됨

# Pairwise Jensen-Shannon divergence
rel_identifiability = {}
for a in range(K):
    for b in range(a+1, K):
        m = 0.5 * (r[a] + r[b])
        js = 0.5 * F.kl_div(m.log(), r[a], reduction='sum').item() + \
             0.5 * F.kl_div(m.log(), r[b], reduction='sum').item()
        rel_identifiability[(a, b)] = js

# 가장 구분하기 어려운 pair들 (JS가 작은 순서)
sorted_pairs = sorted(rel_identifiability.items(), key=lambda x: x[1])
print("Least identifiable pairs (JS distance):")
for (a, b), js in sorted_pairs[:5]:
    print(f"  {class_names[a]}-{class_names[b]}: JS={js:.6f}")
print("Most identifiable pairs:")
for (a, b), js in sorted_pairs[-3:]:
    print(f"  {class_names[a]}-{class_names[b]}: JS={js:.6f}")

# 핵심 해석:
# JS ≈ 0인 pair → J3의 loss가 이 pair를 구분하라는 signal 자체가 없음
# 이 pair와 confusion matrix의 혼동 pair가 일치하면 → objective identifiability failure 확정
# JS가 큰데도 혼동이 크면 → feature separation failure (optimization 문제)
```

### 진단 12: Centroid-text alignment matrix

```python
# True class centroid와 text embedding의 alignment.
# J3이 image feature를 실제로 맞는 text 방향으로 보내고 있는지 확인.

# features: (N, D), labels: (N,)
# text_features: (K, D) — 해당 run의 text features

# Alignment matrix: S_c,k = cos(μ_c, t_k)

print("Centroid-Text Alignment Matrix S (row=true_class, col=text_class):")
print(S)

# 대각선 값 (이상적으로 가장 높아야 함)

print(f"Mean diagonal: {diag.mean().item():.4f}")

# 각 true class centroid에서 text의 ranking
for c in range(K):
    rank_of_correct = (S[c] >= S[c, c]).sum().item()
    print(f"  True {class_names[c]}: correct text rank={rank_of_correct}, "
          f"cos={S[c, c]:.4f}, best_text={class_names[S[c].argmax().item()]} ({S[c].max().item():.4f})")

# 핵심 해석:
# 대각선이 모두 1등 → class mean은 맞는 text에 잘 정렬 → sample variance가 문제
# 대각선이 1등이 아닌 class가 있으면 → 그 class에서 label anchoring failure

```

### Run H2D: H2 adaptation + 12진단

```python
# H2: L_ent + λ·KL(p̄ ∥ π_evid)
# β=0.3, α=0.1, R=5, λ=2.0
# 기존 comprehensive sweep의 axis08 H2 설정 그대로 실행

# H2의 loss (run_comprehensive_sweep.py에서 가져옴):
def compute_evidence_prior(logits, R=5, alpha=0.1, beta=0.3):
    """Evidence prior: top-R candidate support에서 추정"""
    B, K = logits.shape
    topR = logits.topk(R, dim=1).indices  # (B, R)
    e = torch.zeros(K, device=logits.device)
    for k in range(K):
        e[k] = (topR == k).any(dim=1).float().mean()
    pi = (e + alpha).pow(beta)
    pi = pi / pi.sum()
    return pi

# Adaptation loop:
for step, (images, _) in enumerate(batches):
    logits = model(images)
    probs = F.softmax(logits, dim=1)
    p_bar = probs.mean(dim=0)
    
    L_ent = -(probs * probs.log()).sum(dim=1).mean()
    
    pi_evid = compute_evidence_prior(logits, R=5, alpha=0.1, beta=0.3)
    L_kl = F.kl_div(p_bar.log(), pi_evid.detach(), reduction='sum')
    
    loss = L_ent + 2.0 * L_kl
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

# 50 steps 후 최종 모델로 12진단
# Prediction: model(images) → logits.argmax(1)
# text_features: H2도 text LN이 학습되므로, 최종 adapted text 사용
```

## Phase 3 상세: One-sided Excess Penalty 실험

### 목적

H2의 full KL(p̄ ∥ π_evid)는 **과대표현 class를 줄이는 것 + 과소표현 class를 올리는 것** 둘 다 함. 이 중 과대표현만 잡고 과소표현은 안 건드리면 skew에서 덜 해로울 수 있음.

이건 H2의 메커니즘을 "분포 전체를 맞춰라"에서 "과한 class만 깎아라"로 약화하는 것.

```
H2 (full KL):
  p̄_cat = 0.45, π_cat = 0.12 → cat 내려 (과대)
  p̄_dog = 0.03, π_dog = 0.11 → dog 올려 (과소) ← skew에서 해로울 수 있음

One-sided:
  p̄_cat = 0.45, π_cat = 0.12 → cat 내려 (과대) ✓
  p̄_dog = 0.03, π_dog = 0.11 → 무시 (과소지만 안 건드림) ✓
```

### Run OS1: Squared Excess Penalty

```python
# Loss: L_ent + λ · Σ_k [p̄_k - π_evid_k]²₊
# [x]₊ = max(0, x): 과대표현 (p̄ > π)일 때만 벌점

# π_evid는 H2와 동일하게 계산
# configure_model: image + text LN 모두 학습 (H2와 동일)

def one_sided_squared_excess(p_bar, pi_evid):
    """과대표현 class만 벌점. 과소표현은 무시."""
    excess = F.relu(p_bar - pi_evid)  # (K,) — 양수만 남김
    return excess.pow(2).sum()

# Adaptation loop:
for step, (images, _) in enumerate(batches):
    logits = model(images)
    probs = F.softmax(logits, dim=1)
    p_bar = probs.mean(dim=0)
    
    L_ent = -(probs * probs.log()).sum(dim=1).mean()
    
    pi_evid = compute_evidence_prior(logits, R=5, alpha=0.1, beta=0.3)
    L_excess = one_sided_squared_excess(p_bar, pi_evid.detach())
    
    loss = L_ent + 2.0 * L_excess  # λ=2.0, H2와 동일
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

# Prediction: model(images) → logits.argmax(1)
```

### Run OS2: One-sided KL (Positive Part Only)

```python
# Loss: L_ent + λ · Σ_k p̄_k · [log(p̄_k / π_evid_k)]₊
# [x]₊ = max(0, x): p̄_k > π_evid_k일 때만 KL 기여

def one_sided_kl(p_bar, pi_evid, eps=1e-8):
    """과대표현 class만 KL 벌점. 과소표현은 무시."""
    log_ratio = torch.log(p_bar / (pi_evid + eps) + eps)
    positive_part = F.relu(log_ratio)  # p̄ > π일 때만 양수
    return (p_bar * positive_part).sum()

# Adaptation loop:
for step, (images, _) in enumerate(batches):
    logits = model(images)
    probs = F.softmax(logits, dim=1)
    p_bar = probs.mean(dim=0)
    
    L_ent = -(probs * probs.log()).sum(dim=1).mean()
    
    pi_evid = compute_evidence_prior(logits, R=5, alpha=0.1, beta=0.3)
    L_os_kl = one_sided_kl(p_bar, pi_evid.detach())
    
    loss = L_ent + 2.0 * L_os_kl  # λ=2.0
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

# Prediction: model(images) → logits.argmax(1)
```

## 8. 판단 기준

### Part 1 (Text LN 설정):

```
X1 또는 X2 >> X3 (예: +3pp 이상):
  → Text LN drift가 J3의 성능을 해치고 있었음 확정
  → 더 나은 설정을 기반으로 후속 실험 진행

X1 ≈ X2 ≈ X3:
  → Drift가 주요 원인이 아님
  → J3의 한계는 다른 곳에 있음 → 진단이 더 중요
  
X1 > X2:
  → Text를 freeze하는 게 더 나음 → text를 anchor로 고정하는 설계가 맞음

X2 > X1:
  → Text도 같이 adapt하는 게 나음 → text adaptation이 도움됨
```

### Part 2 (진단):

```
진단 6 (split-half NC)이 가장 중요한 분기점:

nc_acc >> offline_acc (+5pp 이상):
  → Feature는 좋은데 text head가 bottleneck
  → 다음: deconvolution head, prototype head, MAP head 등 head 교체
  → J3 방향이 살아남: "J3 + better head = method"

nc_acc ≈ offline_acc (±2pp):
  → Feature 자체가 한계
  → 다음: backbone update가 필수 → L_ent급 sharpening이 필요
  → J3 단독 방향은 한계 → H2 방향 또는 L_ent safe version

진단 9 (deconvolution)이 두 번째 분기점:

deconv_acc >> offline_acc:
  → Text codebook cross-talk가 주요 병목
  → "CLIP의 near-collinear text embedding이 문제이고, 
     deconvolution으로 해결 가능"이라는 강한 논문 스토리
  → J3 + deconvolution head = method

deconv_acc ≈ offline_acc:
  → Cross-talk가 주요 문제가 아님
  → 다른 병목을 찾아야 함

진단 10 (prototype purity)이 J3 특화 분기점:

mean_purity > 0.7:
  → J3이 실제 class를 학습하고 있음. Prototype이 의미 있음.
  → J3의 한계는 prototype 품질이 아니라 다른 곳 (head, margin 등)

mean_purity < 0.5:
  → J3의 training signal이 q contamination으로 오염
  → Prototype이 mixture → L_rel gradient도 오염
  → J3의 한계가 objective 자체의 구조적 문제

진단 11 (relational identifiability) + 진단 3 (confusion matrix) 교차:

confusion이 집중된 pair에서 JS(r_a, r_b) ≈ 0:
  → J3 objective가 그 pair를 구분할 target 자체가 없음
  → Text near-collinearity의 직접적 영향
  → 해결: text geometry 변경 (multi-template, whitening 등)

confusion이 집중된 pair에서 JS(r_a, r_b)가 큼:
  → Target은 있는데 feature separation이 안 됨
  → 해결: 더 강한 adaptation 또는 다른 loss

진단 12 (centroid-text alignment):

대각선이 전부 1등:
  → Class mean은 맞는 text에 잘 정렬됨 → 문제는 sample-level variance
  
대각선이 1등이 아닌 class 존재:
  → 그 class에서 label anchoring failure
  → Text embedding이 그 class를 잘 표현 못함
```

## 9. 주의사항

1. **모든 run 시작 전 model reset.** 매 run은 원래 CLIP weight에서 시작.

2. **X1에서 image encoder LN만 unfreeze 확인.** Text encoder의 어떤 parameter도 requires_grad=True이면 안 됨. `model.visual`의 LayerNorm만 학습.

3. **X2에서 text forward를 loop 안에서 매 배치 호출.** `model.encode_text(tokenized_prompts)`를 매 step 실행. 결과를 detach하지 않음 (gradient가 text LN으로 흐르게).

4. **X3은 기존 J3과 완전히 동일해야 함.** Online acc가 0.537 근처로 재현되는지 확인. 크게 다르면 코드 오류.

5. **진단에서 label 사용은 진단용으로만.** Nearest-centroid, Fisher criterion 등에서 label을 쓰지만, 이건 분석용이지 method에 들어가지 않음.

6. **OOM 방지:** 진단에서 전체 10K를 한 번에 forward하지 말고, B=200씩 나눠서 feature를 모은 후 진단 계산.

7. **진단 9 (deconvolution)에서 G_inv 계산:** K=10이라 10×10 행렬 inverse. 비용 무시 가능. Lambda를 5개 시도하는 것도 즉시 완료.

8. **X2에서 r을 detach할지:** 일단 detach 없이 실행. "r이 gradient를 받아서 text가 relational structure를 유지하는 방향으로 adapt"될 수 있음. 만약 X2 결과가 이상하면 r.detach() 버전도 추가 시도.

9. **Phase 2의 BATCLIP:** 기존 코드의 BATCLIP 모드 그대로 사용. Loss = L_ent + w_i2t * L_i2t. configure_model은 image + text LN 모두 학습. 50 steps 후 12진단.

10. **Phase 2의 H2:** comprehensive sweep의 axis08 H2 설정 그대로. β=0.3, α=0.1, R=5, λ=2.0. 50 steps 후 12진단.

11. **Phase 3의 OS1/OS2:** H2와 동일한 설정에서 KL만 one-sided로 교체. π_evid 계산은 H2와 동일. λ=2.0 동일. 50 steps 후 12진단.

12. **Phase 2/3의 진단에서 text_features:** 
    - F0 (frozen): frozen text features 사용
    - BL (BATCLIP): 최종 adapted text features 사용
    - H2D (H2): 최종 adapted text features 사용
    - OS1/OS2: 최종 adapted text features 사용
