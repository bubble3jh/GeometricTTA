# MINT-TTA 실험 설계서 v1.0

**작성일:** 2026-03-03
**출발점:** SoftLogitTTA v2.1 Best Config (acc=0.666, λ_adj=5, w_pot=0, w_uni=0.5)
**목표:** 동일하거나 더 나은 성능을 달성하되, 방법론의 이론적 기반을 제1원리(MI, Bayesian Decision Theory, Barlow Twins)로 통합

## 0. 핵심 원칙

### 0.1 실험 전략: "Don't Break What Works"

기존 best config(0.666)은 이미 검증된 성취다. 이를 한꺼번에 해체하지 않고,
**한 번에 하나의 컴포넌트만 교체**하면서 각 교체의 효과를 격리 측정한다.
모든 Phase의 비교 기준은 기존 best config이다.

### 0.2 각 교체가 달성해야 할 것

| 교체 | 제거되는 heuristic | 대체하는 원리 |
|------|-------------------|-------------|
| Prior Correction → H(Y) | running_hist, β_hist, Δ_c, λ_adj, clip_M | Information Theory (MI maximization) |
| Prior Correction → Bayesian Inference Rule | (위와 동일) | Decision Theory (Bayes-optimal under label shift) |
| L_uni → L_Barlow | (구조 동일, 해석 변경) | Redundancy Reduction (Barlow Twins) |
| Soft weight에 Norm 추가 | (추가, 제거 아님) | Geometry (Double-Ellipsoid) |

### 0.3 성공/실패 판정 기준

- **성공:** 기존 대비 acc ≥ 0.660 (−0.6pp 이내) 이면서 HP 수 감소 또는 이론적 근거 확보
- **개선:** acc > 0.666
- **실패:** acc < 0.650 (−1.6pp 이상 하락)
- 실패 시 해당 교체를 철회하고 기존 컴포넌트를 복원한 뒤 다음 Phase로 진행

## 1. Phase 0: 사전 진단 (Geometry Diagnostic)

> **목적:** 후속 Phase에서 사용할 Norm 신호의 타당성을 실험 전에 입증한다.
> **모델 변경: 없음.** 진단만 수행.

### 1.1 실험 내용

TTA 없이(Zero-shot), 아래 데이터에 대해 `img_pre` (L2 정규화 전 피처)를 추출한다.

| 데이터 | 설명 |
|--------|------|
| Clean CIFAR-10 test set (10K) | 기준 분포 |
| CIFAR-10-C gaussian_noise sev=5 (10K) | 타겟 분포 |

**측정 항목:**

```
1. Norm 분포 비교
   n_i = ||img_pre[i]||₂
   → Clean vs Corrupted의 히스토그램, mean, std, skewness

2. Norm 이상치 vs OC-wrong 상관
   corrupted 데이터에서:
   - 각 샘플의 n_i 계산
   - MAD score: n̂_i = |n_i - median(n)| / (MAD(n) + ε)
   - n̂_i 상위 20% 그룹 vs 하위 80% 그룹의 OC-wrong 비율 비교
   - n̂_i와 correct/wrong의 AUC 측정 (H14의 s_max AUC=0.697과 비교)

3. Norm vs 기존 신호의 상관
   - corr(n_i, s_max_i): Norm과 최대 logit의 피어슨 상관
   - corr(n_i, margin_i): Norm과 top1-top2 margin의 상관
   → 상관이 높으면 redundant, 낮으면 complementary 신호
```

### 1.2 판정 기준

- Norm AUC > 0.55 이면서 기존 신호와 |corr| < 0.7 → Phase 5에서 Norm weight 도입
- Norm AUC ≤ 0.55 또는 |corr| ≥ 0.7 → Norm weight 도입 보류, 논문에서 "진단했으나 추가 이득 없음" 보고

```python

    logits = img_feat @ text_feat.T * logit_scale
```

## 2. Phase 1: Prior Correction을 H(Y)로 교체

> **목적:** 가장 큰 heuristic 덩어리(running_hist, β_hist, Δ_c, λ_adj, clip_M)를
> 단일 정보이론적 항 H(Y)로 대체할 수 있는지 검증한다.
> **변경:** Prior Correction 제거 → −λ_MI · H(p̄) 추가
> **유지:** L_ent, L_uni(w_uni=0.5), L_i2t(w_i2t=1.0), soft weight(기존 2-signal)

### 2.1 수학적 정의

**제거되는 항:**
```
running_hist ← β · running_hist + (1-β) · mean(q_raw)     [삭제]
Δ_c = clip(-log(running_hist[c] + ε), [-M, M])              [삭제]
adj_logits = raw_logits + λ_adj · Δ                          [삭제]
```

**추가되는 항:**
```
p̄_k = (1/B) Σ_i q[i,k]          ← 배치 내 marginal (uniform weight)
H(p̄) = -Σ_k p̄_k · log(p̄_k)    ← marginal entropy

L_total = L_ent + w_uni · L_uni - w_i2t · L_i2t - λ_MI · H(p̄)
```

**예측 방식:** `argmax(raw_logits)` — Prior Correction이 없으므로 raw logits 사용.

**핵심 차이 vs 기존:**
- 기존: forward-pass에서 logit을 직접 수정 → 같은 배치의 예측이 즉시 보정됨
- 변경: H(p̄)의 gradient가 backward를 통해 LayerNorm 업데이트 → **다음 배치부터** 효과

### 2.2 실험 조건

| 조건명 | λ_MI | 나머지 | 비고 |
|--------|------|--------|------|
| `baseline` | — | 기존 best config 그대로 | 재현 확인 (0.666 예상) |
| `hY_05` | 0.5 | L_uni=0.5, L_i2t=1.0 | |
| `hY_10` | 1.0 | 동일 | |
| `hY_20` | 2.0 | 동일 | |
| `hY_50` | 5.0 | 동일 | |
| `hY_100` | 10.0 | 동일 | λ_adj=5와 비슷한 강도 탐색 |
| `no_ent_hY_50` | 5.0 | L_ent 제거 | H(Y|X)와 H(Y)가 상쇄되는지 확인 |

### 2.3 측정 지표

```
1. acc (최종 정확도) — 기존 0.666과 비교
2. cat_sink_ratio — step별 class 3 예측 비율 (기존 best에서 ~5%)
3. cat_sink_step_profile — step 0~5에서의 cat 비율 추이
   → H(Y)가 초반에 충분히 빠른지 확인하는 핵심 지표
4. H(Y) 값 추이 — step별 marginal entropy (이상적: log(K)=2.30에 수렴)
5. d_eff — step별 유효 차원 (기존 패턴과 비교)
```

### 2.4 예상 시나리오 및 대응

**시나리오 A: acc ≥ 0.660**
→ 성공. Prior Correction 폐기 확정. Phase 2로 진행.

**시나리오 B: acc 0.620~0.659 (BATCLIP 수준이지만 best 미달)**
→ H(Y)가 방향은 맞지만 강도가 부족. 두 가지 보완책 중 택일:
- (B-1) Inference-time adjustment 추가 (Phase 3으로 즉시 이동)
- (B-2) Confidence-weighted marginal p̄_w 도입 (Phase 2로 이동)

**시나리오 C: acc < 0.620**
→ H(Y) gradient 단독으로는 cat sink 억제 불가.
Phase 1을 "H(Y)의 한계를 보인 진단 실험"으로 기록하고,
기존 Prior Correction을 **Bayesian 재해석만 하여** 유지 (논문에서 framing 변경).
→ Phase 3 (Inference Rule)로 이동.

## 3. Phase 2: Confidence-Weighted Marginal 도입

> **전제조건:** Phase 1에서 시나리오 A 또는 B 달성.
> **목적:** p̄를 confidence-weighted p̄_w로 교체하여 노이즈 샘플의 marginal 오염을 줄인다.
> **변경:** p̄ → p̄_w
> **유지:** Phase 1의 최적 λ_MI, L_uni, L_i2t

### 3.1 수학적 정의

```
p̄_w,k = Σ_i w_i · q[i,k] / Σ_i w_i

여기서 w_i = sigmoid(α · ŝ_i) × sigmoid(α · m̂_i)  [기존 2-signal, detach]

H(p̄_w) = -Σ_k p̄_w,k · log(p̄_w,k)
```

**w_i는 반드시 .detach()해야 한다.**
- w_i가 gradient graph에 참여하면 모델이 "높은 w_i를 만들어내는" 방향으로 학습 가능
- 이는 s_max를 인위적으로 키우는 reward hacking
- 기존 SoftLogitTTA에서 검증된 원칙 유지

### 3.2 실험 조건

Phase 1 최적 λ_MI를 λ*로 표기.

| 조건명 | Marginal | λ_MI | 비고 |
|--------|----------|------|------|
| `ph1_best` | uniform p̄ | λ* | Phase 1 최적 조건 재현 |
| `cw_marginal` | weighted p̄_w | λ* | 핵심 비교 대상 |
| `cw_marginal_lmi_high` | weighted p̄_w | λ* × 2 | weighted 시 강도 재조정 필요 가능 |

### 3.3 판정 기준

- `cw_marginal` ≥ `ph1_best` → p̄_w 채택
- `cw_marginal` < `ph1_best` → p̄ 유지 (cat sink 샘플이 이미 낮은 q를 가져서 marginal에 큰 영향 없을 수 있음)

## 4. Phase 3: Inference-Time Bayesian Decision Rule

> **목적:** 학습과 추론을 분리. 학습은 MI gradient로, 추론은 Bayes-optimal rule로.
> **이론적 근거:** Menon et al. (2021) — logit adjustment는 label shift 하에서 Bayes-optimal.
> **변경:** 예측 시 argmax(logits) → argmax(logits − τ · log(p̄_w))
> **유지:** 학습 loss는 Phase 1~2의 최적 그대로

### 4.1 수학적 정의

```
학습: L_total = L_ent - λ_MI · H(p̄ 또는 p̄_w) + w_uni · L_uni - w_i2t · L_i2t
      → backward → LayerNorm 업데이트

추론: ŷ = argmax_k [ logit_k - τ · log(p̄_w,k) ]

해석:
  logit_k ∝ log P(x|y=k)   (CLIP의 likelihood)
  -τ·log(p̄_w,k) = τ·log(1/p̄_w,k) ∝ log(1/P̂(y=k))  (prior 보정)
  
  따라서 ŷ = argmax log P(x|y=k) + τ·log(1/P̂(y=k))
            ∝ argmax P(y=k|x) under estimated prior P̂(y)
  
  τ = 1: exact Bayes
  τ > 1: aggressive correction (기존 λ_adj=5와 유사)
```

### 4.2 실험 조건

Phase 1~2 최적 학습 설정을 고정하고 τ만 sweep.

| 조건명 | τ | p̄ 종류 | 비고 |
|--------|---|--------|------|
| `no_inf_adj` | 0 | — | 학습만, 추론 보정 없음 (Phase 1~2 결과) |
| `inf_tau_1` | 1.0 | p̄_w | Exact Bayes |
| `inf_tau_2` | 2.0 | p̄_w | |
| `inf_tau_5` | 5.0 | p̄_w | 기존 λ_adj=5와 비슷한 강도 |
| `inf_tau_10` | 10.0 | p̄_w | |
| `inf_raw_marginal` | 5.0 | p̄ (uniform weight) | weighted vs raw 비교 |

### 4.3 핵심 관찰

```
Q1: τ_optimal이 기존 λ_adj=5와 비슷한 값인가?
    → 비슷하다면 "기존 heuristic이 사실 Bayesian correction의 근사였음" 확인

Q2: Phase 1~2에서 H(Y) gradient + Inference adjustment의 조합이
    기존 Prior Correction 단독(0.666)을 능가하는가?
    → 학습에서 LayerNorm이 이미 cat bias를 줄이고,
      추론에서 잔여 bias를 추가 보정 → 상승 효과 기대

Q3: cat_sink_step_profile에서 step 0~2의 cat 비율이
    inference adjustment 덕에 즉시 감소하는가?
    → 기존 Prior Correction의 "같은 배치 즉시 보정" 효과 복원
```

### 4.4 이 Phase가 "최후의 보루"가 아닌 이유

기존 설계서에서는 Phase 4를 "만약 초반이 느리면"의 fallback으로 놓았지만,
이론적으로 **학습과 추론의 분리는 Bayesian framework의 본질적 구조**다.

- 학습(파라미터 적응): MI maximization → 모델의 representation을 개선
- 추론(예측 생성): MAP estimation under estimated prior → 최적 decision

이 둘을 하나의 heuristic(adj_logits)으로 뭉쳤던 기존 접근이 오히려 비표준적.
따라서 이 Phase는 **방법론의 핵심 설계**이며, Phase 1~2의 결과와 무관하게 테스트한다.

## 5. Phase 4: L_uni의 이론적 강화 (Logit-Barlow)

> **목적:** 기존 L_uni(off-diagonal penalty)에 Barlow Twins/VICReg의 이론적 근거를 부여하고,
> variance collapse 방어를 추가할 수 있는지 검증한다.
> **변경:** L_uni → L_Barlow
> **유지:** Phase 1~3의 최적 설정

### 5.1 수학적 정의

기존 L_uni와의 차이를 명확히 하기 위해 두 가지 변형을 테스트한다.

**변형 A: 기존 L_uni (재해석만, 수식 동일)**
```
L̂ = (logits - μ) / σ     ← 클래스별 표준화
R = (1/B) · L̂.T @ L̂      ← 배치 상관행렬

L_uni = Σ_{i≠j} R[i,j]²  ← off-diagonal penalty

이론적 재해석: Barlow Twins의 covariance term과 동치.
"로짓 차원 간 redundancy 제거"로 프레이밍.
수식 변경 없이 논문의 이론적 서사만 강화.
```

**변형 B: Variance Hinge 추가 (VICReg 스타일)**
```
# 표준화 전 logits로 variance 계산
σ_k = std_batch(raw_logits[:, k])

L_var = Σ_k max(0, γ_var - σ_k)    ← 각 클래스 logit의 분산이 γ_var 이상 유지

# 표준화 후 logits로 off-diagonal 계산 (기존과 동일)
L_cov = Σ_{i≠j} R[i,j]²

L_Barlow = λ_var · L_var + λ_cov · L_cov
```

**왜 표준화 전/후를 분리하는가:**
- 표준화 후 R의 대각선은 정의상 ≈1이므로 (R_kk−1)² 항은 gradient가 거의 없다.
- 대신 표준화 전 logit의 std를 직접 hinge로 제어하면 실질적 variance collapse 방어가 된다.
- γ_var: clean CIFAR-10의 logit std를 참고하여 설정 (예: 0.5~2.0 범위 탐색).

### 5.2 실험 조건

Phase 1~3 최적 설정을 기반으로.

| 조건명 | L_uni/Barlow | λ_cov | λ_var | γ_var | 비고 |
|--------|-------------|-------|-------|-------|------|
| `ph3_best` | 기존 L_uni | w_uni=0.5 | — | — | Phase 3 최적 재현 |
| `barlow_reframe` | 변형 A | 0.5 | — | — | 수식 동일, 해석만 변경 (성능 동일 확인) |
| `barlow_var_05` | 변형 B | 0.5 | 0.1 | 0.5 | |
| `barlow_var_10` | 변형 B | 0.5 | 0.1 | 1.0 | |
| `barlow_var_20` | 변형 B | 0.5 | 0.1 | 2.0 | |
| `barlow_cov_sweep` | 변형 B | [0.1, 0.5, 1.0] | 0.1 | (최적 γ) | |
| `no_i2t` | 변형 B (최적) | (최적) | (최적) | (최적) | L_i2t=0: I2T가 필요한지 검증 |

### 5.3 측정 지표

```
1. acc — 기존 대비 변화
2. d_eff — step별 유효 차원
   → 변형 B가 변형 A보다 d_eff를 더 빠르게 회복하는가?
3. logit_std_per_class — step별 각 클래스 logit의 std
   → 변형 B에서 variance collapse가 실제로 방지되는가?
4. off_diag_norm — R의 off-diagonal Frobenius norm
   → 두 변형 모두에서 감소하는지 확인
```

### 5.4 판정 기준

- `barlow_var_*` 중 최적이 `ph3_best` 대비 acc ≥ 0 이면서 d_eff 개선 → 변형 B 채택
- d_eff 개선 없거나 acc 하락 → 변형 A 채택 (수식 동일, 해석만 Barlow로)
- `no_i2t`가 `barlow_var_*` 최적 대비 acc 하락 < 1pp → I2T 제거 고려 (방법론 단순화)

## 6. Phase 5: Norm-Based Evidence Weight (조건부)

> **전제조건:** Phase 0에서 Norm AUC > 0.55 & |corr with s_max| < 0.7
> **목적:** Double-Ellipsoid 기하학의 Norm 정보를 soft weight에 반영한다.
> **변경:** w_i에 세 번째 신호 추가
> **유지:** Phase 1~4의 최적 설정

### 6.1 수학적 정의

```
n_i = ||img_pre[i]||₂
n̂_i = |n_i - median(n)| / (MAD(n) + ε)    ← 양방향 이상치 점수

w_i = sigmoid(α · ŝ_i) × sigmoid(α · m̂_i) × sigmoid(-γ_n · n̂_i)

γ_n: Norm 패널티 강도 (새 HP)
```

**주의: w_i 전체를 .detach()하는 원칙은 변함없다.**

### 6.2 실험 조건

| 조건명 | γ_n | 비고 |
|--------|-----|------|
| `ph4_best` | — | Phase 4 최적 (Norm 없음) |
| `norm_05` | 0.5 | 약한 패널티 |
| `norm_10` | 1.0 | |
| `norm_20` | 2.0 | 강한 패널티 |

### 6.3 판정 기준

- 최적 `norm_*`이 `ph4_best` 대비 acc 개선 → 채택
- 개선 없음 → 논문에서 "진단 결과 보고하되 최종 방법론에는 미포함" (음성 결과도 가치 있음)

## 7. Phase 6: 최종 통합 및 Ablation

> **목적:** 각 Phase에서 채택된 컴포넌트를 결합한 최종 방법론을 확정하고,
> 논문용 ablation table을 생성한다.

### 7.1 최종 Objective (예상 형태)

```
L_total = (1/B) Σ_i w_i · H(q_i)        ← Conditional Entropy (Min)
        - λ_MI · H(p̄_w)                  ← Marginal Entropy (Max)
        + λ_cov · Σ_{i≠j} R[i,j]²        ← Logit Decorrelation (Barlow)
        + λ_var · Σ_k max(0, γ-σ_k)      ← Variance Floor (VICReg, 조건부)
        - w_i2t · L_i2t                   ← I2T Alignment (조건부)

Inference: ŷ = argmax_k [ logit_k - τ · log(p̄_w,k) ]

w_i = sigmoid(α·ŝ_i) × sigmoid(α·m̂_i) [× sigmoid(-γ_n·n̂_i)]  ← detach
```

각 `[조건부]` 항은 해당 Phase에서 채택/기각된 결과에 따라 포함 여부 결정.

### 7.2 Ablation Table 구성

논문 Table 1에 들어갈 ablation. 최종 config에서 하나씩 빼는 실험.

| 조건 | 변경 내용 | 검증 대상 |
|------|----------|----------|
| Full MINT-TTA | 전체 최적 | — |
| − H(p̄) | λ_MI=0 | Marginal entropy의 기여 |
| − Inference adj | τ=0 | Bayesian decision rule의 기여 |
| − L_Barlow | λ_cov=0, λ_var=0 | Logit decorrelation의 기여 |

| − I2T | w_i2t=0 | Text alignment의 기여 (Phase 4에서 이미 확인) |
| − Norm weight | γ_n=0 | Geometry 신호의 기여 (Phase 5에서 이미 확인) |
| BATCLIP | 원본 베이스라인 | 전체 개선폭 |
| SoftLogitTTA v2.1 | 기존 best | heuristic → principled 전환의 효과 |
