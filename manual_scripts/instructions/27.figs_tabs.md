## 실험 1: 15-Corruption Cone Compression Table

### 목적
논문 main analysis table. Corruption type별 geometric 변형을 정량화.

### 방법
Inst 26 Block A (clean geometry)는 이미 있음. Block B의 B5 (cone deformation)를 15개 corruption 전체로 확장.

**모델 실행 없음 — frozen CLIP으로 feature 추출만.**

### 계산할 양 (corruption별)

```python
"""
각 corruption에 대해 frozen CLIP으로 N=10000 corrupted images를 인코딩.
Clean features (F_clean)은 Inst 26 Block A에서 이미 계산됨 — 재사용.
Clean features가 저장되어 있지 않으면 다시 계산.
"""

for corr_name in CORRUPTIONS:
    # 1. Corrupted features 추출
    F_corr = encode_all_images(corr_name, severity=5)  # (N, D), L2 normalized
    
    # 2. Effective rank
    F_corr_centered = F_corr - F_corr.mean(dim=0)
    S = torch.linalg.svdvals(F_corr_centered.float())  # float32로 cast
    p = S / S.sum()
    eff_rank = torch.exp(-(p * p.log()).sum()).item()
    
    # 3. Cone mean pairwise cosine (subsample 1000)
    idx = torch.randperm(N)[:1000]
    pw = F_corr[idx] @ F_corr[idx].T
    triu = torch.triu(torch.ones(1000, 1000, dtype=torch.bool), diagonal=1)
    cone_mean_cos = pw[triu].mean().item()
    
    # 4. Cone shift
    cone_shift = F.cosine_similarity(
        F_clean.mean(dim=0).unsqueeze(0),
        F_corr.mean(dim=0).unsqueeze(0)
    ).item()
    
    # 5. Sink class (가장 많이 confused되는 class)
    # Frozen zero-shot prediction
    logits = F_corr @ T.T  # (N, K), T = text features
    preds = logits.argmax(dim=1)  # (N,)
    
    # 전체 prediction에서 가장 많이 나오는 class
    pred_counts = torch.bincount(preds, minlength=K)
    sink_class_idx = pred_counts.argmax().item()
    sink_class_name = CIFAR10_CLASSES[sink_class_idx]
    sink_class_pct = pred_counts[sink_class_idx].item() / N
    
    # 6. Frozen zero-shot accuracy
    correct = (preds == labels).float().mean().item()
    
    # 7. Top-5 SV ratio
    sv_ratio_top5 = (S[:5].sum() / S.sum()).item()
    
    # 8. Per-class "most confused" — 각 true class에서 가장 많이 예측되는 wrong class
    confusion = {}
    for k in range(K):
        mask_k = (labels == k)
        preds_k = preds[mask_k]
        wrong_mask = (preds_k != k)
        if wrong_mask.sum() > 0:
            wrong_preds = preds_k[wrong_mask]
            wrong_counts = torch.bincount(wrong_preds, minlength=K)
            wrong_counts[k] = 0  # 자기 자신 제외
            most_confused_idx = wrong_counts.argmax().item()
            confusion[CIFAR10_CLASSES[k]] = CIFAR10_CLASSES[most_confused_idx]
        else:
            confusion[CIFAR10_CLASSES[k]] = "none"
```

## 실험 3: Evidence π vs Uniform π — 15-Corruption Ablation

### 목적
"Evidence-based prior가 uniform prior보다 왜 나은가"의 실험적 근거. 이론의 핵심 prediction을 검증.

### 방법
15개 corruption에서 두 가지 variant를 비교:
1. **H2 (evidence π):** 현재 best method. π_k ∝ (e_k + α)^β
2. **H2-uniform (uniform π):** evidence 대신 π_k = 1/K. 나머지 전부 동일.

### 구현

```python
"""
H2-uniform은 H2 코드에서 π 계산만 바꾸면 됨.
나머지 (L_ent, λ, optimizer, adapted params) 전부 동일.
"""

def compute_pi_evidence(logits, alpha=0.1, beta=0.3):
    """현재 H2 C-variant의 evidence prior"""
    B, K = logits.shape
    # Harmonic simplex weights
    ranks = logits.argsort(dim=1, descending=True).argsort(dim=1).float() + 1
    weights = 1.0 / ranks  # (B, K)
    weights = weights / weights.sum(dim=1, keepdim=True)  # per-sample normalize
    e = weights.mean(dim=0)  # (K,)
    pi = (e + alpha).pow(beta)
    pi = pi / pi.sum()
    return pi.detach()

def compute_pi_uniform(logits):
    """Uniform prior (standard IM)"""
    K = logits.shape[1]
    return torch.ones(K, device=logits.device) / K

# 각 corruption에서:
# Run A: loss = L_ent + 2.0 * KL(p_bar, pi_evidence)
# Run B: loss = L_ent + 2.0 * KL(p_bar, pi_uniform)
#
# 동일: L_ent 계산, λ=2.0, optimizer, adapted params, steps, batch size, seed
```

## 실험 4: Confusion Matrix Evolution 시각화

### 목적
Adaptation 과정에서 sink class가 해소되는 것을 시각화.

### 방법
H2 adaptation 중 step 0 (frozen corrupted), step 25 (중간), step 50 (최종)에서 confusion matrix 추출.

**중요: 실험 2에서 H2 adaptation을 이미 돌림. 동일 run에서 중간 step의 prediction도 저장하면 실험 2와 4를 한 번에 처리 가능.**

### 구현

```python
"""
H2 adaptation loop에서 특정 step에 전체 데이터에 대한 prediction을 저장.
step 0은 adaptation 전 (frozen corrupted model).

메모리 주의: 전체 N=10000에 대한 forward pass를 중간에 삽입.
            eval mode로 전환 → forward → train mode 복귀.
            또는 매 step의 batch prediction만 모아서 근사 (less accurate).
            
추천: step 0, 25, 50에서만 전체 데이터 forward pass (3번).
     OOM 우려 시 B=200 단위로 나눠서.
"""

SNAPSHOT_STEPS = [0, 25, 50]

# Gaussian noise (primary)
# 추가로: impulse_noise, glass_blur (다른 sink class 확인)

for corr_name in ['gaussian_noise', 'impulse_noise', 'glass_blur']:
    
    # H2 adaptation loop
    for step in range(50):
        # ... (기존 H2 training step)
        
        if step in SNAPSHOT_STEPS or step == 0:
            model.eval()
            all_preds = []
            all_labels_list = []
            with torch.no_grad():
                for batch_imgs, batch_lbls in full_dataloader:
                    logits = model(batch_imgs)  # (B, K)
                    preds = logits.argmax(dim=1)
                    all_preds.append(preds.cpu())
                    all_labels_list.append(batch_lbls.cpu())
            all_preds = torch.cat(all_preds)
            all_labels_cat = torch.cat(all_labels_list)
            
            # Confusion matrix
            cm = torch.zeros(K, K, dtype=torch.long)
            for true_k in range(K):
                for pred_k in range(K):
                    cm[true_k, pred_k] = ((all_labels_cat == true_k) & (all_preds == pred_k)).sum()
            
            # 저장
            torch.save(cm, f'cm_{corr_name}_step{step}.pt')
            
            model.train()
    
    # Step 0 = adaptation 전이므로, loop 진입 전에 한 번 해야 함
```

## 실험 6: I_batch 측정 (Theorem 3 → Corollary 3.1 검증)

### 목적
Theorem 3의 핵심 가정 (A1: 모든 sample의 prediction이 동일)이 현실에서 얼마나 성립하는지 정량화.
Corollary 3.1의 ε_cc를 empirically measure.

### 이론적 배경

```
I_batch = H(p̄) - (1/B) Σ_i H(p_i)

이것은 batch 내 prediction의 "다양성"을 측정.
- I_batch = 0: 모든 sample의 prediction이 완전히 동일 (Theorem 3의 가정 A1)
- I_batch > 0: sample 간 prediction이 다름

Corollary 3.1에서: I_batch ≤ ε_cc이면
  ||p̄* - p†||₁ ≤ √(2·ε_cc / (λ-1))

즉 ε_cc가 작을수록 Theorem 3의 이론적 예측(p†)과 실제(p̄*)가 가까움.
```

### 방법

**기존 H2 C-variant adaptation run에서 매 step I_batch를 계산.**
실험 2+4에서 이미 H2를 돌리므로, 거기에 I_batch 로깅만 추가하면 됨.
별도 run 불필요.

```python
"""
H2 adaptation loop 내에서 매 step (또는 매 5 step):
"""

with torch.no_grad():
    # 이미 계산된 probs: (B, K) softmax predictions
    
    # H(p̄)
    p_bar = probs.mean(dim=0)  # (K,)
    H_pbar = -(p_bar * p_bar.log()).sum().item()
    
    # (1/B) Σ H(p_i)
    H_per_sample = -(probs * probs.log()).sum(dim=1)  # (B,)
    mean_H_pi = H_per_sample.mean().item()
    
    # I_batch
    I_batch = H_pbar - mean_H_pi  # ≥ 0 by Jensen's inequality
    
    # Corollary 3.1의 bound
    # ||p̄* - p†||₁ ≤ √(2·I_batch / (λ-1))
    lambda_val = 2.0
    corollary_bound = (2 * I_batch / (lambda_val - 1)) ** 0.5
    
    # 로그
    step_log.append({
        'step': step,
        'I_batch': I_batch,
        'H_pbar': H_pbar,
        'mean_H_pi': mean_H_pi,
        'corollary_bound_L1': corollary_bound,
    })
```

## 실험 7: λ Phase Transition (Theorem 3의 핵심 prediction 검증)

### 목적
Theorem 3의 가장 강력한 prediction을 직접 검증:
**"λ > 1이면 collapse 방지, λ ≤ 1이면 collapse."**

### 이론적 배경

```
Theorem 3 (cone-compressed simplex에서):
  λ < 1  →  loss가 concave → minimum이 vertex (= collapse)
  λ = 1  →  loss가 linear → minimum이 vertex (= collapse)
  λ > 1  →  loss가 strictly convex → unique interior minimum (= non-collapse)

따라서 λ = 1이 정확한 phase transition point.

λ > 1일 때 equilibrium:
  p†_k = π_k^α / Σ_j π_j^α,   α = λ/(λ-1)
```

### 방법

7개 λ 값에서 H2 C-variant를 gaussian_noise sev=5로 실행.


### 이론적 예측 계산 (비교용)

```python
def theoretical_equilibrium(pi, lam):
    """Theorem 3: p†_k = π_k^α / Σ π_j^α, α = λ/(λ-1)"""
    if lam <= 1.0:
        return None  # collapse predicted
    alpha = lam / (lam - 1)
    p_dag = pi ** alpha
    p_dag = p_dag / p_dag.sum()
    return p_dag

# λ별 α 값:
# λ=0.5 → collapse predicted
# λ=0.8 → collapse predicted
# λ=1.0 → collapse predicted
# λ=1.2 → α=6.0   (매우 peaked)
# λ=1.5 → α=3.0
# λ=2.0 → α=2.0   (현재 설정)
# λ=3.0 → α=1.5   (π에 가까움)
```
