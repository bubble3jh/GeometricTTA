# Q1 진단: Text-Derived Entropy Anchor 실현 가능성 검증

**목적:** Text geometry에서 유도한 "이론적 정상 entropy"가 실제 CLIP 예측의 entropy와 얼마나 괴리가 있는지 측정. 이 아이디어가 살아있는지 죽었는지를 실험 전에 판정.
**비용:** Forward pass만, adaptation 없음. ~5분.
**수정 대상 코드:** 없음 (진단 스크립트 신규 작성)

---

## 배경

### 아이디어
Source data 없이 CLIP text embedding만으로 "정상적 예측의 entropy 수준"을 유도:
```
"Class j를 완벽히 맞춘 이미지"의 이론적 logit = cos(t_j, t_k) / tau
→ softmax → entropy → h_j (class j의 이론적 정상 entropy)
→ 배치 평균 entropy가 이 범위에서 벗어나면 벌점
```

### 성립 조건
1. 이론적 entropy (text-only)와 실제 clean entropy (clean CIFAR-10 forward)가 비슷해야 함
2. Clean entropy와 corrupted entropy의 차이가 충분히 커야 signal이 됨
3. Overconfident-wrong (collapse) 상태에서 entropy가 이론적 값보다 낮아야 함

## 측정 항목

### M1: 이론적 entropy (text-only, data 불필요)
```python
# CLIP의 text embedding만으로 계산
tau = model.logit_scale.exp().item()  # CLIP의 learned temperature (보통 ~100, tau≈0.01)

sim = text_features @ text_features.T  # (K, K), 이미 L2-normalized
logits_theory = sim * tau  # CLIP은 logit = cos * logit_scale

for j in range(K):
    p_j = F.softmax(logits_theory[j], dim=0)
    h_j = -(p_j * p_j.log()).sum().item()
    # h_j = "class j를 완벽히 맞춘 이미지의 이론적 entropy"

h_theory_mean = mean(h_j for all j)
h_theory_std = std(h_j for all j)
h_theory_per_class = [h_0, h_1, ..., h_9]
```

### M2: 실제 clean entropy (clean CIFAR-10 test set forward)
```python
# Clean CIFAR-10 test set (corruption 없는 원본) 10,000장을 frozen CLIP으로 forward
# Adaptation 없음, 순수 zero-shot

clean_entropies = []
clean_per_class_entropies = {k: [] for k in range(K)}

for images, labels in clean_dataloader:
    logits = model(images)  # zero-shot, no adaptation
    probs = F.softmax(logits, dim=1)
    H_i = -(probs * probs.log()).sum(dim=1)  # (B,)
    
    for i in range(len(images)):
        clean_entropies.append(H_i[i].item())
        clean_per_class_entropies[labels[i].item()].append(H_i[i].item())

h_clean_mean = mean(clean_entropies)
h_clean_std = std(clean_entropies)
h_clean_per_class = {k: mean(v) for k, v in clean_per_class_entropies.items()}
```

### M3: Corrupted entropy (gaussian_noise sev=5, no adaptation)
```python
# Corrupted CIFAR-10-C gaussian_noise sev=5, 10,000장, frozen CLIP, no adaptation
# BATCLIP도 아님, 순수 zero-shot forward만

corrupted_entropies = []
corrupted_per_class_entropies = {k: [] for k in range(K)}  # true label 기준
corrupted_per_pred_entropies = {k: [] for k in range(K)}   # predicted label 기준

for images, labels in corrupted_dataloader:
    logits = model(images)
    probs = F.softmax(logits, dim=1)
    preds = probs.argmax(dim=1)
    H_i = -(probs * probs.log()).sum(dim=1)
    
    for i in range(len(images)):
        corrupted_entropies.append(H_i[i].item())
        corrupted_per_class_entropies[labels[i].item()].append(H_i[i].item())
        corrupted_per_pred_entropies[preds[i].item()].append(H_i[i].item())

h_corrupt_mean = mean(corrupted_entropies)
h_corrupt_std = std(corrupted_entropies)
```

## 판단 기준

### Gate 1: 이론적 entropy ≈ 실제 clean entropy?
```
|h_theory_mean - h_clean_mean| / h_clean_std < 1.0 → 괴리 작음 (이론이 현실적)
|h_theory_mean - h_clean_mean| / h_clean_std > 2.0 → 괴리 큼 (modality gap 때문에 이론 무의미)
```

### Gate 2: Clean entropy ≠ Corrupted entropy?
```
(h_clean_mean - h_corrupt_mean) / h_clean_std > 0.5 → signal 있음
(h_clean_mean - h_corrupt_mean) / h_clean_std < 0.2 → signal 없음 (entropy 차이가 너무 작아서 anchor로 못 씀)
```

### Gate 3: Corruption 강도와 entropy 차이가 비례?
```
brightness (easy) → entropy 차이 작음
gaussian_noise (hard) → entropy 차이 큼
contrast (medium) → 중간
→ 이 순서가 맞으면 entropy anchor가 corruption severity를 반영
```

### 종합 판단
| Gate 1 | Gate 2 | Gate 3 | 판단 |
|---|---|---|---|
| ✓ | ✓ | ✓ | Entropy anchor 유망. 실험 설계 진행. |
| ✗ | ✓ | ✓ | 이론값 대신 clean forward 값을 쓰면 가능 (하지만 source data 필요) |
| ✓ | ✗ | — | Entropy 차이가 작아서 signal 없음. 방향 기각. |
| ✗ | ✗ | — | 완전 기각. |

### 핵심 주의사항

1. **Adaptation 하지 않음.** 순수 forward만. Model parameter 변경 없음.
   기존 run_mint_tta.py에서 adaptation loop를 빼고 forward만 하면 됨.

2. **logit_scale 사용.** `model.logit_scale.exp()`로 CLIP의 learned temperature를 가져와야 함.
   이론적 entropy 계산에서 이 값을 사용해야 실제와 비교 가능.

3. **Clean CIFAR-10 test set.** `torchvision.datasets.CIFAR10(root='./data', train=False, download=True)`
   이건 corruption 없는 원본. 10,000장, class당 1,000장.

4. **Preprocessing.** CLIP의 image preprocessing을 동일하게 적용해야 함.
   open_clip의 `create_model_and_transforms()`에서 반환하는 preprocess 사용.

5. **Entropy 계산에서 log(0) 방지.** `probs.clamp(min=1e-8)` 후 log.
