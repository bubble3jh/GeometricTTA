# CALM 포괄적 방향 탐색: 12축 가설 검증 대규모 Sweep

**목적:** 지금까지 결과에서 도출 가능한 모든 가설을 체계적으로 검증하여, H(p̄) 없이 collapse를 막을 수 있는 방법을 찾거나, 그것이 불가능함을 확정.
**비용:** ~12시간 연속 실행
**원칙:** 모든 실험은 기존 결과에 근거한 가설 또는 논리적 확장. "무작정 많이"가 아니라 "왜 이걸 해야 하는지" 근거가 있는 것만.

---

## 공통 설정

- Backbone: ViT-B-16 (OpenAI CLIP, QuickGELU), open_clip 2.20.0
- Optimizer: AdamW, lr=1e-3, LayerNorm only
- Seed: 1
- Corruptions: **gaussian_noise sev=5** (전체 실험) + **moderate_skew gaussian** (축 12, conditional)
- Batch size: 200
- Metric: overall accuracy, cat%, H_pbar_final, mean_entropy

## 축 4: Candidate-Masked Inference (Adaptation 없음)

**근거:** E6에서 top-5 recall=80.7%. Mask로 noise class를 제거하면?

q_masked = q_masked / q_masked.sum(1, keepdim=True)  # renormalize

## 축 5: Candidate-Masked Distillation (GPT-Pro 핵심 제안)

**근거:** L_ent 대신 candidate 안에서만 sharpening. 전체 K가 아닌 R개 후보만.

```python
# Candidate soft target
C_i = Top_R(z_i) ∪ Top_R(z_i_flip)  # flip union
q_tilde_ik = 1[k ∈ C_i] * exp(z_ik / tau_c) / Z_i  # candidate 내 softmax
L_distill = CE(q_tilde.detach(), q)  # soft target → student
```

## 축 13: Hinge H(p̄) — Collapse 감지 시에만 H(p̄) 활성화

**근거:** H(p̄)를 항상 쓰면 uniform 강제 → skew에서 해로움. 하지만 collapse가 시작될 때만 H(p̄)를 켜면? "필요할 때만 최소한으로 개입"하는 타협안.

**핵심 아이디어:**
```
H(p̄)의 최댓값 = log(K) = log(10) ≈ 2.303 (uniform일 때)
H(p̄)가 threshold 이하로 떨어지면 = collapse 시작
→ 그때만 H(p̄) gradient를 활성화

L = L_ent + λ · max(0, H_thresh - H(p̄))

H_thresh = log(K) - margin
margin이 크면: 잘 안 켜짐 (skew-safe, 하지만 collapse 방지력 약)
margin이 작으면: 자주 켜짐 (collapse 방지 강, 하지만 skew에서 해로울 수 있음)
```

**왜 H(p̄)와 다른가:**
```
기존 H(p̄): 항상 "uniform으로 가라" → skew에서 minority over-predict
Hinge H(p̄): H(p̄) > threshold이면 아무것도 안 함 → skew에서 자연스러운 분포 유지
             H(p̄) < threshold이면 "collapse를 멈춰라" → balanced에서 collapse 방지
```

**리뷰어 방어:** "Uniform을 항상 강제하지 않고, collapse가 감지될 때만 개입한다. Threshold는 data-free로 설정 가능 (log(K)에서 margin만 결정)."

| Run | margin | H_thresh | λ | 추가항 | Dataset |
|---|---|---|---|---|---|
| M1 | 0.3 | 2.003 | 2.0 | 없음 | balanced |
| M2 | 0.5 | 1.803 | 2.0 | 없음 | balanced |
| M3 | 1.0 | 1.303 | 2.0 | 없음 | balanced |
| M4 | 0.5 | 1.803 | 2.0 | + NCE(w=1,τ=1) | balanced |
| M5 | 0.5 | 1.803 | 2.0 | + Rel(w=1) | balanced |
| M6 | 0.5 | 1.803 | 2.0 | + Flip(w=1) | balanced |

```python
# Hinge H(p̄) 구현
H_thresh = math.log(K) - margin
H_pbar = -(p_bar * p_bar.log()).sum()
L_hinge = lam * F.relu(H_thresh - H_pbar)
loss = L_ent + L_hinge  # + optional 보조항
```

**판단:**
- M2 > 0.63 AND M2 ≈ CALM v1 (0.6458): balanced에서 동등 → skew 검증 (축 12)
- M2 < 0.55: margin 0.5가 너무 관대, collapse 방지 부족
- M4/M5/M6 > M2: 보조항이 hinge 위에서도 도움

### 주의사항

1. **모든 run 시작 전 model reset.** 매 run은 원래 CLIP weight에서 시작.

2. **Candidate mask 실험 (축 4)은 adaptation 없음.** Forward만. 전체 10K를 배치로 나눠서 각 배치 독립 처리.

3. **축 5의 L_distill에서 q̃는 detach.** Soft target은 현재 model의 logit에서 만들되, gradient는 student prediction q를 통해서만 흐름.

4. **축 8/9의 KL(p̄ ∥ π)에서 π는 detach.** Prior는 gradient 흐르지 않게.

5. **축 10에서 L_ent 없이 adaptation이 가능한지 주의.** NCE만 있으면 prototype을 통해 간접적으로 LayerNorm이 업데이트됨. 가능은 하지만 gradient가 약할 수 있어.

6. **축 12의 skew dataset 생성.** 기존 skewed test 코드 (run_skewed_test.py)의 create_skewed_dataset() 재사용. Setting A (moderate skew)만 사용.

7. **Flip forward는 with torch.no_grad().** Flip 이미지에 대한 forward는 gradient 불필요. 메모리 절약.

8. **축 5, 6, 7에서 candidate set C_i 계산.** 매 배치 시작 시 현재 model의 logit으로 top-R 추출. Flip union이면 flip forward도 추가.

9. **축 13 (Hinge H(p̄))에서 H_thresh 계산.** `H_thresh = math.log(K) - margin`. K=10이면 log(10)=2.3026. Margin=0.5면 H_thresh=1.8026. H(p̄)가 이 값 이상이면 loss 기여 = 0 (inactive). 이하이면 `λ * (H_thresh - H(p̄))` 만큼 gradient 발생.

### 조기 종료 규칙

- Adaptation run에서 step 20 (4000장 처리) 시점에 cat% > 85%이면 → 조기 종료 (collapse 확정)
- 이렇게 하면 확실한 실패를 빠르게 기각하고 시간 절약 가능
