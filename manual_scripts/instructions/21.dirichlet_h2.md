# Instruction 21 (Revised): H2 이론적 정당화 실험 + Ablation Variants

## 1. 배경: GPT-Pro 분석 결과 반영

### 핵심 발견: 현재 H2의 이론적 정당화

현재 H2의 evidence prior:
```
π_k ∝ (e_k + α)^β,  α=0.1, β=0.3
```

이것은 **KL barycenter (정보기하적 geometric interpolation)**로 정확히 해석됨:

```
s_k(α) = (e_k + α) / (R + Kα)     ← smoothed evidence distribution
u_k = 1/K                           ← uniform

π(α,β) = argmin_π [β · KL(π ∥ s(α)) + (1-β) · KL(π ∥ u)]

해: π_k ∝ s_k(α)^β · u_k^(1-β) = s_k(α)^β / K^(1-β)
```

**따라서:**
- β는 heuristic exponent가 아니라 **evidence-vs-uniform trust weight**
- KL(p̄ ∥ π) = β·KL(p̄ ∥ s(α)) + (1-β)·KL(p̄ ∥ u) + C (p̄-independent constant)
- **H2의 batch regularizer는 정확히 "evidence matching + uniform balancing"의 가중합**

### 결론: 현재 H2를 교체할 필요 없음

현재 코드 그대로가 이론적으로 가장 깔끔. Dirichlet posterior는 β=1 special case로 ablation에 넣을 것.

### 이번 실험의 목적

1. **β의 역할 검증:** β=1 (linear shrinkage, Dirichlet-like) vs β=0.3 (current, log-odds compression) 비교
2. **Weak-label variant 탐색:** GPT-Pro 제안의 soft count + Dirichlet 형태
3. **α sensitivity 확인** (current H2와 weak-label 둘 다)
4. **Variance-aware adaptive shrinkage** 가능성 확인

## 3. Evidence Prior 구현: 3가지 Variant

### Variant 0: Current H2 (baseline, 이미 결과 있음)

```python
def evidence_prior_current(logits, R=5, alpha=0.1, beta=0.3):
    """현재 H2: (e_k + α)^β → normalize"""
    B, K = logits.shape
    topR = logits.topk(R, dim=1).indices
    e = torch.zeros(K, device=logits.device)
    for k in range(K):
        e[k] = (topR == k).any(dim=1).float().mean()  # 비율 (0~1)
    pi = (e + alpha).pow(beta)
    pi = pi / pi.sum()
    return pi
```

### Variant 1: β=1 (Linear Shrinkage / Dirichlet-like)

```python
def evidence_prior_linear(logits, R=5, alpha=0.1):
    """β=1: π_k ∝ (e_k + α)^1 = (e_k + α) / Σ(e_j + α)
    = s_k(α) = smoothed evidence distribution 그 자체.
    Log-odds compression 없음. Linear shrinkage만."""
    B, K = logits.shape
    topR = logits.topk(R, dim=1).indices
    e = torch.zeros(K, device=logits.device)
    for k in range(K):
        e[k] = (topR == k).any(dim=1).float().mean()
    pi = e + alpha
    pi = pi / pi.sum()
    return pi
```

**이것은 current H2에서 β=1로 바꾼 것. 나머지 전부 동일.**
**비교 목적: β<1의 log-odds compression이 필요한지 확인.**

### Variant 2: Weak-label Dirichlet (GPT-Pro 추천)

```python
def evidence_prior_weaklabel(logits, R=5, alpha_D=10.0, tau_e=1.0):
    """Top-R candidate 내에서 softmax weight로 soft count 생성.
    Σ_k c_k = B (한 sample의 weight 합 = 1).
    Dirichlet posterior mean: π_k = (c_k + α) / (B + Kα).
    
    tau_e: evidence temperature. 
      tau_e 작으면 → top-1에 집중 (hard label에 가까움)
      tau_e 크면 → top-R 내에서 균등 (binary indicator에 가까움)
    """
    B, K = logits.shape
    topR_vals, topR_idx = logits.topk(R, dim=1)  # (B, R)
    
    # Per-sample soft weight over top-R candidates
    # w_ik = 1[k ∈ S_i] * exp(z_ik / τ_e) / Σ_{j∈S_i} exp(z_ij / τ_e)
    w = torch.zeros(B, K, device=logits.device)
    topR_probs = F.softmax(topR_vals / tau_e, dim=1)  # (B, R) softmax within top-R
    for r in range(R):
        w.scatter_add_(1, topR_idx[:, r:r+1], topR_probs[:, r:r+1])
    # w: (B, K), w[i].sum() = 1.0 for all i
    
    # Soft count
    c = w.sum(dim=0)  # (K,), Σ c_k = B
    
    # Dirichlet posterior mean
    pi = (c + alpha_D) / (B + K * alpha_D)
    
    return pi
```

**핵심 차이점:**
```
Current H2:   e_k = binary indicator (top-R에 있으면 1, 없으면 0) → mean → power
Variant 2:    w_ik = softmax within top-R → soft count → linear shrinkage

Current:  Σ e_k = R (비율이므로 sum ≈ R/K × K 아님, 각 e_k ∈ [0,1])
Variant 2: Σ c_k = B (정확히 B)
```

### Variant 3: Weak-label + Tempering (통합 family)

```python
def evidence_prior_weaklabel_tempered(logits, R=5, alpha_D=10.0, tau_e=1.0, beta=0.3):
    """Variant 2의 soft count + current H2의 β tempering.
    π_k ∝ ((c_k + α) / (B + Kα))^β
    
    GPT-Pro의 "가장 자연스러운 다음 단계":
    c_k = Σ_i w_ik (weak-label count)
    s_k = (c_k + α) / (B + Kα) (Dirichlet posterior mean)
    π = T_β(s) = KL barycenter of s and uniform
    """
    B, K = logits.shape
    topR_vals, topR_idx = logits.topk(R, dim=1)
    
    w = torch.zeros(B, K, device=logits.device)
    topR_probs = F.softmax(topR_vals / tau_e, dim=1)
    for r in range(R):
        w.scatter_add_(1, topR_idx[:, r:r+1], topR_probs[:, r:r+1])
    
    c = w.sum(dim=0)  # (K,)
    s = (c + alpha_D) / (B + K * alpha_D)  # Dirichlet posterior mean
    
    # Tempering (KL barycenter)
    pi = s.pow(beta)
    pi = pi / pi.sum()
    
    return pi
```

### Phase D: Variance-aware Adaptive Shrinkage (2 runs, ~14분)

**ρ를 배치의 evidence variance에서 자동 결정.**

| Run | Method | 목적 |
|---|---|---|
| VA1 | Current H2 + adaptive ρ (V0 구조) | Evidence variance 기반 α 자동 결정 |
| VA2 | Weak-label + adaptive ρ (V2 구조) | Weak-label에서의 adaptive ρ |

```python
def evidence_prior_adaptive(logits, R=5, beta=0.3):
    """Variance-aware adaptive shrinkage.
    ρ = V / ||w̄ - u||² where V = sample variance of per-sample evidence.
    """
    B, K = logits.shape
    topR = logits.topk(R, dim=1).indices
    
    # Per-sample binary evidence vector
    w = torch.zeros(B, K, device=logits.device)
    for i in range(B):
        for r in range(R):
            w[i, topR[i, r]] = 1.0
    w = w / w.sum(dim=1, keepdim=True)  # normalize to simplex (Σ_k w_ik = 1)
    
    # Batch mean evidence
    w_bar = w.mean(dim=0)  # (K,)
    u = torch.ones(K, device=logits.device) / K
    
    # Variance: mean per-sample squared distance from batch mean
    V = ((w - w_bar.unsqueeze(0)).pow(2).sum(dim=1)).mean() / (B - 1)
    
    # Squared distance from uniform
    D = (w_bar - u).pow(2).sum()
    
    # Adaptive rho
    rho = (V / (D + 1e-8)).clamp(0, 1).item()
    
    # Convert to π
    s = (1 - rho) * w_bar + rho * u  # linear shrinkage
    
    # Apply tempering
    pi = s.pow(beta)
    pi = pi / pi.sum()
    
    return pi, rho  # rho도 반환해서 로깅
```

## 5. Adaptation Loop (공통)

모든 run에서 동일한 adaptation loop. Evidence prior 함수만 교체.

```python
model = load_pretrained_clip()
model, params = configure_model(model)  # image + text LN
optimizer = AdamW(params, lr=1e-3)
K = 10

online_correct = 0
online_total = 0

for step, (images, labels) in enumerate(batches):
    logits = model(images)  # (B, K)
    probs = F.softmax(logits, dim=1)
    
    preds = logits.argmax(dim=1)
    online_correct += (preds == labels).sum().item()
    online_total += len(labels)
    
    # 1. Entropy
    L_ent = -(probs * probs.log()).sum(dim=1).mean()
    
    # 2. Evidence prior (variant에 따라 함수 교체)
    pi_evid = evidence_prior_VARIANT(logits, ...)  # detach 불필요 (pi는 logits에서 계산되지만 detach해야 함)
    pi_evid = pi_evid.detach()  # target은 gradient에서 분리
    
    # 3. Batch marginal
    p_bar = probs.mean(dim=0)
    
    # 4. KL
    L_kl = F.kl_div(p_bar.log(), pi_evid, reduction='sum')
    
    # 5. Total
    loss = L_ent + 2.0 * L_kl  # λ=2.0 고정
    
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    
    # Step log (매 5 step)
    if (step + 1) % 5 == 0:
        cat_pct = (preds == 3).float().mean().item()
        ent = -(probs * probs.log()).sum(dim=1).mean().item()
        H_pbar = -(p_bar * p_bar.log()).sum().item()
        uniform = torch.ones(K, device=logits.device) / K
        pi_l1 = (pi_evid - uniform).abs().sum().item()
        print(f"Step {step+1}: online_acc={online_correct/online_total:.4f}, "
              f"cat%={cat_pct:.3f}, ent={ent:.3f}, H_pbar={H_pbar:.3f}, "
              f"pi_L1={pi_l1:.4f}, loss={loss.item():.4f}")

# Offline evaluation
model.eval()
# ... (기존과 동일)
```

**주의: pi_evid.detach() 필수.** 모든 variant에서 동일하게 detach.

## 8. 판단 기준 종합

```
가장 중요한 질문: "β < 1이 필요한가?"

B2 (β=1) ≈ B1 (β=0.3):
  → β 불필요 → method가 더 단순해짐 (HP 하나 줄어듦)
  → 논문: "linear shrinkage만으로 충분"
  → Dirichlet 해석이 정확해짐

B2 < B1 (-2pp+):
  → β < 1이 필요 → contaminated evidence에 대한 tempering
  → 논문: "KL barycenter의 β가 evidence reliability를 반영"
  → 현재 H2 유지, β의 역할을 실험적으로 정당화

두 번째 질문: "Weak-label이 binary indicator보다 나은가?"

W1~W3 best ≈ H2:
  → Soft count가 binary count와 동등
  → 이론적으로 더 깨끗한 weak-label 사용 가능

W1~W3 < H2:
  → Binary indicator의 "information bottleneck"이 오히려 도움
  → Top-R의 binary 특성이 collapse robustness에 기여

W4 (weak-label + temper) > W1 (weak-label only):
  → Tempering이 weak-label에서도 필요
  → β < 1의 범용적 중요성

세 번째: "α에 sensitive한가?"

A1~A5에서 ±1pp: → insensitive → 최고의 방어
특정 값에서만 작동: → sensitive → 약점

네 번째: "Adaptive ρ가 고정보다 나은가?"

VA1 ≈ A3 (best fixed α): → adaptive 불필요 → "robust to α" 방어가 더 강함
VA1 > A3: → adaptive가 도움 → 하지만 complexity 증가
```

## 9. 주의사항

1. **모든 run에서 model reset.** 원래 CLIP weight에서 시작.

2. **B1 (current H2)과 A3은 같은 설정.** 둘 다 α=0.1, β=0.3. 하나만 돌리면 됨.

3. **Variant 2/3의 w 계산에서 scatter_add_ 사용.** Efficient하지만 정확한지 확인 필요. 대안:
```python
# 명시적 loop (느리지만 확실)
w = torch.zeros(B, K, device=logits.device)
for i in range(B):
    candidates = topR_idx[i]  # (R,)
    candidate_logits = logits[i, candidates]  # (R,)
    candidate_probs = F.softmax(candidate_logits / tau_e, dim=0)  # (R,)
    for r in range(R):
        w[i, candidates[r]] += candidate_probs[r]
```

4. **VA (adaptive)에서 rho를 매 step 출력.** Rho가 수렴하는지, 발산하는지 관찰.

5. **τ_e (evidence temperature)는 weak-label variant에서만 사용.** Current H2에는 해당 없음. τ_e=1.0으로 고정 (logit scale 그대로 사용).
