## Instruction 19: Batch Mean Logit Centering Exploration

### 배경

L_ent가 collapse하는 메커니즘: 배치에서 cat이 top-1인 샘플이 다수 → L_ent gradient가 cat 방향으로 coherent → LayerNorm이 cat으로 shift → 양성 피드백.

**핵심 아이디어:** logit에서 batch mean을 빼면, 배치 전체가 공유하는 bias (cat이 높은 것)가 제거되고, 각 샘플 고유의 class 신호만 남아서 sharpen. 새 HP 없이, 분포 가정 없이 collapse를 막을 수 있는지 탐색.

```
기존 L_ent:
  dog image logit: [0.3, 0.1, 0.1, 0.8, 0.5, ...]  → cat(0.8) top-1 → cat 올림

Batch mean centering:
  batch mean logit: [0.2, 0.1, 0.1, 0.6, 0.2, ...]  (cat이 전체적으로 높음)
  centered logit:   [0.1, 0.0, 0.0, 0.2, 0.3, ...]  → dog(0.3) > cat(0.2) 가능!
  → 공통 bias 제거 후 각 샘플의 고유 신호로 sharpen
```

### Loss 구현

```python
def batch_centered_entropy_loss(logits, tau=1.0):
    """
    Batch mean logit을 빼고 entropy minimization.
    
    logits: (B, K) — model의 raw logits
    tau: softmax temperature
    
    핵심: logits.mean(dim=0)이 batch 공통 bias (cat이 높은 것).
    이걸 빼면 각 sample 고유의 class 신호만 남음.
    """
    logits_centered = logits - logits.mean(dim=0, keepdim=True)  # (B, K)
    probs = F.softmax(logits_centered / tau, dim=1)
    loss = -(probs * probs.log()).sum(dim=1).mean()
    return loss

# K1:
loss = batch_centered_entropy_loss(logits, tau=1.0)

# K2:
L_bce = batch_centered_entropy_loss(logits, tau=1.0)
L_rel = relational_loss(...)  # J3과 동일 구현 (run_comprehensive_sweep.py의 axis10 J3 참조)
loss = L_bce + 1.0 * L_rel

# K3:
loss = batch_centered_entropy_loss(logits, tau=0.5)

# K4:
loss = batch_centered_entropy_loss(logits, tau=2.0)
```

**L_rel 구현은 기존 comprehensive sweep의 J3과 동일:**
```python
# text_features: (K, D), frozen
t_bar = text_features.mean(0)
Delta_t = F.normalize(text_features - t_bar, dim=1)
tau_t = 1.0
r_k = F.softmax(Delta_t @ Delta_t.T / tau_t, dim=1)  # (K, K) text relational target

# prototype 계산 (현재 batch의 logits에서)
q = F.softmax(logits, dim=1)  # raw logits 사용
f = image_features  # model.encode_image(images)
m_k = (q.T @ f) / (q.sum(0).unsqueeze(1) + 1e-8)  # (K, D)
m_bar = m_k.mean(0)
Delta_m = F.normalize(m_k - m_bar, dim=1)
tau_nce = 1.0
p_k = F.softmax(Delta_m @ Delta_t.T / tau_nce, dim=1)  # (K, K)

L_rel = sum(F.kl_div(p_k[k].log(), r_k[k], reduction='sum') for k in range(K)) / K
```

**중요:** Prediction (accuracy 계산)은 **원래 raw logits**로 함. `preds = logits.argmax(dim=1)`. Centered logits는 loss 계산에만 사용.

### 측정 지표

**모든 run에서 아래 전부 측정.** acc만이 아니라 "왜 되는지/안 되는지"를 진단하기 위한 지표.

```python
# 매 5 step마다 기록:
step_metrics = {
    "step": int,
    "online_acc": float,          # 누적 accuracy (correct/seen)
    "batch_acc": float,           # 이번 배치만의 accuracy
    "cat_pct": float,             # 이번 배치에서 cat 예측 비율
    "mean_entropy_raw": float,    # raw logits의 softmax entropy 평균
    "mean_entropy_centered": float, # centered logits의 softmax entropy 평균
    "H_pbar": float,              # marginal entropy (raw logits 기반)
    "logit_mean_cat": float,      # batch mean logit의 cat (class 3) 값
    "logit_mean_std": float,      # batch mean logit의 std (class 간 편차)
    "centered_top1_match_raw": float,  # centered top-1과 raw top-1이 같은 비율
    "margin_raw": float,          # raw logits의 top1-top2 margin 평균
    "margin_centered": float,     # centered logits의 top1-top2 margin 평균
}
```

**핵심 진단 지표 설명:**

1. **`cat_pct` (step별):** Collapse 진행 여부. Step마다 증가하면 collapse, 감소하면 anti-collapse 작동.

2. **`mean_entropy_raw` vs `mean_entropy_centered`:** Raw entropy가 낮은데 centered entropy가 높으면 → "raw에서는 confident하지만 공통 bias 때문이지, 진짜 confident한 게 아님." Raw와 centered 모두 낮으면 → 진짜로 sharp해진 것.

3. **`logit_mean_cat` (step별):** Batch mean logit에서 cat 값의 변화. 이게 증가하면 → cat bias가 adaptation 중 커지고 있음. 감소하면 → bias 해소.

4. **`logit_mean_std`:** Batch mean logit의 class 간 standard deviation. 이게 크면 → 배치 전체가 특정 class로 biased. 작으면 → 배치가 균등.

5. **`centered_top1_match_raw`:** Centered 후 top-1이 raw top-1과 다른 비율. 높으면 → centering이 안 바꿈 (bias가 약하거나 이미 정확). 낮으면 → centering이 많은 sample의 예측을 바꿈 (bias가 강했음). **이 지표가 "batch mean centering이 실제로 예측을 바꾸는가"를 직접 보여줌.**

6. **`margin_raw` vs `margin_centered`:** Raw에서 margin이 작고 centered에서 margin이 크면 → centering이 discriminability를 개선. 둘 다 작으면 → centering으로도 부족.

```python
# 측정 구현
def compute_diagnostics(logits_raw, labels, tau=1.0):
    B, K = logits_raw.shape
    
    # Centering
    logits_centered = logits_raw - logits_raw.mean(dim=0, keepdim=True)
    
    # Raw predictions
    probs_raw = F.softmax(logits_raw, dim=1)
    preds_raw = logits_raw.argmax(dim=1)
    
    # Centered predictions
    probs_centered = F.softmax(logits_centered / tau, dim=1)
    preds_centered = logits_centered.argmax(dim=1)
    
    # Accuracies
    batch_acc = (preds_raw == labels).float().mean().item()
    
    # Cat percentage (raw prediction 기준)
    cat_pct = (preds_raw == 3).float().mean().item()
    
    # Entropies
    mean_entropy_raw = -(probs_raw * probs_raw.log()).sum(1).mean().item()
    mean_entropy_centered = -(probs_centered * probs_centered.log()).sum(1).mean().item()
    
    # Marginal entropy
    p_bar = probs_raw.mean(0)
    H_pbar = -(p_bar * p_bar.log()).sum().item()
    
    # Batch mean logit analysis
    logit_mean = logits_raw.mean(dim=0)  # (K,)
    logit_mean_cat = logit_mean[3].item()
    logit_mean_std = logit_mean.std().item()
    
    # Centered vs raw top-1 match
    centered_top1_match = (preds_centered == preds_raw).float().mean().item()
    
    # Margins
    top2_raw = logits_raw.topk(2, dim=1).values
    margin_raw = (top2_raw[:, 0] - top2_raw[:, 1]).mean().item()
    
    top2_centered = logits_centered.topk(2, dim=1).values
    margin_centered = (top2_centered[:, 0] - top2_centered[:, 1]).mean().item()
    
    return {
        "batch_acc": batch_acc,
        "cat_pct": cat_pct,
        "mean_entropy_raw": mean_entropy_raw,
        "mean_entropy_centered": mean_entropy_centered,
        "H_pbar": H_pbar,
        "logit_mean_cat": logit_mean_cat,
        "logit_mean_std": logit_mean_std,
        "centered_top1_match_raw": centered_top1_match,
        "margin_raw": margin_raw,
        "margin_centered": margin_centered,
    }
```

**K1 (핵심 실험):**
- `cat_pct`가 step에 따라 감소하거나 안정적 (< 30%): → collapse 방지 작동 ✓
- `cat_pct`가 step에 따라 증가 (> 50%): → collapse 여전히 발생 ✗
- `online_acc` > 0.55: → BATCLIP에 근접, sharpening 효과 있음 ✓✓
- `online_acc` > 0.60: → BATCLIP 돌파 ✓✓✓
- `centered_top1_match_raw` < 0.7 (초반 step): → centering이 실제로 예측을 바꾸고 있음 (bias 제거 확인)
- `margin_centered` > `margin_raw`: → centering이 discriminability 개선

**K2 (K1 + Rel):**
- K2 > K1: → Rel이 centering 위에서 추가 기여 ✓
- K2 ≈ K1: → Rel이 redundant

- 어떤 τ에서 가장 좋은지 → τ의 sensitivity 파악

### 주의사항

1. **모든 run 시작 전 model reset.** 매 run은 원래 CLIP weight에서 시작.
2. **Prediction은 raw logits로.** Loss만 centered logits로. `preds = raw_logits.argmax(1)`.
3. **logits.mean(dim=0)은 매 배치마다 새로 계산.** 이전 배치의 mean을 쓰지 않음. 현재 배치의 mean만 사용.
4. **L_rel (K2)에서 prototype 계산은 raw logits 기반.** `q = softmax(raw_logits)`. Centered logits가 아님.
5. **조기 종료: step 20에서 cat% > 80%이면 collapse 확정, 해당 run 종료.**
