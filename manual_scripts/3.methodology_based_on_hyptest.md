# Task: Implement and Ablate "Trusted Set" TTA Pipeline

## 1. Background & Hypothesis Testing Summary
Recent diagnostic tests on BATCLIP under severe additive noise (CIFAR-10-C, severity 5) revealed critical failure modes:
1. The mean resultant length (`r_bar_k`) fails as a class reliability proxy (Spearman rho = -0.291) because noise corruptions generate a high volume (~25%) of "overconfident-wrong" samples.
2. Inter-class variance (`Var_inter`) collapses significantly under noise (up to 3.5x drop), which correlates almost perfectly with accuracy degradation (rho = 0.957).
3. Margin-based confidence (`q_k`) is a strictly superior correctness predictor compared to geometric concentration (`r_bar_k`).

**Conclusion:** We must abandon `r_bar_k`. To safely restore `Var_inter`, we must first construct a "Trusted Set" of samples using a dual-condition filter (Margin + an independent axis) to eliminate the 25% overconfident-wrong poison. We then compute robust class prototypes exclusively from this Trusted Set and apply the InterMean separation loss as a floor weight.

## 2. Methodology & Ablation Design

Implement the following modular TTA pipeline. Steps A and B are grouped into 3 distinct Options for algorithmic ablation to determine the most effective filtering axis. Step C is applied universally to the outputs of Step B.

### Step A & B: Trusted Set Construction & Prototype Accumulation
For each incoming test batch, construct the Trusted Set (T) and compute the class prototype (`mu_k`).

**Baseline Constraint:** All methods must first pass the Margin Gate: 
`Condition 1: margin_i > tau_margin`

**[Option 1] Cross-Modal Consistency Filter (I2T Agreement)**
* **Step A (Filter):** Add `Condition 2: argmax(x_i @ mu_k_ema) == argmax(x_i @ t_k)`
  * *Concept:* The pseudo-label assigned by the moving image centroid must match the zero-shot prediction from the frozen text anchor.
* **Step B (Prototype):** For samples in T, compute batch mean `mu_k_batch`. Update EMA: 
  `mu_k_ema = alpha * mu_k_ema_prev + (1 - alpha) * mu_k_batch`.

**[Option 2] Multi-View Consistency Filter**
* **Step A (Filter):** Generate N augmented views of `x_i`.
  Add `Condition 2: majority_vote(preds_aug) == base_pred`
  * *Concept:* Reliable samples should be structurally robust to data augmentation.
* **Step B (Prototype):** Same EMA update as Option 1 using filtered samples.

**[Option 3] Parametric vs. Non-Parametric Filter (kNN Cache)**
* **Step A (Filter):** Maintain a feature cache of the current shift.
  Add `Condition 2: argmax(x_i @ t_k) == kNN_predict(x_i, cache)`
  * *Concept:* Parametric (text) and non-parametric (feature cache) classifiers must agree.
* **Step B (Prototype):** Same EMA update as Option 1 using filtered samples.

### Step C: Objective Function (Loss) Computation
Using ONLY the trusted prototypes (`mu_k_ema`), compute the adaptation loss.

1. **Class-level Confidence (`q_k`):** Compute the mean margin of the trusted samples assigned to class `k`.
2. **I2T Alignment:** `L_I2T = - sum_k [ q_k * cos(mu_k_ema, t_k) ]`
3. **Var_inter Preservation:** `L_InterMean` = BATCLIP's inter-class repulsion term, applied directly to `mu_k_ema`.
4. **Total Loss:** `L_total = Entropy(p_i) + lambda_1 * L_I2T + lambda_2 * L_InterMean`
